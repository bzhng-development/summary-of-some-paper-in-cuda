## 1. Executive Summary
This paper addresses the high memory I/O cost of cross-attention layers during long generation in hybrid architectures by introducing the Gated Memory Unit (GMU), a mechanism that shares memory readout states across layers to create the `SambaY` decoder-hybrid-decoder architecture. Their largest model, `Phi4-mini-Flash-Reasoning` (3.8B parameters), achieves up to 10× higher decoding throughput on 2K-length prompts with 32K generation length under the vLLM framework while outperforming the strong `Phi4-mini-Reasoning` baseline on reasoning tasks like Math500 and AIME24/25 without using Reinforcement Learning. Extensive scaling experiments up to 3.4B parameters and 600B tokens demonstrate that `SambaY` exhibits a significantly lower irreducible loss ($C=0.58$) compared to YOCO baselines, indicating superior performance scalability under large-scale compute regimes while maintaining linear pre-filling time complexity.

## 2. Context and Motivation

### The Efficiency Challenge in Long-Context Language Models

Modern large language models face a fundamental tension between **capability** and **efficiency**. As models are tasked with increasingly complex reasoning problems—such as mathematical proofs, code generation, or scientific question-answering—they must generate lengthy Chains-of-Thought (CoTs) that can span tens of thousands of tokens. The paper identifies a critical bottleneck: while recent architectural innovations have improved how models *process* long input prompts, the *generation* phase remains prohibitively expensive in terms of memory I/O operations.

This matters for both practical and theoretical reasons:

- **Real-world impact:** Reasoning models that produce 32K+ token generations require substantial computational resources during inference. A 10× improvement in decoding throughput directly translates to lower serving costs and faster response times for end users.
- **Theoretical significance:** The problem exposes a gap in how hybrid architectures share information across layers. If representations computed in early layers could be reused efficiently in later layers without expensive attention operations, models could scale more effectively.

### The Landscape of Sequence Modeling Architectures

To understand where this paper fits, we need to trace the evolution of sequence modeling approaches:

**Pure Transformers** established the dominant paradigm with self-attention mechanisms that enable powerful in-context learning. However, attention has quadratic complexity $O(n^2)$ with respect to sequence length $n$, making long-context processing expensive. During generation, each new token requires reading the entire Key-Value (KV) cache from memory, creating an I/O bottleneck.

**State Space Models (SSMs)** emerged as an alternative, offering linear complexity $O(n)$ for both training and inference. Models like Mamba and modern RNN variants (e.g., RWKV, xLSTM) achieve this through recurrent-style updates that maintain a fixed-size hidden state. However, pure SSMs exhibit a **theoretical expressiveness gap** relative to Transformers on in-context retrieval tasks—they struggle to selectively attend to specific tokens from distant positions in the sequence.

**Hybrid Architectures** attempt to bridge this gap by combining SSMs with attention mechanisms. For example:
- **Samba** interleaves Mamba layers with Sliding Window Attention (SWA), achieving linear complexity while improving retrieval capability.
- **YOCO (You Only Cache Once)** introduces a decoder-decoder architecture where the first half of layers (self-decoder) processes the input and produces a single KV cache from one full-attention layer. The second half (cross-decoder) then reuses this cache through cross-attention operations, avoiding repeated KV computation.

### Where Prior Approaches Fall Short

The paper identifies a specific limitation in YOCO that becomes critical for long-generation scenarios:

> "YOCO has delivered substantial efficiency gains when processing user prompts with long sequences, but challenges remain; it does not mitigate the attention memory I/O cost for its cross-attentions during the generation stage of the model responses."

This distinction is subtle but important. YOCO reduces computation during the **prefill stage** (processing the input prompt) by computing the KV cache only once. However, during **generation** (producing output tokens token-by-token), each cross-attention layer in the cross-decoder must still read the full KV cache from memory. For a model generating 32K tokens, this memory I/O cost accumulates significantly.

Consider the memory access pattern: if the KV cache has dimension $d_{kv}$ and the sequence length is $N$, each cross-attention layer incurs $O(d_{kv} N)$ memory I/O per generated token. With multiple cross-attention layers, this becomes a dominant cost factor—especially since modern GPUs are often memory-bandwidth bound rather than compute-bound during inference.

Additionally, prior hybrid architectures have not investigated **representation sharing between SSM layers**. While YOCO shares KV caches across cross-attention layers, it does not explore whether the internal states from SSM layers could be shared more efficiently through lighter-weight mechanisms.

### The Paper's Positioning

This work positions itself at the intersection of three research directions:

1. **Efficient long-context architectures:** Building on YOCO's decoder-decoder design but addressing its generation-stage inefficiency.
2. **SSM-Transformer hybrids:** Leveraging Samba's combination of Mamba and attention while adding a novel memory-sharing mechanism.
3. **Reasoning-optimized models:** Targeting the specific workload pattern of long CoT generation rather than general-purpose language modeling.

The key innovation is the **Gated Memory Unit (GMU)**, which replaces approximately 50% of the expensive cross-attention layers in YOCO's cross-decoder. Instead of performing full attention over the KV cache, GMUs apply element-wise gating to share representations from the final SSM layer in the self-decoder. This reduces memory I/O complexity from $O(d_{kv} N)$ to $O(d_h)$ per layer during decoding, where $d_h$ is the hidden dimension—typically much smaller than the sequence length $N$ for long generations.

The paper explicitly contrasts its approach with naive combinations like **Samba+YOCO** (which simply uses Samba as the self-decoder in YOCO without GMUs), demonstrating that the GMU mechanism itself—not just the choice of self-decoder—drives the efficiency gains.

By eliminating the need for explicit positional encodings (e.g., RoPE) and maintaining linear pre-filling complexity while improving decoding throughput, the architecture targets the specific bottleneck of **long-generation reasoning workloads** rather than general language modeling tasks. This focused positioning allows the authors to optimize for a well-defined use case: models that must produce extensive reasoning traces efficiently.

## 3. Technical Approach

This is an architecture design paper that introduces a novel memory-sharing mechanism called the Gated Memory Unit (GMU) to reduce decoding costs in hybrid language models. The core idea is to replace expensive cross-attention operations with lightweight element-wise gating that reuses representations computed earlier in the network, cutting memory I/O by approximately 50% during long generation without sacrificing retrieval capability.

### 3.1 Reader orientation (approachable technical breakdown)

The system being built is a language model architecture called `SambaY` that processes input prompts efficiently and generates long reasoning traces with substantially reduced memory bandwidth requirements. The solution takes the form of a decoder-hybrid-decoder structure where the first half of layers computes rich representations once, and the second half reuses those representations through cheap gating operations instead of repeatedly reading large attention caches from memory.

### 3.2 Big-picture architecture (diagram in words)

The `SambaY` architecture consists of three major components arranged sequentially. First, the **self-decoder** (approximately the first half of layers) processes input tokens using interleaved Mamba SSM layers and Sliding Window Attention (SWA), culminating in a single full-attention layer that produces a Key-Value (KV) cache. Second, the **cross-decoder** (approximately the second half of layers) receives the self-decoder's output and generates final predictions using a mixture of cross-attention layers and Gated Memory Units. Third, the **Gated Memory Unit (GMU)** itself acts as a lightweight alternative to cross-attention, taking the current layer's input and a cached representation from the self-decoder's final SSM layer, then applying learned element-wise gating to produce an output without reading the full KV cache. Information flows from input tokens through the self-decoder (where KV caches are computed once during prefill), then through the cross-decoder (where half the layers use GMUs instead of cross-attention), finally reaching the output projection layer for token prediction.

### 3.4 Detailed, sentence-based technical breakdown

#### The Gated Memory Unit (GMU) Mechanism

The Gated Memory Unit operates on two inputs to produce a gated output that reuses previously computed representations. At a given layer $l$ in the cross-decoder, the GMU receives the current layer's input hidden state $X_l \in \mathbb{R}^{n \times d_m}$ and a mixed representation $M^{(l')} \in \mathbb{R}^{n \times d_h}$ from a previous layer $l'$ in the self-decoder (where $l' &lt; l$). The GMU produces an output $Y_l \in \mathbb{R}^{n \times d_m}$ through the following computation:

$$Y_l = (M^{(l')} \odot \sigma(X_l W_1^T)) W_2$$

In plain language, this equation means the GMU first projects the current input through a learned weight matrix $W_1$, applies a SiLU activation function $\sigma(\cdot)$ to create gating values between 0 and 1, then multiplies these gates element-wise with the cached memory representation $M^{(l')}$, and finally projects the result through a second weight matrix $W_2$. The symbol $\odot$ denotes element-wise multiplication, and both $W_1, W_2 \in \mathbb{R}^{d_h \times d_m}$ are learnable weight matrices.

The paper also introduces a normalized variant called nGMU that applies RMSNorm after the element-wise multiplication but before the final projection. This normalization is crucial when the memory originates from linear attention mechanisms, as detailed in Section B and Section H of the paper. The normalized GMU computes:

$$Y_l = \text{RMSNorm}(M^{(l')} \odot \sigma(X_l W_1^T)) W_2$$

To understand why GMU is more efficient than cross-attention, consider what each operation must read from memory. A cross-attention layer must read the full KV cache of size $O(d_{kv} N)$ where $N$ is the sequence length and $d_{kv}$ is the key-value dimension. In contrast, GMU only reads the cached SSM state of size $O(d_h)$ where $d_h$ is the hidden dimension—typically much smaller than $N$ for long sequences. The paper states that $d_h / d_{kv}$ typically does not exceed 128, meaning the memory I/O reduction becomes substantial when $N \gg d_h / d_{kv}$, a condition easily met in practice for long-generation scenarios.

The mathematical insight behind GMU reveals how it reweights previous token mixing without recomputing attention. For each element $H_{ik}$ in the gated output $H = M^{(l')} \odot G^{(l)}$ where $G^{(l)} = \sigma(X_l W_1^T)$, we can expand:

$$H_{ik} = G^{(l)}_{ik} \sum_j A^{(l')}_{ij} V^{(l')}_{jk} = \sum_j G^{(l)}_{ik} A^{(l')}_{ij} V^{(l')}_{jk} = \sum_j \tilde{A}_{ijk} V^{(l')}_{jk}$$

where $\tilde{A}_{ijk} = G^{(l)}_{ik} A^{(l')}_{ij}$ represents a third-order tensor. This shows that the gate $G^{(l)}$ effectively lifts the previous token mixing matrix $A^{(l')}$ into a channel-specific reweighting while maintaining linearity on the original value matrix $V^{(l')}$ at a later layer $l$. In practical terms, this means GMU can dynamically recalibrate which parts of the previously mixed representation matter for the current layer's computation, conditioned on the current input, without paying the full cost of attention.

#### The SambaY Decoder-Hybrid-Decoder Architecture

Figure 1 illustrates the complete SambaY architecture, which applies the GMU mechanism within the YOCO decoder-decoder framework using Samba as the self-decoder backbone. The architecture divides layers into two distinct phases with different computational characteristics. The self-decoder comprises the first half of layers and uses Samba (interleaved Mamba SSM and Sliding Window Attention) for token mixing, with a single full-attention layer at the end that computes and caches KV pairs during the prefill stage. The cross-decoder comprises the second half of layers and traditionally would use only cross-attention to attend to the cached KV pairs, but SambaY replaces approximately 50% of these cross-attention layers with GMUs.

The memory caching strategy differs between prefill and generation stages. During prefill (processing the input prompt), the self-decoder computes the full KV cache from its final attention layer and also caches the SSM kernel output state $m \in \mathbb{R}^{d_h}$ from the final Mamba layer, where $d_h = 2d_m$. This SSM state overhead is negligible in size compared to the KV cache. During generation (producing output tokens one-by-one), the cross-decoder reuses these cached representations, with GMU layers reading only the small SSM state instead of the full KV cache.

The paper explicitly contrasts SambaY with a naive combination called Samba+YOCO, which uses Samba as the self-decoder in the original YOCO architecture without any GMU replacements. This comparison isolates the contribution of the GMU mechanism itself from the choice of self-decoder backbone. The results show that SambaY achieves significantly lower irreducible loss compared to Samba+YOCO when scaling with training FLOPs, indicating that the memory-sharing mechanism—not just the SSM backbone—drives the architectural improvements.

#### Memory I/O Complexity Analysis

The efficiency gains from GMU can be quantified precisely in terms of memory I/O complexity during the decoding stage. For a standard cross-attention layer, the memory I/O cost scales linearly with sequence length as $O(d_{kv} N)$ where $N$ is the sequence length and $d_{kv}$ is the dimension of key-value vectors. For a GMU layer, the memory I/O cost is constant with respect to sequence length at $O(d_h)$ where $d_h$ is the hidden dimension. Since SambaY replaces approximately 50% of cross-attention layers with GMUs, the overall memory I/O for the cross-decoder is approximately halved during generation.

This reduction becomes particularly significant for the workload pattern targeted by this paper: long Chain-of-Thought generation with 32K+ output tokens. The paper reports that Phi4-mini-Flash-Reasoning delivers up to 10× higher decoding throughput on 2K-length prompts with 32K generation length under the vLLM inference framework. Figure 4 shows throughput measurements across various architectures, with SambaY achieving the best throughput in both long-context (32K prompt, 500 generation) and long-generation (2K prompt, 32K generation) settings across various numbers of concurrent clients.

It is important to distinguish between prefill complexity and generation complexity. The SambaY architecture maintains linear pre-filling time complexity, matching YOCO's efficiency advantage during input processing. The GMU mechanism specifically targets the generation stage, where traditional attention-based models incur repeated memory reads for each generated token. This distinction matters because some architectures optimize prefill at the expense of generation, while SambaY improves both stages.

#### Normalization Placement and Training Stability

The placement of normalization layers relative to the gating operation significantly affects model performance, particularly for linear attention mechanisms. Section B of the paper explains that in linear attention architectures (including Gated DeltaNet), a normalization operator is typically applied immediately after token mixing to stabilize training. However, placing normalization before the output gating breaks the associativity between the gating matrix and the token-mixing operator, preventing GMU from directly re-weighting the token mixing based on current layer input.

To resolve this issue, the paper proposes postponing normalization until after output gating for Gated DeltaNet variants (denoted as GDN-A), following the design of Mamba-2. Concretely, for layer $l'$ the computation becomes:

$$M^{(l')} = A^{(l')} V^{(l')}$$
$$Y^{(l')} = \text{Norm}(M^{(l')} \odot G^{(l')}) W_2^{(l')}$$

The cross-decoder then employs normalized GMU (nGMU) at layer $l > l'$ to maintain training stability while allowing the gate to modulate $A^{(l')}$ directly. Section H presents ablation studies confirming this design choice: shifting normalization before the gate and replacing nGMU with simpler GMU leaves short-context benchmarks largely unaffected but leads to severe performance degradation on the Phonebook 32K long-context retrieval task. For example, PB-32K accuracy drops by 56.3 points for S-GDNY (from 83.6% to 27.3%) and by 35.1 points for GDNY (from 89.8% to 54.7%), despite minimal changes in Wiki perplexity and commonsense reasoning scores.

#### Differential Attention Augmentation

The paper augments SambaY with Differential Attention (DA) to create SambaY+DA, which uses DA for all attention layers in the architecture. Differential Attention, introduced in prior work [YDX+24], enhances the model's ability to distinguish between relevant and irrelevant tokens in the attention computation. The Phi4-mini-Flash architecture builds on SambaY+DA with specific configuration choices: 3.8B parameters, sliding window attention size of 512, and Grouped Query Attention with group size 2.

Table 1 shows that SambaY+DA achieves superior performance on the RULER long-context benchmark compared to SambaY without DA, particularly on multi-key retrieval tasks (MK-1: 64.6% vs. 54.6%) and single-needle retrieval (S-1: 99.8% vs. 83.2%). The average retrieval accuracy improves from 42.9% for SambaY to 47.6% for SambaY+DA. However, this comes at a computational cost: Table 2 shows training speed decreases from 1.11 MTPS for SambaY to 0.91 MTPS for SambaY+DA, reflecting the additional computation required by Differential Attention.

The paper notes that their Differential Attention implementation relies on a naive four-pass of the FlashAttention operator for vLLM compatibility, rather than the optimized custom kernel proposed in the original DA paper. This leaves significant room for further speed optimization of Phi4-mini-Flash-Reasoning to catch up to the efficiency of base SambaY.

#### µP++ Hyperparameter Scaling Methodology

To enable fair comparison of scaling capabilities across different architectures, the paper designs a principled µP++ hyperparameter transfer scheme that accounts for both depth and width scaling. The methodology integrates three components: standard µP for width scaling, Depth-µP for depth scaling, and zero weight decay for vector-like or scalar-like parameters for training stability. Since the AdamW optimizer is used, batch-size-based scaling applies with learning rate $\eta \propto \sqrt{B}$. The learning rate is further scaled as $\eta \propto 1/\sqrt{d}$ following Depth-µP.

Formally, the learning rate and training token schedule follow:

$$\eta = \eta_0 \sqrt{\frac{B d_0}{B_0 d}}, \quad B = B_0, \quad T = T_0 \frac{N(d)}{N(d_0)}$$

where the base learning rate is set as $\eta_0 = 4 \times 10^{-4}$, the base batch size $B_0 = 2^{21} = 2\text{M}$ tokens, the base model depth $d_0 = 16$ (corresponding to approximately 1B parameters), and the base training token count $T_0 = 100\text{B}$ (5× the Chinchilla-optimal ratio to study over-training regimes).

A critical innovation in the methodology is the iso-parametric equation approach for comparing hybrid architectures with Transformers. Since different token mixers have different inner dimension expansion ratios, simply matching total parameter counts by adjusting depth would change memory cache sizes unfairly. The paper solves for aspect ratios $\alpha$ that align total non-embedding parameters to the Transformer baseline. For SambaY, the equation is:

$$N(d) = 144\alpha d^3 + 14.5\alpha^2 d^3 = 237568 d^3$$

Solving for $\alpha$ yields $\alpha_1 \approx 124$ for SambaY and $\alpha_2 \approx 126$ for Samba+YOCO. This ensures that comparisons between architectures reflect genuine architectural differences rather than parameter count imbalances.

Figure 2 presents the scaling experiment results using this methodology. For FLOPs scaling (Figure 2a), SambaY exhibits the lowest irreducible loss ($C = 0.58$) compared to Transformer++ ($C = 0.64$) and other baselines, suggesting superior validation loss when scaled with substantially increased computational resources. Under µP++, all architectures share the same compute efficiency exponent ($b = 0.07$), indicating that hybrid architectures did not yield improvements in learning efficiency with respect to compute, but did achieve better asymptotic performance. For data scaling (Figure 2b), Transformer++ trained with µP++ exhibits a large validation loss gap compared to SambaY within the measured range, but its fitted irreducible loss ($C = 1.82$) is nearly identical to hybrid models, suggesting that with infinite data, Transformer++ could eventually catch up—albeit with slower convergence.

#### Architectural Variants and Ablation Design Choices

Section 4 systematically studies design choices through several architectural modifications of SambaY, illustrated in Figure 5. SambaY-2 substitutes Mamba layers with Mamba-2 in the self-decoder, while S-GDNY uses Gated DeltaNet with normalization after output gate (GDN-A). SambaY-A applies GMUs to gate intermediate representations from the last full attention layer in the self-decoder rather than from Mamba. SambaY-AA entirely removes cross-attention in the cross-decoder and instead uses GMU to gate representations from the middle full attention layer. SambaY-MLP uses GMUs to gate intermediate representation from the linear projection branch of the SwiGLU right after the full attention layer.

Table 5 presents ablation results that reveal important design principles. For long-context retrieval (Phonebook 32K), there is a clear hierarchy: SambaY (78.13%) > SambaY-MLP (64.84%) > SambaY-A (58.59%) > SambaY-AA (46.88%). This pattern indicates that GMUs remain effective with alternative memory sources, but their performance on retrieval tasks depends significantly on the memory source's inherent characteristics. Gating attention or MLP representations performs worse than the original SambaY on Phonebook because they lack the recency bias that SSMs naturally provide, which is beneficial for encoding contiguous local information. SambaY-AA, which completely removes cross-attention, shows significant degradation, highlighting the importance of having a sufficient number of cross-attention layers for successful retrievals from a large pool of multiple key-value pairs.

For the self-decoder choice, replacing Mamba with Mamba-2 leads to a significant drop in Phonebook accuracy (from 78.13% to 40.63%), attributed to Mamba-2's coarse, scalar-valued forget gates that reduce the self-decoder's capacity to encode fine-grained positional information. GDN-based models achieve impressive PB-32K accuracy (S-GDNY: 83.59%, GDNY: 89.84%) due to enhanced retrieval capabilities with delta update rules, but interleaving GDN with short-range SWA notably accelerates training without significantly degrading performance.

#### Large-Scale Pre-training Configuration

The Phi4-mini-Flash prototype model applies the SambaY+DA architecture at 3.8B parameters with specific configuration choices that differ from the scaling study models. It uses a sliding window attention size of 512 (compared to 128-256 in scaling experiments), Grouped Query Attention with group size 2, aspect ratio $\alpha = 80$, and attention head dimension of 64. The model is trained with standard parameterization rather than µP++ due to resource constraints at the time of the scaling study. Training occurs on 5T tokens from the Phi4-mini data corpus using 1K A100-80GB GPUs for 14 days, with a 200K token vocabulary size consistent with Phi4-mini.

During large-scale pre-training, the authors encountered severe loss divergence, which they mitigated through two specific tricks illustrated in Figure 6. First, they up-cast the weight and input to FP32 during chunk-wise matrix multiplication for the fused linear cross-entropy loss kernel, which stabilizes gradient norms compared to normal BF16 matrix multiplications that show up-shooting trends. Second, they add attention dropout of 0.05 at specific checkpoints (310K steps for self-decoder, 520K steps for cross-decoder), which are the last checkpoint steps before loss divergences occur. The optimization setup is explicitly noted as non-optimal, as the primary goal was to evaluate architectural viability at larger scales rather than achieve peak performance.

Table 3 demonstrates that Phi4-mini-Flash outperforms the Phi4-mini baseline across 7 out of 8 benchmarks, with notable improvements on knowledge-intensive tasks (MMLU: 71.9 vs. 67.3) and coding tasks (MBPP pass@1: 69.8 vs. 65.3). This consistent performance advantage is particularly significant given that Phi4-mini-Flash achieves these gains while maintaining substantially higher computational efficiency during inference.

#### Reasoning Model Development and Evaluation

The Phi4-mini-Flash-Reasoning model is continually trained from Phi4-mini-Flash using the same multi-stage distillation data following Phi4-mini-Reasoning, but with only Supervised Fine-Tuning (SFT) and Direct Preference Optimization (DPO) stages—excluding the Reinforcement Learning stage used by Phi4-mini-Reasoning due to resource constraints. Table 4 shows that despite lacking RL training, Phi4-mini-Flash-Reasoning achieves significantly better performance than Phi4-mini-Reasoning on challenging reasoning benchmarks: AIME24 (52.29 vs. 48.13), AIME25 (33.59 vs. 31.77), Math500 (92.45 vs. 91.20), and GPQA Diamond (45.08 vs. 44.51).

The evaluation protocol uses a sampling temperature of 0.6, top-p value of 0.95, and maximum sequence length of 32,768 tokens. For AIME24/25 and Math500, the instruction "Please reason step by step, and put your final answer within \boxed{}." is prepended, while GPQA Diamond uses "Please reason step by step, and put your final choice of one letter from A/B/C/D within \boxed{}." Pass@1 accuracy is averaged over 64 samples for AIME24/25 and 8 samples for Math500 and GPQA Diamond to ensure evaluation robustness.

Example 1 and Example 2 in Section G provide case studies demonstrating that the model exhibits advanced reasoning techniques such as double checking, reflection, and backtracking despite being trained only on math distillation data. In Example 2 (counting R's in "strawberrry"), the model even guesses user intention ("But wait, sometimes people might confuse the number of R's in a double letter") to provide the correct answer, suggesting emergent reasoning capabilities beyond the training distribution.

## 4. Key Insights and Innovations

This section distills the paper's most significant contributions, distinguishing between incremental engineering improvements and fundamental shifts in architectural design. We focus on *why* these findings matter for the broader field of efficient language modeling, rather than merely restating the mechanical details covered in the Technical Approach.

### 4.1 Redefining Cross-Layer Memory: From KV Caches to Recurrent States

The most fundamental innovation in this work is the conceptual shift in how "memory" is shared across layers during inference. Prior decoder-decoder architectures like YOCO [SDZ+24] share **Key-Value (KV) caches** generated by a full attention layer. While this reduces computation during prefilling, it retains a memory I/O complexity of $O(d_{kv} N)$ during generation, where $N$ is the sequence length. This means that as the generated context grows, the cost of reading the shared memory grows linearly.

SambaY introduces a paradigm shift by sharing **SSM recurrent states** instead of KV caches for half of the cross-decoder layers via the Gated Memory Unit (GMU).

*   **Complexity Class Change:** By sharing the SSM state $m \in \mathbb{R}^{d_h}$ rather than the KV cache, the memory I/O complexity for these layers becomes constant with respect to sequence length, $O(d_h)$. This is not merely an optimization; it changes the asymptotic behavior of the cross-decoder during long generation.
*   **Positional Encoding Elimination:** A subtle but critical consequence of this shift is the removal of explicit positional encodings (e.g., RoPE). Because the SSM state inherently encodes temporal order through its recurrent update, the model achieves **zero-shot 2× retrievable context extrapolation** without positional embeddings (Section F, Table 8). This suggests that for hybrid architectures, recurrence provides sufficient positional signal, simplifying the architecture and reducing potential failure modes at extreme context lengths.
*   **Significance:** This insight challenges the assumption that attention-based memory (KV caches) is necessary for cross-layer communication in decoder-decoder structures. It demonstrates that recurrent states can serve as a more efficient "broadcast medium" for later layers, decoupling inference cost from sequence length in a way that KV-sharing cannot.

### 4.2 The "Small Window" Paradox in Long-Context Retrieval

A counter-intuitive finding in this paper challenges the prevailing heuristic that "larger attention windows equal better long-context performance." Conventional wisdom suggests that to retrieve information from a 32K context, the attention mechanism needs a large sliding window to "see" enough tokens. However, the empirical results in **Figure 3** reveal the opposite for hybrid architectures.

*   **Empirical Evidence:** On the Phonebook benchmark with 32K evaluation length, `SambaY` achieves peak accuracy with a Sliding Window Attention (SWA) size of **256**, while larger windows (e.g., 1024 or 2048) do not consistently improve results and sometimes degrade performance.
*   **Mechanism:** The authors hypothesize that learned full attention involves both local patterns and non-local patterns (like global retrieval or attention sinks). Using small sliding windows enables the model to focus on local patterns more easily and **mitigate attention sinks** [GPD+25], where attention scores disproportionately concentrate on initial tokens. The SSM layers handle the long-range dependency tracking, freeing the attention layers to refine local interactions without needing a large receptive field.
*   **Significance:** This insight has profound implications for memory efficiency. If a model can achieve strong 32K retrieval performance with a window size of 256 rather than 2048, the KV cache memory footprint is reduced by **8×**. This suggests that future long-context models should not blindly scale window sizes but instead optimize the interplay between SSM recurrence (for long-range) and attention (for local refinement).

### 4.3 Architectural Inductive Bias for Reasoning Without Reinforcement Learning

The performance of `Phi4-mini-Flash-Reasoning` on complex reasoning benchmarks offers a critical insight into the relationship between architecture and reasoning capability. Typically, achieving state-of-the-art performance on tasks like AIME or Math500 requires extensive Reinforcement Learning (RL) post-training to align the model's chain-of-thought process.

*   **Performance Discrepancy:** As shown in **Table 4**, `Phi4-mini-Flash-Reasoning` (3.8B) outperforms the strong `Phi4-mini-Reasoning` baseline on AIME24 (52.29 vs. 48.13) and Math500 (92.45 vs. 91.20), despite **excluding any stage of Reinforcement Learning** that the baseline utilized.
*   **Inductive Bias Hypothesis:** This suggests that the hybrid architecture itself possesses a stronger **inductive bias for reasoning traces** than a pure Transformer. The state-tracking capability of the SSM layers may naturally facilitate the maintenance of intermediate variables and logical states required for multi-step reasoning, reducing the reliance on expensive RL tuning to "learn" this behavior.
*   **Significance:** If hybrid architectures can reduce or eliminate the need for RL in reasoning tasks, this represents a massive reduction in training cost and complexity. It shifts the burden of reasoning capability from *post-training alignment* to *pre-training architecture design*, suggesting that future reasoning models should prioritize architectural expressiveness over brute-force alignment tuning.

### 4.4 Methodological Rigor: Iso-Parametric Scaling for Hybrid Comparison

Beyond the architecture itself, the paper introduces a methodological innovation for fairly comparing hybrid models against Transformers. A common pitfall in prior hybrid research is adjusting model depth to match parameter counts, which inadvertently changes memory cache sizes and confounds efficiency comparisons.

*   **Iso-Parametric Equations:** The authors derive specific aspect ratios ($\alpha$) by solving iso-parametric equations that align total non-embedding parameters to a Transformer baseline while keeping memory cache dimensions consistent. For example, for `SambaY`, they solve $144\alpha d^3 + 14.5\alpha^2 d^3 = 237568 d^3$ to find $\alpha \approx 124$ (Section 3.1).
*   **Scaling Law Insight:** Using this rigorous setup, **Figure 2a** shows that `SambaY` exhibits a lower irreducible loss ($C = 0.58$) compared to Transformer++ ($C = 0.64$) under FLOPs scaling. Crucially, the compute efficiency exponent ($b = 0.07$) remains the same across architectures under µP++, indicating that the benefit is not faster convergence but a **better asymptotic performance floor**.
*   **Significance:** This establishes a new standard for architectural evaluation. By ensuring that performance differences are not artifacts of parameter mismatches or cache size variations, the paper provides robust evidence that hybrid architectures offer superior scalability under large-scale compute regimes. This methodology allows future researchers to distinguish genuine architectural advances from hyperparameter tuning effects.

## 5. Experimental Analysis

This section critically examines the experimental methodology and results that support the paper's claims about SambaY's efficiency and performance advantages. We analyze whether the evidence convincingly demonstrates the architecture's benefits, identify conditions under which claims hold or weaken, and assess the robustness of the findings through ablation studies.

### 5.1 Evaluation Methodology and Experimental Setup

The paper employs a multi-tiered evaluation strategy spanning three model scales: (1) 1B-parameter models for architectural comparison and ablation studies, (2) 3.4B-parameter models for scaling law analysis, and (3) a 3.8B-parameter prototype (Phi4-mini-Flash) for downstream task evaluation. This tiered approach allows the authors to isolate architectural effects from scale effects.

**Datasets and Training Configuration:**

| Experiment Type | Dataset | Sequence Length | Training Tokens | Parameters |
|----------------|---------|-----------------|-----------------|------------|
| Scaling laws | SlimPajama [SAKM+23] | 4K | 100B–600B | 1B–3.4B |
| Long-context | ProLong-64k [GWYC24] | 32K | 40B | 1B |
| Downstream | Phi4-mini corpus [MAA+25] | Variable | 5T | 3.8B |

For scaling experiments, the authors adopt the **µP++ hyperparameter transfer scheme** described in Section 3.1, which integrates width scaling (µP), depth scaling (Depth-µP), and zero weight decay for vector-like parameters. The base configuration uses depth $d_0 = 16$ (approximately 1B parameters), base learning rate $\eta_0 = 4 \times 10^{-4}$, and base batch size $B_0 = 2^{21} = 2\text{M}$ tokens. Training tokens scale linearly with parameters following Chinchilla scaling laws [HBM+22].

**Baseline Architectures:**

The paper compares SambaY against five baseline architectures (Section 3.1):
- **Transformer++**: Standard Transformer with SwiGLU MLP and RoPE (base frequency 10,000)
- **Samba+YOCO**: Naive combination of Samba self-decoder with original YOCO cross-decoder (no GMUs)
- **SWA+YOCO**: Original YOCO with Sliding Window Attention layers as self-decoder
- **TransformerLS**: Interleaved SWA with full attention using 3:1 layer ratio
- **SambaY+DA**: SambaY augmented with Differential Attention [YDX+24] for all attention layers

All hybrid SSM architectures omit explicit positional encodings (NoPE), while Transformer-based models use RoPE. The sliding window size is standardized to 128 for all SWA-containing architectures during scaling experiments, with ablations on window size effects in Section 3.2.

**Evaluation Metrics:**

The paper uses distinct metrics for different evaluation domains:
- **Scaling behavior**: Validation loss fitted to power law $L(D_{\text{FLOPs}}) = A \cdot D_{\text{FLOPs}}^{-b} + C$, where $C$ represents irreducible loss
- **Long-context retrieval**: Accuracy on Phonebook benchmark [JBKM24] and RULER [HSK+24] tasks
- **Downstream tasks**: Perplexity (WikiText), accuracy on commonsense reasoning (ARC, HellaSwag, PIQA, WinoGrande, LAMBADA)
- **Efficiency**: Throughput in tokens/second under vLLM [KLZ+23] inference framework
- **Reasoning**: Pass@1 accuracy on Math500, AIME24/25, GPQA Diamond

### 5.2 Scaling Law Experiments: Evidence for Superior Asymptotic Performance

The scaling experiments in **Figure 2** provide the most rigorous evidence for SambaY's architectural advantages. The authors fit validation loss as a function of compute (FLOPs) and training tokens to power laws, extracting two key parameters: the scaling exponent $b$ (learning efficiency) and irreducible loss $C$ (asymptotic performance floor).

**FLOPs Scaling Results (Figure 2a):**

| Architecture | Irreducible Loss $C$ | Scaling Exponent $b$ |
|--------------|---------------------|---------------------|
| SambaY | **0.58** | 0.07 |
| Transformer++ | 0.64 | 0.07 |
| Samba+YOCO | ~0.60 | 0.07 |
| SWA+YOCO | ~0.61 | 0.07 |

The key finding is that **SambaY achieves the lowest irreducible loss ($C = 0.58$)** compared to Transformer++ ($C = 0.64$), indicating superior asymptotic performance under large-scale compute regimes. However, all architectures share the same compute efficiency exponent ($b = 0.07$) under µP++, meaning hybrid architectures do not converge faster—they simply reach a better final loss.

**Data Scaling Results (Figure 2b):**

| Architecture | Irreducible Loss $C$ (Data) |
|--------------|----------------------------|
| Transformer++ | 1.82 |
| SambaY | ~1.82 |
| Samba+YOCO | ~1.82 |

Notably, under data scaling with fixed model size (1B parameters), Transformer++ exhibits a large validation loss gap compared to SambaY within the measured range (100B–600B tokens), but its fitted irreducible loss ($C = 1.82$) is nearly identical to hybrid models. The paper interprets this as evidence that "with an infinite amount of data, Transformer++ can eventually catch up—albeit with slower convergence."

**Critical Assessment:**

This finding reveals an important conditionality in the paper's claims. The advantage of SambaY is **not universal** but depends on the scaling regime:
- Under **compute scaling** (increasing both model size and data proportionally), SambaY shows clear advantages in irreducible loss.
- Under **data scaling** (fixed model size, increasing tokens), the advantage diminishes at the asymptotic limit.

This distinction matters for practitioners: if you have a fixed model budget but can train on more data, Transformers may eventually match hybrid performance. However, if you can scale both compute and data (the more common scenario for large-scale training), SambaY offers better returns.

The goodness of fit for all scaling curves is $R^2 \geq 0.999$, indicating near-perfect fits and lending credibility to the extrapolated irreducible loss values. However, the paper acknowledges that these are extrapolations—the largest model trained is 3.4B parameters, while the claims concern behavior at much larger scales.

### 5.3 Long-Context Retrieval: Small Windows, Strong Performance

The long-context retrieval experiments in **Section 3.2** test whether SambaY's efficiency improvements come at the cost of retrieval capability—a common trade-off in efficient architectures.

**Phonebook Benchmark (Figure 3):**

The Phonebook benchmark tests multi-key-value retrieval with 32K context length (1,850 name-number pairs). **Figure 3** plots accuracy against Sliding Window Attention (SWA) size, revealing a counter-intuitive result:

| SambaY | 256 | ~92% |
| SambaY+DA | 512 | ~96% |
| Samba+YOCO | 1024 | ~83% |
| TransformerLS | 256 | ~60% |

The paper notes that "larger SWA sizes do not consistently provide better results." SambaY achieves strong performance with SWA size 256, while Samba+YOCO requires SWA size 1024 for comparable results. This represents a **4× reduction in KV cache memory** for the attention layers, compounding with the GMU's memory savings.

**Training Stability Issues:**

**Critical Assessment:**

The downstream results support the paper's claims, but with conditions:
- The model uses **standard parameterization** rather than µP++ due to resource constraints, making direct comparison to scaling experiments difficult.
- The vocabulary size (200K tokens) differs from scaling experiments (32K tokens), introducing a confounding variable.
- Training stability required non-standard tricks (FP32 up-casting, attention dropout), suggesting the architecture may be more fragile than standard Transformers at scale.

### 5.6 Ablation Studies: Isolating Design Choices

Section 4 and Appendix H provide systematic ablation studies that isolate the contribution of each architectural component.

**Memory Source Ablation (Table 5):**

| Model | Phonebook 32K | Avg. Accuracy | Speed (MTPS) |
|-------|---------------|---------------|--------------|
| SambaY | **78.13%** | 52.16% | 1.10 |
| SambaY-MLP | 64.84% | **52.65%** | 1.15 |
| SambaY-A | 58.59% | 52.26% | 1.11 |
| SambaY-AA | 46.88% | 52.06% | 1.25 |
| MambaY | 12.50% | 51.87% | 0.94 |

The hierarchy for long-context retrieval is clear: **SambaY > SambaY-MLP > SambaY-A > SambaY-AA**. This demonstrates that:
1. Gating SSM states (SambaY) outperforms gating attention (SambaY-A) or MLP (SambaY-MLP) representations for retrieval tasks.
2. Completely removing cross-attention (SambaY-AA) causes significant degradation, indicating some cross-attention layers are necessary for multi-key retrieval.
3. Short-context performance is less sensitive to memory source choice (all variants within 0.6% of each other).

**SSM Variant Ablation (Table 5):**

| Model | Phonebook 32K | Speed (MTPS) |
|-------|---------------|--------------|
| SambaY (Mamba) | 78.13% | 1.10 |

| GDNY (GDN-A only) | 89.84% | 1.22 |

Replacing Mamba with Mamba-2 causes a **37.5 point drop** in Phonebook accuracy (78.13% → 40.63%), attributed to Mamba-2's "coarse, scalar-valued forget gates" that reduce capacity for fine-grained positional information. However, Mamba-2 is 30% faster (1.43 vs. 1.10 MTPS), revealing a speed-accuracy trade-off.

| Model | Phonebook 32K | Wiki PPL |
|-------|---------------|----------|
| S-GDNY (nGMU) | **83.59%** | 16.78 |

Shifting normalization before the gate and replacing nGMU with simpler GMU causes a **56.3 point drop** for S-GDNY (83.59% → 27.34%) while leaving short-context perplexity nearly unchanged (16.78 → 17.76). This is one of the most striking ablation results in the paper, demonstrating that normalization placement is critical for long-context retrieval but irrelevant for short-context tasks.

**Training Data Ablation (Figure 8, Section F):**

Figure 8 compares Phonebook accuracy when training on SlimPajama vs. ProLong-64K (without variable-length training):
- On SlimPajama, overall accuracy is modest, with SambaY+DA peaking at SWA size 128 before declining.
- On ProLong-64K, all architectures show notable performance boosts, with SSM-based models enjoying larger improvements than Transformer++.

This indicates that **training data quality matters more for SSM-based models**, which can "learn to switch contexts between different data samples within packed sequences more easily than pure attention models."

**Critical Assessment:**

The ablation studies are thorough and reveal important design principles:
1. **Memory source hierarchy**: SSM states > MLP > Attention for long-context retrieval.
2. **Normalization is critical**: nGMU with post-gate normalization is essential for linear attention variants.
3. **Cross-attention cannot be fully removed**: SambaY-AA's degradation shows some attention layers are necessary.
4. **Data matters**: ProLong-64K significantly outperforms SlimPajama for long-context training.

However, some ablations are limited:
- The paper does not ablate the **ratio of GMU to cross-attention layers** (fixed at ~50% throughout).
- The **optimal placement** of GMUs within the cross-decoder is not explored (always interleaved).
- Resource constraints prevented exhaustive hyperparameter search for each architecture variant.

### 5.7 Overall Assessment of Experimental Evidence

The experimental evidence supports the paper's claims with varying degrees of confidence:

**Strongly Supported:**
- SambaY achieves lower irreducible loss than Transformer++ under compute scaling (Figure 2a, $C = 0.58$ vs. $0.64$).
- GMUs reduce memory I/O during decoding, enabling 10× throughput improvement on long-generation workloads (Figure 4).
- Normalization placement (nGMU with post-gate Norm) is critical for long-context retrieval (Table 9, 56.3 point difference).
- SSM-based memory sharing outperforms attention/MLP memory sharing for retrieval tasks (Table 5).

**Moderately Supported:**
- SambaY enables 2× zero-shot context extrapolation without positional encoding (Table 8), but fails at 4× extrapolation.
- Phi4-mini-Flash-Reasoning outperforms Phi4-mini-Reasoning without RL (Table 4), but the margin is modest (1–4 points).
- Small SWA sizes (256–512) suffice for strong 32K retrieval (Figure 3), but the explanation (attention sink mitigation) is hypothesized rather than proven.

**Weakly Supported:**
- The claim that hybrid architectures have "superior scaling potential" relies on extrapolation from 3.4B to much larger scales.
- The assertion that SSMs provide "inductive bias for reasoning" is suggestive but not definitively proven (RL exclusion may confound the comparison).
- Training stability at scale required non-standard tricks (FP32 up-casting, attention dropout), suggesting the architecture may be more fragile than reported.

**Notable Gaps:**
- No ablation of GMU-to-cross-attention ratio (always ~50%).
- No analysis of failure modes on specific reasoning tasks.
- Limited exploration of why NoPE enables extrapolation (explicitly left as future work).
- Differential Attention implementation is sub-optimal, leaving efficiency claims potentially conservative.

Overall, the experiments are methodologically rigorous (iso-parametric comparisons, µP++ scaling, multiple baselines) and provide convincing evidence for SambaY's efficiency advantages. The performance advantages are real but conditional on the scaling regime and workload pattern. The paper is transparent about limitations and implementation caveats, which strengthens confidence in the reported results.

## 6. Limitations and Trade-offs

This section critically examines the constraints, assumptions, and unresolved challenges in the SambaY architecture. While the paper demonstrates compelling efficiency gains, understanding where the approach breaks down is essential for practitioners considering adoption. We organize limitations into four categories: architectural assumptions, unaddressed scenarios, resource constraints, and open research questions.

### 6.1 Architectural Assumptions and Dependencies

The SambaY design relies on several assumptions that may not hold universally across deployment contexts.

**Assumption 1: Long-generation workloads dominate inference cost.** The 10× throughput improvement applies specifically to scenarios with extensive token generation (e.g., 32K output tokens). For short-response workloads (e.g., classification, brief QA), the efficiency gains diminish significantly. Figure 4(a) shows that for long-context prefills with short generation (32K prompt, 500 tokens), the throughput advantage is substantially smaller than in Figure 4(b) (2K prompt, 32K generation). This limits the architecture's appeal for applications where prompt processing dominates over generation.

**Assumption 2: SSM states provide sufficient positional signal.** The elimination of explicit positional encodings (NoPE) works for 2× context extrapolation but fails at 4×. Table 8 in Section F shows SambaY achieves 96.09% accuracy at 64K context but drops to 0.00% at 128K. The paper explicitly acknowledges this gap: "We leave the explanations of why NoPE can enable limited extrapolations on retrieval tasks as an interesting future work." This suggests the positional encoding mechanism is not fully understood and may fail unpredictably at extreme context lengths.

**Assumption 3: Cross-attention layers remain necessary.** The ablation study in Table 5 reveals that completely removing cross-attention (SambaY-AA) causes significant degradation on long-context retrieval (46.88% vs. 78.13% for SambaY on Phonebook 32K). This means the architecture cannot achieve pure constant-complexity decoding—some layers must still pay the $O(d_{kv} N)$ memory I/O cost. The paper states in Section J: "Our architecture still includes a full-attention layer, which leads to linear per-token computation complexity during decoding."

**Assumption 4: Memory source hierarchy is fixed.** Table 5 demonstrates a clear performance hierarchy for GMU memory sources: SSM states (SambaY: 78.13%) > MLP representations (SambaY-MLP: 64.84%) > Attention representations (SambaY-A: 58.59%). This limits architectural flexibility—if an application requires gating attention states for other reasons, retrieval performance will suffer. The paper attributes this to SSMs providing "recency bias that is beneficial for encoding contiguous local information," but this constraint may not be desirable for all tasks.

### 6.2 Unaddressed Scenarios and Edge Cases

Several important deployment scenarios receive limited or no treatment in the paper.

**Streaming and interleaved inference.** The evaluation assumes a clear separation between prefill (processing input) and generation (producing output). Real-world applications often involve streaming inputs where tokens arrive incrementally, requiring interleaved prefill and generation. The decoder-decoder architecture's efficiency depends on computing the self-decoder's KV cache once during prefill—this advantage may erode if the model must repeatedly re-process partial inputs.

**Multi-modal and multi-turn conversations.** All benchmarks evaluate single-turn completion tasks. Multi-turn dialogue requires maintaining conversation history across multiple generation episodes, potentially invalidating the "cache once" assumption if context windows must be recomputed between turns. The paper does not address how SambaY handles conversation state management or whether the SSM states can be preserved across turn boundaries.

**Extreme context lengths beyond 128K.** Table 8 shows all models fail at 128K context length. The paper does not explore whether architectural modifications (e.g., hierarchical memory, sparse attention patterns) could extend this limit. For applications requiring hundred-thousand-token contexts (e.g., full-codebase analysis, long-document summarization), SambaY offers no clear path forward beyond existing baselines.

**Privacy and safety considerations.** The NeurIPS checklist (Section 10, Broader Impacts) marks societal impact as "NA," and Section 11 (Safeguards) states "Our paper poses no such risks." However, more efficient reasoning models could lower the barrier for generating harmful content at scale. The paper does not discuss whether the architectural changes affect safety fine-tuning or whether the improved efficiency could enable new misuse patterns.

**Hardware-specific optimizations.** The efficiency claims depend on the vLLM inference framework with specific implementation choices. Section 3.4 notes: "Our Differential Attention implementation relies on a naive four-pass of the FlashAttention operator for vLLM compatibility, rather than the optimized custom kernel proposed in the original paper." This means the reported throughput may not generalize to other inference engines or hardware configurations without additional engineering effort.

### 6.3 Computational, Data, and Scalability Constraints

Resource limitations affected both the research process and the resulting architecture's practical deployment.

**Training stability requires non-standard tricks.** Section 3.3 and Figure 6 document severe loss divergence during Phi4-mini-Flash pre-training, mitigated through:
1. FP32 up-casting for fused linear cross-entropy loss kernels
2. Attention dropout of 0.05 added at specific checkpoints (310K steps for self-decoder, 520K steps for cross-decoder)

The authors explicitly state: "The optimization setup here is by no means optimal." This suggests the architecture may be more fragile than standard Transformers, requiring careful monitoring and intervention during training. Practitioners adopting SambaY should anticipate potential stability challenges not present in conventional architectures.

**No exhaustive hyperparameter search.** Section J (Limitations) acknowledges: "Due to resource constraints, we do not perform an exhaustive hyperparameter search for each architecture. Instead, we adopt a generic optimization setup based on Transformer++." This means the reported performance may underestimate what's achievable with architecture-specific tuning. Conversely, it also means the architecture may require more tuning effort to reach peak performance in new domains.

**Parameterization mismatch between scaling study and large-scale model.** The scaling experiments use µP++ hyperparameter transfer, but Phi4-mini-Flash uses standard parameterization "due to resource constraints at the time of scaling study" (Section 3.3). This creates a methodological gap: the scaling law conclusions (Figure 2) do not directly apply to the largest model evaluated (Table 3, Table 4). Practitioners cannot confidently extrapolate the irreducible loss benefits to the 3.8B parameter regime.

**GMU-to-cross-attention ratio is fixed.** Throughout all experiments, approximately 50% of cross-attention layers are replaced with GMUs. Section 4 does not ablate this ratio—there is no evidence that 50% is optimal. A more aggressive replacement could improve efficiency further, while a more conservative ratio might improve retrieval capability. This represents an unexplored design dimension.

**Vocabulary size confounds comparisons.** The scaling experiments use a 32K token vocabulary (Llama-2 tokenizer), while Phi4-mini-Flash uses 200K tokens "consistent with Phi4-mini" (Table 7 footnote). Vocabulary size affects both model capacity and training dynamics, making it difficult to isolate architectural effects from vocabulary effects in the large-scale results.

### 6.4 Open Research Questions

Several important questions remain unresolved, representing opportunities for future work.

**Why does NoPE enable limited extrapolation?** The paper observes 2× zero-shot context extrapolation without positional encodings but offers no mechanistic explanation. Understanding this phenomenon could inform architectural designs for even longer contexts. The explicit acknowledgment in Section F ("We leave the explanations...as an interesting future work") indicates this is a genuine gap in the authors' understanding.

**Can RL further improve hybrid reasoning models?** Section J states: "Reinforcement Learning (RL) remains under-explored in the context of hybrid architectures." Since Phi4-mini-Flash-Reasoning outperforms Phi4-mini-Reasoning without RL, a natural question is whether adding RL would yield additional gains—or whether the architectural inductive bias already captures most reasoning capability. This has significant implications for training cost: if hybrids reduce RL dependence, training becomes substantially cheaper.

**What is the optimal GMU placement strategy?** GMUs are interleaved with cross-attention layers in the cross-decoder, but the paper does not explore whether early vs. late placement matters. Some layers may benefit more from full cross-attention (e.g., layers responsible for final token prediction), while others could use GMUs exclusively. This represents an unexplored optimization dimension.

**How does SambaY interact with other efficiency techniques?** Section I (Related Work) mentions orthogonal techniques like Layer-Condensed KV Cache [WT24] and InfiniGen [LLSS24], but does not evaluate combinations. It is unclear whether GMU-based memory sharing composes additively with these methods or whether there are diminishing returns.

**Does the speed-accuracy trade-off favor Mamba-2 in some regimes?** Table 5 shows Mamba-2 variants are 30% faster than Mamba (1.43 vs. 1.10 MTPS) but suffer 37.5 points lower Phonebook accuracy. For applications prioritizing throughput over retrieval precision, Mamba-2 may be preferable. The paper does not provide guidance on when to accept this trade-off.

**Can dynamic sparse attention eliminate the full-attention bottleneck?** The Conclusion suggests: "Future work could explore dynamic sparse attention mechanisms to further improve efficiency on extremely long sequence generation." This acknowledges that the remaining full-attention layer is a fundamental bottleneck. Whether sparse attention can replace it without sacrificing retrieval capability remains an open question.

### 6.5 Summary Assessment

The limitations reveal a architecture optimized for a specific niche: **long-generation reasoning workloads with moderate context lengths (up to 64K)**. Outside this niche, the advantages diminish or disappear:

| Scenario | SambaY Advantage | Confidence |
|----------|-----------------|------------|
| 32K+ token generation | 10× throughput | High |
| 64K context retrieval | 2× extrapolation works | High |
| 128K+ context | Fails like baselines | High |
| Short-response workloads | Minimal gain | Medium |
| Multi-turn dialogue | Unknown | Low |
| RL-enhanced reasoning | Unexplored | Low |
| Training stability | Requires tricks | High |

For practitioners, the key takeaway is that SambaY represents a **specialized optimization** rather than a universal improvement. The architecture excels at its target workload but introduces complexity (normalization placement, stability tricks, implementation requirements) that may not be justified for general-purpose applications. The transparent reporting of limitations in Section J strengthens confidence in the reported results while clearly delineating the boundary of current knowledge.

## 7. Implications and Future Directions

This section synthesizes how the SambaY architecture and Gated Memory Unit mechanism reshape the landscape of efficient language modeling, identifies concrete research directions this work enables, and provides practical guidance for practitioners considering adoption. We distinguish between near-term actionable insights and longer-term research opportunities.

### 7.1 Shifting the Efficiency Frontier: From Prefill to Generation

The most significant landscape-level contribution of this work is reframing where efficiency optimizations should focus for reasoning-oriented language models. Prior efficient architecture research predominantly targeted **prefill-stage computation**—reducing the cost of processing long input prompts through techniques like sparse attention, linear attention, or KV cache compression. SambaY redirects attention to the **generation stage**, which becomes the dominant cost for models producing lengthy Chains-of-Thought.

**Why this matters:**

- **Workload pattern shift:** Modern reasoning models (e.g., o1-style systems) generate 10K–50K token reasoning traces per query. For a 2K prompt with 32K generation, the generation stage accounts for ~94% of total inference tokens. Optimizing prefill alone captures only a fraction of potential efficiency gains.
- **Memory-bandwidth bottleneck:** As Figure 4 demonstrates, the 10× throughput improvement comes from reducing memory I/O during decoding, not compute reduction. This aligns with the reality that modern GPUs are memory-bandwidth bound during autoregressive generation, not compute-bound.
- **Architectural precedent:** By demonstrating that SSM states can replace KV caches for cross-layer communication, this work opens a new design space for decoder-decoder architectures. Future models need not assume attention-based memory is necessary for layer-to-layer information flow.

**Concrete impact on the field:**

The paper establishes that hybrid architectures can achieve **better asymptotic performance under compute scaling** (Figure 2a: $C = 0.58$ for SambaY vs. $C = 0.64$ for Transformer++) while simultaneously improving inference efficiency. This dual benefit—better scaling *and* better efficiency—challenges the traditional trade-off assumption that efficiency optimizations sacrifice model quality.

### 7.2 Enabling Research Directions

The SambaY architecture creates several concrete avenues for follow-up research, ranging from immediate engineering improvements to fundamental architectural questions.

#### 7.2.1 Near-Term Engineering Optimizations

**Differential Attention kernel optimization.** Section 3.4 explicitly notes that the current DA implementation uses "a naive four-pass of the FlashAttention operator" rather than optimized custom kernels. This leaves "significant room for further speed optimization." A follow-up project could:
- Implement a fused DA kernel matching the original [YDX+24] specification
- Benchmark whether optimized DA closes the throughput gap between SambaY+DA (0.91 MTPS) and base SambaY (1.11 MTPS) from Table 2
- Explore whether DA can be selectively applied only to layers where retrieval matters most

**GMU-to-cross-attention ratio ablation.** Throughout all experiments, approximately 50% of cross-attention layers are replaced with GMUs. This ratio is never ablated. Systematic exploration could reveal:
- Whether more aggressive replacement (75% GMUs) maintains retrieval capability while further improving throughput
- Whether layer-specific placement matters (e.g., GMUs in early cross-decoder layers, attention in later layers)
- Whether the optimal ratio depends on target context length or task type

**Training stability improvements.** Section 3.3 documents severe loss divergence during Phi4-mini-Flash pre-training, mitigated through FP32 up-casting and attention dropout. The authors state this setup is "by no means optimal." Follow-up work could:
- Identify the root cause of instability (e.g., gradient explosion in GMU layers, SSM state drift)
- Develop architecture-specific initialization schemes beyond the generic Transformer++ setup
- Explore whether µP++ parameterization (used in scaling studies) improves large-scale stability compared to standard parameterization

#### 7.2.2 Medium-Term Architectural Extensions

**Reinforcement Learning for hybrid architectures.** Section J explicitly states RL "remains under-explored in the context of hybrid architectures." Given that Phi4-mini-Flash-Reasoning outperforms Phi4-mini-Reasoning *without* RL (Table 4), critical questions include:
- Does RL provide diminishing returns for hybrid architectures compared to pure Transformers?
- Are there RL algorithms better suited to SSM-based state tracking (e.g., reward shaping that leverages recurrent state information)?
- Can hybrid architectures achieve equivalent reasoning performance with 50% less RL training compute?

**Dynamic sparse attention integration.** The Conclusion suggests exploring "dynamic sparse attention mechanisms to further improve efficiency on extremely long sequence generation." This would target the remaining full-attention layer that still incurs $O(d_{kv} N)$ memory I/O. Research directions include:
- Replacing the single full-attention layer with sparse attention patterns (e.g., sinkhorn, block-sparse)
- Evaluating whether sparse attention maintains the retrieval capability demonstrated in Table 1
- Determining whether sparsity patterns can be learned adaptively based on task requirements

**Understanding NoPE extrapolation.** Section F acknowledges the unexplained phenomenon that SambaY achieves "zero-shot 2× retrievable context extrapolation" without positional encodings (Table 8: 92.19% at 32K → 96.09% at 64K), but fails at 128K. Mechanistic understanding could enable:
- Architectural modifications to extend extrapolation beyond 2×
- Hybrid positional encoding schemes that combine SSM recurrence with minimal explicit position signals
- Theoretical analysis of how SSM state dynamics encode temporal order

#### 7.2.3 Long-Term Fundamental Questions

**Inductive bias for reasoning.** The paper suggests hybrid architectures may possess "stronger inductive bias for reasoning traces" than pure Transformers (Section 4.3), potentially reducing RL dependence. This hypothesis warrants deeper investigation:
- Do SSM layers naturally maintain intermediate variable states required for multi-step reasoning?
- Can we design SSM variants specifically optimized for logical state tracking (e.g., explicit variable binding mechanisms)?
- Does the recency bias of SSMs align with how humans structure reasoning traces (recent steps matter more)?

**Iso-parametric methodology for architectural comparison.** The paper's approach to solving iso-parametric equations for fair comparison (Section 3.1) establishes a new standard for architectural evaluation. This methodology could be extended to:
- Compare emerging architectures (e.g., state-space hybrids, mixture-of-experts variants) under consistent parameter budgets
- Develop automated tools for deriving aspect ratios given arbitrary layer type combinations
- Create benchmark suites that control for cache size, memory footprint, and parameter count simultaneously

### 7.3 Practical Applications and Use Cases

The SambaY architecture is not a universal replacement for Transformers but excels in specific deployment scenarios. Table 10 summarizes when to prefer SambaY over alternative architectures.

| Workload Characteristic | Preferred Architecture | Rationale |
|------------------------|----------------------|-----------|
| Generation length > 10K tokens | **SambaY** | 10× throughput gain on long generation (Figure 4b) |
| Context length ≤ 64K | **SambaY** | 2× extrapolation works reliably (Table 8) |
| Context length > 128K | **Transformer++** | SambaY fails at 128K like baselines (Table 8) |

**Specific application domains:**

1. **Mathematical reasoning systems:** The Phi4-mini-Flash-Reasoning results (Table 4) demonstrate strong performance on AIME, Math500, and GPQA Diamond. Applications requiring step-by-step mathematical solutions with 10K+ token explanations are ideal targets.

2. **Code generation and debugging:** Table 3 shows MBPP pass@1 improvement from 65.3% to 69.8%. Code generation often requires lengthy reasoning traces (explaining design decisions, debugging logic), making the long-generation efficiency gains directly applicable.

3. **Scientific question-answering:** GPQA Diamond performance (45.08% vs. 44.51%) suggests applicability to domain-specific QA requiring extended reasoning. The efficiency gains reduce serving costs for high-volume scientific QA systems.

4. **Cost-sensitive inference deployments:** For organizations serving reasoning models at scale, the 10× throughput improvement translates directly to reduced GPU hours and lower operational costs. A deployment generating 1M queries/day with 32K average output could reduce inference infrastructure by ~90%.

5. **Edge and resource-constrained environments:** While the paper evaluates on A100 GPUs, the memory I/O reduction principle applies to any memory-bandwidth-constrained hardware. Future work could target mobile or edge deployments where memory bandwidth is severely limited.

### 7.4 Reproduction and Integration Guidance

For practitioners considering implementing or extending SambaY, we provide concrete guidance based on the paper's disclosed implementation details.

#### 7.4.1 Codebase and Dependencies

The authors release their training codebase at `https://github.com/microsoft/ArchScale` (Abstract). Key implementation requirements include:

- **vLLM customization:** Section 3.4 notes they "customize the official vLLM framework with version 0.7.3 to support our Phi4-mini-Flash architecture." Standard vLLM will not support GMU layers without modification.
- **FlashAttention integration:** Differential Attention requires FlashAttention [Dao23] operator access. The current implementation uses four passes; optimized kernels would require custom CUDA development.
- **Fused loss kernel:** Section 3.3 mentions a "fused linear cross-entropy loss kernel which is modified from the Liger-Kernel." This modification is necessary for training stability with the 200K vocabulary.

#### 7.4.2 Critical Implementation Details

**Normalization placement is non-negotiable.** Section H (Table 9) demonstrates that shifting normalization before the gate causes a 56.3 point drop in Phonebook accuracy (83.59% → 27.34%). When implementing GMU:
- Apply RMSNorm *after* element-wise gating, not before
- Use nGMU (normalized GMU) for linear attention variants, not standard GMU
- Verify normalization placement matches Section B's GDN-A formulation

**Iso-parametric configuration for fair comparison.** If comparing SambaY to Transformer baselines:
- Solve the aspect ratio equation: $144\alpha d^3 + 14.5\alpha^2 d^3 = 237568 d^3$ for SambaY
- Use $\alpha \approx 124$ for SambaY, $\alpha \approx 126$ for Samba+YOCO (Section C)
- Maintain consistent head dimension ($\alpha_0 = 128$) and GQA group size (4) across architectures

**Training stability mitigations.** If encountering loss divergence during large-scale pre-training:
- Up-cast weights and inputs to FP32 during chunk-wise matrix multiplication for the loss kernel (Figure 6a)
- Add attention dropout of 0.05 at checkpoints before expected divergence (310K steps for self-decoder, 520K for cross-decoder)
- Monitor gradient norms for vector-like parameters; apply zero weight decay to these parameters per µP++ (Section 3.1)

#### 7.4.3 When Not to Use SambaY

Practitioners should avoid SambaY in the following scenarios:

- **Extreme context requirements (>128K):** All models fail at 128K in Table 8. If your application requires 100K+ context reliably, SambaY offers no advantage over standard architectures.
- **Short-response workloads:** For classification, brief QA, or summarization with &lt;500 token outputs, the generation-stage efficiency gains are minimal (Figure 4a).
- **Multi-turn dialogue without cache management:** The decoder-decoder architecture assumes KV caches computed once during prefill. Multi-turn conversations may require cache invalidation or recomputation, eroding efficiency benefits.
- **Limited engineering resources:** The architecture requires vLLM customization, stability monitoring, and careful normalization placement. Teams without infrastructure engineering capacity may find standard Transformers more practical.

### 7.5 Concluding Perspective

The SambaY architecture represents a meaningful step toward **workload-specialized language models** rather than one-size-fits-all architectures. By explicitly optimizing for long-generation reasoning workloads, the authors demonstrate that architectural inductive biases can reduce dependence on expensive post-training methods (RL) while simultaneously improving inference efficiency.

The broader implication extends beyond this specific architecture: **efficiency optimizations should match workload patterns.** As LLM applications diversify—from brief chatbot responses to 50K-token reasoning traces—no single architecture will dominate all scenarios. SambaY establishes a template for designing architectures targeted at specific computational bottlenecks (in this case, generation-stage memory I/O).

For the research community, the paper's methodological contributions—iso-parametric comparison, µP++ scaling for hybrids, transparent reporting of stability issues—may prove as valuable as the architecture itself. These practices enable more rigorous evaluation of future architectural innovations, reducing the risk of confounded comparisons that have plagued prior hybrid model research.

For practitioners, the key takeaway is conditional: **adopt SambaY if your workload matches its optimization target** (long generation, moderate context, compute-constrained training). Otherwise, standard Transformers remain the safer choice. The architecture's success demonstrates that specialized designs can outperform general-purpose models on well-defined tasks—a principle that should guide future efficient language model development.
