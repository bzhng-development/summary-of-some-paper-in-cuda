## 1. Executive Summary

This paper addresses the $O(n^2)$ computational bottleneck of scaled dot-product attention in long-context language models by introducing Higher-order Linear Attention (HLA), a causal streaming mechanism that generalizes linear attention beyond first-order approximations. In the second-order case, HLA maintains a constant-size state of $O(d^2)$ per head and computes per-token outputs in $O(d^2 + d d_v)$ time without materializing $n \times n$ matrices, achieved through compact prefix sufficient statistics and associative scans for parallel training. This approach enables strictly causal, data-dependent mixing with the efficiency of modern recurrent architectures, offering a principled scalable building block that supports exact masked streaming and extensions to third-order interactions.

## 2. Context and Motivation

### The Core Problem: Quadratic Attention Bottleneck

The fundamental obstacle this paper tackles is the **quadratic computational and memory complexity** of scaled dot-product attention with respect to sequence length. As stated in the Introduction (Section 1), standard Transformer attention requires computing an $n \times n$ attention matrix where $n$ is the sequence length. This creates an $O(n^2)$ cost that becomes prohibitive for long contexts.

To understand why this matters, consider the attention computation from Section 2.1:

$$
\text{Attn}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d}} + \Lambda\right)V,
$$

where $Q, K \in \mathbb{R}^{n \times d}$ are queries and keys, $V \in \mathbb{R}^{n \times d_v}$ are values, and $\Lambda$ is the causal mask. The matrix product $QK^\top$ alone produces an $n \times n$ matrix, requiring $O(n^2)$ memory just to store attention weights. For a sequence of 100,000 tokens, this matrix would contain 10 billion entries—clearly infeasible for practical deployment.

This bottleneck constrains **autoregressive language models** specifically, where each new token must attend to all previous tokens. During inference, the model cannot parallelize across time steps, making the quadratic cost a hard limit on achievable context length.

### Why This Matters: Long-Context Language Models

The importance of solving this problem extends across both practical and theoretical dimensions:

**Real-World Impact:**
- **Document Understanding:** Analyzing full legal contracts, research papers, or technical manuals requires contexts far beyond typical 4K–32K token limits.
- **Code Generation:** Understanding entire codebases or multi-file projects demands tracking dependencies across thousands of lines.
- **Conversation History:** Maintaining coherent long-term dialogue requires remembering information from many turns ago without degradation.

**Theoretical Significance:**
- **Expressivity vs. Efficiency Trade-off:** The quadratic attention mechanism provides rich, data-dependent token mixing. Any efficient alternative must preserve this expressivity while reducing complexity.
- **Causality Constraints:** For autoregressive generation, the mechanism must be **strictly causal**—each token can only depend on previous tokens, not future ones. This constraint eliminates many naive approximations.
- **Training-Inference Mismatch:** Many efficient methods work at inference but require different algorithms during training, creating deployment complexity and potential accuracy gaps.

### Prior Approaches and Their Limitations

The paper situates HLA within a rich ecosystem of prior work (Section 8, Related Work). Understanding these approaches clarifies what gap HLA fills.

#### Linear Attention (First-Order Approximations)

Linear attention methods (Katharopoulos et al., 2020; Wang et al., 2020; Choromanski et al., 2020) approximate the softmax kernel using explicit feature maps $\phi: \mathbb{R}^d \to \mathbb{R}^r$. As described in Section 2.2, the attention output becomes:

$$
\text{Attn}(Q, K, V)_i \approx \frac{\phi(q_i)^\top \left(\sum_j \phi(k_j)v_j^\top\right)}{\phi(q_i)^\top \left(\sum_j \phi(k_j)\right)}.
$$

The key insight is maintaining **running sums** $\sum_j \phi(k_j)v_j^\top$ and $\sum_j \phi(k_j)$, which reduces complexity to $O(n \cdot r(d + d_v))$ time and $O(r d_v)$ memory.

**Where Linear Attention Falls Short:**
- **First-Order Only:** These methods maintain only first-order statistics (sums of outer products). They cannot capture higher-order interactions between tokens.
- **Feature Map Limitations:** The quality of approximation depends heavily on the choice of $\phi$. Random features (Performer) or identity maps (Linear Transformer) may not capture complex dependencies.
- **Expressivity Ceiling:** As the paper notes in Section 3, setting $S_t^K = I$ in HLA reduces it to linear attention with kernel $K(q_t, q_i) = q_t^\top q_i$. This reveals linear attention as a **special case** of HLA, not a general solution.

#### State Space Models (SSMs)

State Space Models like S4 (Gu et al., 2021) and Mamba (Gu and Dao, 2023) achieve $O(1)$ per-token state updates through linear recurrences. These models excel at long-range dependencies but handle data-dependent mixing differently from attention.

**Where SSMs Fall Short:**
- **Different Inductive Bias:** SSMs use fixed or selectively parameterized linear dynamics rather than attention-style query-key matching.
- **Limited Data-Dependence:** While Mamba introduces selection mechanisms, the core recurrence structure differs fundamentally from attention's content-addressable retrieval.

#### Modern RNNs

Recent recurrent architectures (Peng et al., 2024; Sun et al., 2024; Yang et al., 2023) emphasize gating, decay mechanisms, and associative scan-friendly recurrences for parallel training.

**Where Modern RNNs Fall Short:**
- **First-Order Statistics:** Like linear attention, these typically maintain first-order sufficient statistics only.
- **Fixed Linear Dynamics:** Many rely on predetermined recurrence structures rather than learning data-dependent metrics from the input itself.

#### Fast Weight Programmers

Fast weight approaches (Schlag et al., 2021; Ba et al., 2016) implement input-dependent synaptic changes through outer-product updates $\Delta W_t \propto k_t v_t^\top$. Section 8 notes the formal equivalence between linearized attention and fast weight programmers.

**Where Fast Weights Fall Short:**
- **Matrix Inversion Overhead:** Some higher-order variants require inverting second-moment matrices, demanding heavier linear algebra that breaks streaming efficiency.
- **HIA Avoids This:** As Section 8 explicitly states, HLA works directly with $S_t^K$ as a learned kernel without explicit matrix inversion, preserving streaming updates.

### The Gap HLA Fills

The paper identifies two central challenges that prior work does not fully address (Section 1):

1. **Strict Autoregressive Causality at Higher Order:** How do you enforce that each token depends only on previous tokens when computing second-order (or higher) interactions, without materializing $n \times n$ matrices or sacrificing streaming efficiency?

2. **Exact Parallel Training:** How do you enable chunk-parallel training that **exactly reproduces** the activations of serial recurrence, avoiding approximate backpropagation through time?

Prior linear attention methods handle causality through simple masking of running sums. However, extending this to **second-order interactions** requires careful algebraic manipulation. The paper's key observation (Section 1) is that "higher attention-like operators admit factorized forms in terms of low-order moments (e.g., sums of key outer products), enabling exact causal streaming without constructing attention matrices."

### How HLA Positions Itself

HLA occupies a distinct niche in the efficiency landscape:

| Property | Standard Attention | Linear Attention | SSMs/RNNs | **HLA** |
|----------|-------------------|------------------|-----------|---------|
| Complexity | $O(n^2)$ | $O(n)$ | $O(n)$ | $O(n)$ |
| Per-token State | $O(n)$ | $O(1)$ | $O(1)$ | $O(1)$ |
| Order of Interactions | Unlimited | First-order | First-order | **Second/Third-order** |
| Data-Dependent Metric | Yes | Limited | Limited | **Yes ($S_t^K$)** |
| Exact Causal Streaming | Yes | Yes | Yes | **Yes** |
| Parallel Training | Yes | Yes | Via Scans | **Via Scans (Exact)** |

**Key Positioning Claims from the Paper:**

- **"Drop-in, attention-like mixer"** (Section 1): HLA replaces only the attention sublayer while keeping feed-forward and normalization unchanged. This enables integration into existing Transformer architectures without redesign.

- **"Attention-style, data-dependent weighting"** (Section 1): Unlike SSMs with fixed dynamics, HLA learns a data-adaptive metric $S_t^K = \sum_{i \leq t} k_i k_i^\top$ that depends on past keys.

- **"Principled, scalable building block"** (Abstract): The paper emphasizes algorithmic structure and implementation over empirical benchmarks, providing formal proofs for masked streaming identities (Theorem 3.1) and scan equivalence (Theorem 4.1).

- **Higher Expressivity with Same Efficiency:** Section 4 notes that HLA "inherits the throughput characteristics of these systems with strictly higher expressivity" compared to first-order linear attention, because it maintains the full key moment matrix rather than compressed feature maps.

The paper deliberately focuses on **algorithmic correctness** and **theoretical guarantees** rather than empirical performance alone. This positions HLA as a foundational mechanism that other work can build upon, rather than a task-specific optimization.

## 3. Technical Approach

### 3.1 Reader orientation (approachable technical breakdown)

This paper presents Higher-order Linear Attention (HLA), a sequence modeling mechanism that computes attention-like token mixing through running statistical summaries instead of explicit attention matrices. The solution addresses the quadratic bottleneck by factorizing higher-order token interactions into compact prefix statistics that can be updated incrementally per token while maintaining exact causal masking and supporting parallel training through associative scan operations.

### 3.2 Big-picture architecture (diagram in words)

The HLA system consists of four interconnected components that work together to replace standard attention while preserving efficiency and expressivity. First, the **Prefix Summary Maintainer** tracks running statistics of keys, queries, and values as tokens arrive, storing matrices like $S_t^K = \sum_{i \leq t} k_i k_i^\top$ that capture second-order moments without growing with sequence length. Second, the **Causal Masking Corrector** applies algebraic corrections using extended summaries ($G_t$ and $h_t$) to ensure each token only depends on previous tokens, not future ones, without materializing any $n \times n$ matrices. Third, the **Associative Scan Operator** enables chunk-parallel training by defining a semidirect-product operation that combines segment summaries while exactly reproducing serial recurrence activations. Fourth, the **Output Computation Module** combines the current query with accumulated summaries to produce per-token outputs in $O(d^2 + d d_v)$ time, where $d$ is the query/key dimension and $d_v$ is the value dimension. Information flows from incoming tokens through summary updates, then through masking corrections, and finally to output generation, with the scan operator providing an alternative parallel path during training that merges chunk-level summaries before local computation.

### 3.3 Roadmap for the deep dive

- We begin with second-order HLA's core mechanism because it establishes the fundamental algebraic insight that higher-order interactions factorize through prefix moments, which all subsequent variants build upon.
- We then explain causal masking via extended summaries because strict autoregression is the most subtle algebraic challenge—without it, the mechanism would cheat by seeing future tokens.
- We cover associative scans for parallel training next because this addresses the practical deployment gap between efficient inference and efficient training on modern hardware.
- We examine the asymmetric variant (AHLA) to show how the same principles apply to different factorization patterns, demonstrating the framework's flexibility.
- We conclude with third-order extensions to illustrate how the algebraic approach generalizes beyond second order, though with increased state complexity.
- Throughout, we reference implementation details and complexity bounds to ground the theory in practical constraints.

### 3.4 Detailed, sentence-based technical breakdown

This is an algorithmic systems paper that introduces a new attention mechanism through formal algebraic derivations, streaming update rules, and parallel training operators rather than empirical benchmarks alone. The core idea is that higher-order attention-like operators can be rewritten as products of low-order prefix moments (running sums of outer products), enabling exact causal streaming without ever constructing the full $n \times n$ attention matrix that makes standard Transformers expensive.

#### Second-Order HLA: The Foundation

The second-order HLA mechanism starts from the observation that a second-order tensor attention can be written as $T_2 := (QK^\top)(QK^\top)^\top = Q(K^\top K)Q^\top \in \mathbb{R}^{n \times n}$, as shown in Section 3. This formulation reveals that the attention weights depend on the second moment matrix $K^\top K \in \mathbb{R}^{d \times d}$, which is a sum of key outer products rather than a sequence-length-dependent matrix. The critical insight is that $K^\top K$ can be maintained as a running prefix summary $S_t^K = \sum_{i \leq t} k_i k_i^\top$ that updates incrementally as each new token arrives, keeping memory constant at $O(d^2)$ regardless of how many tokens have been processed.

At each time step $t$, HLA maintains three core prefix summaries that capture different aspects of the token history. The key moment matrix $S_t^K \in \mathbb{R}^{d \times d}$ accumulates outer products of past keys, acting as a learned, data-dependent metric on query space that adapts based on what keys have appeared. The query-value accumulator $C_t^{QV} \in \mathbb{R}^{d \times d_v}$ sums outer products of queries with their corresponding values, capturing what information has been associated with different query directions. The query mass vector $m_t^Q \in \mathbb{R}^d$ simply sums past queries, providing a scalar-like summary for optional normalization. All three summaries update in streaming fashion with $O(d^2)$ cost for $S_t^K$ and $O(d d_v)$ cost for $C_t^{QV}$ per token, as stated in Section 3.

The unnormalized output at time $t$ follows from combining these summaries with the current query: $o_t = q_t^\top S_t^K C_t^{QV}$. This equation avoids forming any $n \times n$ matrices because $S_t^K$ is $d \times d$ (independent of sequence length) and the multiplication order ensures we never materialize intermediate tensors that scale with $n$. For numerical stability and scale control, HLA optionally supports a normalized variant that divides by a denominator: $o_t = \frac{q_t^\top S_t^K C_t^{QV}}{q_t^\top S_t^K m_t^Q + \varepsilon}$, where $\varepsilon > 0$ is a small constant (Equation 3.2 in Section 3). The paper notes that the unnormalized form is the default operator, with normalization as an option built from the same summaries.

The connection to first-order linear attention clarifies what HLA generalizes. If we set $S_t^K = I$ (the identity matrix), the numerator becomes $q_t^\top C_t^{QV} = \sum_{i \leq t} (q_t^\top q_i) v_i^\top$, which is exactly linear attention with kernel $K(q_t, q_i) = q_t^\top q_i$. When queries and keys are tied ($q_i \equiv k_i$), this coincides with linear attention using the identity feature map $\phi(x) = x$. However, in general second-order HLA implements a data-adaptive degree-2 polynomial kernel $K_t(q, q') = q^\top S_t^K q'$ whose metric $S_t^K = \sum_{i \leq t} k_i k_i^\top$ depends on past keys, strictly enriching first-order linearizations while retaining streaming capability (Section 3, "Connection with linear attention").

#### Causal Masking via Extended Summaries

The most subtle algebraic challenge in HLA is enforcing strict autoregressive causality at second order without sacrificing streaming updates or introducing any $n \times n$ intermediates. In standard attention, causality is enforced through a mask matrix $\Lambda$ that sets future attention weights to $-\infty$ before the softmax. In linear attention, causality is simpler because running sums naturally only include past tokens. However, second-order interactions create a complication: the term $(QK^\top)(QK^\top)^\top$ involves products of attention weights at different positions, and naively using prefix summaries would allow token $t$ to indirectly depend on future tokens through the second-moment structure.

The paper solves this through Theorem 3.1, which derives an exact masked formulation by augmenting the state with two additional summaries. Let $L$ denote the binary causal mask (lower-triangular, including the diagonal, with ones on and below the diagonal and zeros above). For the masked second-order matrix, the entry at position $(t, j)$ is $[ (L \odot QK^\top)(L \odot QK^\top)^\top ]_{t,j} = \sum_{i \leq \min(t,j)} (q_t^\top k_i)(q_j^\top k_i) = q_t^\top S_{\min(t,j)}^K q_j$, where $\odot$ denotes the Hadamard (elementwise) product. The strictly causal output at time $t$ requires masking on the right before applying values, written in matrix form as $o_t = \left( [ (L \odot QK^\top)(L \odot QK^\top)^\top ] \odot L \right)_{t,:} V$, which enforces the restriction $j \leq t$ when multiplying by $V$ (Section 3.1).

To achieve this masking in streaming form, the paper defines two additional prefix summaries beyond the core three. The first is $G_t = \sum_{i \leq t} (k_i k_i^\top) C_{i-1}^{QV} \in \mathbb{R}^{d \times d_v}$, which accumulates key outer products weighted by the query-value summary up to the previous token. The second is $h_t = \sum_{i \leq t} (k_i k_i^\top) m_{i-1}^Q \in \mathbb{R}^d$, which does the same for the query mass vector. These summaries capture the "cross terms" that would otherwise allow future tokens to influence current outputs through the second-order structure.

Theorem 3.1 (Masked streaming identity for second order) states that for each $t$, the masked numerator is $\text{num}_t^{\text{mask}} = q_t^\top (S_t^K C_t^{QV} - G_t)$ and the masked denominator is $\text{den}_t^{\text{mask}} = q_t^\top (S_t^K m_t^Q - h_t)$. The strictly causal, masked default unnormalized output is then $o_t = q_t^\top (S_t^K C_t^{QV} - G_t)$ (Equation 3.3). The optional linear normalization divides by the masked denominator: $o_t = \frac{q_t^\top (S_t^K C_t^{QV} - G_t)}{q_t^\top (S_t^K m_t^Q - h_t) + \varepsilon}$ (Equation 3.4). The proof in Section 3.1 shows this by expanding the masked matrix product and interchanging finite sums to isolate terms that depend only on past tokens, with $G_t$ precisely capturing the correction needed to remove future dependence.

The online updates for all five summaries maintain constant per-token cost. Using the fact that $(k k^\top) X = k (k^\top X)$ to avoid cubic operations, the updates are: $S_t^K = S_{t-1}^K + k_t k_t^\top$, $C_t^{QV} = C_{t-1}^{QV} + q_t v_t^\top$, $m_t^Q = m_{t-1}^Q + q_t$, $G_t = G_{t-1} + k_t (k_t^\top C_{t-1}^{QV})$, and $h_t = h_{t-1} + k_t (k_t^\top m_{t-1}^Q)$. Therefore, the per-token cost remains $O(d^2 + d d_v)$ in total, as stated in Section 3.1 under "Online updates." The key optimization is computing $k_t^\top C_{t-1}^{QV}$ as a matrix-vector product ($O(d d_v)$) before the outer product with $k_t$, rather than forming the full $d \times d$ matrix $(k_t k_t^\top)$ first and then multiplying by $C_{t-1}^{QV}$ (which would be $O(d^2 d_v)$).

#### Associative Scans for Parallel Training

While the streaming recurrence is efficient for inference, training a purely recurrent model is inefficient on GPUs because it prevents parallelization across the sequence dimension. The paper addresses this through chunk-parallel training based on associative scans, a technique widely used in linear attention literature (Yang et al., 2023; Qin et al., 2024) but extended here to handle HLA's more complex state structure. The approach adopts within-chunk scans with width $w$ and inter-chunk scans across $B_c$ chunks, where $B_c$ denotes the number of chunks to avoid overloading notation (Section 4).

The foundation is defining an associative operator that combines segment summaries such that a standard exclusive Blelloch scan produces forward activations identical to those of a serial recurrence. For the unmasked case, the paper defines a token "delta" tuple $T_t = (\Delta S_t, \Delta C_t, \Delta m_t)$ where $\Delta S_t = k_t k_t^\top$, $\Delta C_t = q_t v_t^\top$, and $\Delta m_t = q_t$. The additive monoid operation is $(S_A, C_A, m_A) \oplus (S_B, C_B, m_B) = (S_A + S_B, C_A + C_B, m_A + m_B)$, where $A$ then $B$ denotes adjacent segments in time (all indices in $A$ precede those in $B$). An exclusive Blelloch scan on $\{T_1, \ldots, T_w\}$ yields per-token prefixes $P_t = \bigoplus_{i &lt; t} T_i$, from which the inclusive state at $t$ is obtained locally by adding $T_t$ (Section 4.1).

For the masked case, the state tuple expands to $S = (S, C, m, G, h)$, and for a single-token segment, $G = h = 0$. The concatenation operator becomes a semidirect product: $(S_A, C_A, m_A, G_A, h_A) \oplus (S_B, C_B, m_B, G_B, h_B) = (S_A + S_B, C_A + C_B, m_A + m_B, G_A + G_B + S_B C_A, h_A + h_B + S_B m_A)$ (Equation 4.1 in Section 4.2). The cross-terms $S_B C_A$ and $S_B m_A$ are critical: they capture how keys in segment $B$ interact with query-value and query-mass summaries from segment $A$, which is exactly what the streaming recurrence would compute if run serially. This operator is associative by direct expansion, meaning $(A \oplus B) \oplus C = A \oplus (B \oplus C)$, which is the mathematical property that enables parallel scan algorithms.

The paper also supports exponential decay, which controls spectral growth and improves recency bias while maintaining associativity. Let $\gamma \in (0, 1)$ be a fixed exponential decay, and let a segment $X$ carry its length $\ell(X)$ and attenuation $\rho(X) := \gamma^{\ell(X)}$. For the unmasked triple, the decayed concatenation is $(S_A, C_A, m_A, \rho_A) \oplus_\gamma (S_B, C_B, m_B, \rho_B) = (\rho_B S_A + S_B, \rho_B C_A + C_B, \rho_B m_A + m_B, \rho_A \rho_B)$. The masked version extends this analogously with additional terms for $G$ and $h$ that incorporate the attenuation factors (Section 4.2, "Decay-aware monoid"). Decay is important in practice because without it, the prefix summaries can grow unbounded over very long sequences, causing numerical instability or overwhelming recent token information.

Theorem 4.1 (Scan equivalence: serial vs. (decayed) associative scans) formally establishes that the parallel scan produces identical activations to serial recurrence. The theorem states: consider a sequence of token segments $\{T_1, \ldots, T_n\}$ and either $\oplus$ (no decay) or $\oplus_\gamma$ (with decay). Let $P_t$ be the exclusive prefix obtained by a Blelloch scan under the chosen operator. For each $t$, the inclusive state computed locally from $P_t$ and $T_t$ equals the state produced by a serial left-to-right recurrence on tokens $1:t$. Consequently, the per-token masked outputs are identical to those of the serial algorithm. The proof relies on the associativity of $\oplus_\gamma$ and the fact that the Blelloch scan computes $P_t = E \oplus_\gamma T_1 \oplus_\gamma \cdots \oplus_\gamma T_{t-1}$ where $E$ is the identity (zero-length segment with all-zero summaries and $\rho = 1$), so the local inclusive update $P_t \oplus_\gamma T_t$ equals the serial state $X_t$ by associativity (Section 4.2).

The training parallelism operates at two levels. Within a chunk of width $w$, an exclusive Blelloch scan over $\{T_1, \ldots, T_w\}$ under $\oplus$ (or $\oplus_\gamma$) yields $P_t$ for all $t$ in $O(\log w)$ span and $O(1)$ auxiliary memory per position. The per-token inclusive states are then computed independently as $I_t = P_t \oplus T_t$. For inter-chunk parallelism, each chunk $c$ produces a single summary $S^{(c)} = \bigoplus_{t \in \text{chunk } c} T_t$. An exclusive scan across the $B_c$ summaries gives carry-in prefixes $\tilde{P}^{(c)}$ for every chunk. Each position $t$ in chunk $c$ then uses the merged prefix $\tilde{P}^{(c)} \oplus P_t$ before adding its local $T_t$ to obtain the inclusive state. This is the same parallel skeleton widely used in modern linear-attention and recurrent networks (Section 4.2, "Intra-chunk parallelism" and "Inter-chunk parallelism").

For backward pass gradients, the paper defines $\oplus^*$ as the vector-Jacobian adjoint of $\oplus$ evaluated at the forward states. A reverse (decayed) scan applying $\oplus_\gamma^*$ with checkpointing at tile boundaries yields gradients that match those of the serial recurrence, by Theorem 4.1 and the chain rule (Section 4.2, "Backward for gradients"). This is important because it means HLA does not require approximate backpropagation through time—the exact gradients are computable through the same scan infrastructure used for the forward pass.

Algorithm 1 in Section 5.1 provides explicit pseudocode for masked second-order HLA with within-chunk scan. The algorithm takes chunk tokens $(q_{[1:w]}, k_{[1:w]}, v_{[1:w]})$, stability constant $\varepsilon$, optional ridge $\lambda$, optional decay $\gamma$, and an optional normalize flag. It first computes token deltas, then performs an exclusive scan using the $\oplus$ operator from Equation 4.1, then computes inclusive states and outputs in parallel across the chunk. The optional ridge regularization adds $\lambda I$ to the effective $S$ matrix for stability, though the paper notes this does not correspond to the exact masked bilinear form of $(L \odot QK^\top)$ (Remark after Algorithm 1).

#### Asymmetric Higher-order Linear Attention (AHLA)

Beyond the symmetric second-order formulation, the paper introduces a complementary asymmetric variant called AHLA (Asymmetric Higher-order Linear Attention) that uses a different factorization pattern. The motivation is that the second-order HLA in Section 3 realizes the symmetric triple product $A A^\top V$ with $A = QK^\top$ (masked later), but there is an alternative left-cascaded product $A A V = Q (K^\top Q) (K^\top V)$ that also admits strictly causal streaming with $O(d^2 + d d_v)$ per-token cost (Section 6).

The AHLA weights are defined as $(AA)_{t,j} = \sum_{i=j}^t (q_t^\top k_i) (q_i^\top k_j)$ for $j \leq t$, where $A = L \odot (QK^\top)$ is the causally masked affinity. To obtain strictly causal outputs when applying to values, the final multiplication is interpreted as $((AA) \odot L) V$, with the streaming identity in Theorem 6.1 implementing exactly this row-wise masking. The unnormalized output is $o_t^{\text{AHLA}} = \sum_{j \leq t} \sum_{i=j}^t (q_t^\top k_i)(q_i^\top k_j) v_j^\top$ (Equation 6.1 in Section 6.1).

AHLA introduces different streaming prefix summaries than symmetric HLA. The key-value accumulator $P_t^{KV} = \sum_{j \leq t} k_j v_j^\top \in \mathbb{R}^{d \times d_v}$ sums key-value outer products. The key mass $m_t^K = \sum_{j \leq t} k_j \in \mathbb{R}^d$ sums keys. The cross-summary $E_t = \sum_{i \leq t} k_i (q_i^\top P_i^{KV}) \in \mathbb{R}^{d \times d_v}$ accumulates keys weighted by how much their queries attend to past key-value information. The denominator summary $n_t = \sum_{i \leq t} k_i (q_i^\top m_i^K) \in \mathbb{R}^d$ does the same for key mass. These differ from symmetric HLA's summaries because the factorization pattern is different—AHLA routes through intermediate key indices rather than through the key moment matrix.

Theorem 6.1 (Masked streaming identity for AHLA) states that with the above definitions, $o_t^{\text{AHLA}} = q_t^\top E_t$ and the optional normalized variant is $\hat{o}_t^{\text{AHLA}} = \frac{q_t^\top E_t}{q_t^\top n_t + \varepsilon}$. The online strictly causal updates are: $P_t^{KV} = P_{t-1}^{KV} + k_t v_t^\top$, $m_t^K = m_{t-1}^K + k_t$, $E_t = E_{t-1} + k_t (q_t^\top P_t^{KV})$, and $n_t = n_{t-1} + k_t (q_t^\top m_t^K)$. The proof follows from fixing $i$ and summing over $j \leq i$ to get $\sum_{j \leq i} (q_i^\top k_j) v_j^\top = q_i^\top P_i^{KV}$, then substituting into the double sum (Section 6.1, Proof).

For streaming inference, the dominant work is forming $q_t^\top P_t^{KV} \in \mathbb{R}^{1 \times d_v}$ and the outer product $k_t (\cdot)$, giving $O(d d_v)$ time and $O(d d_v + d)$ state per head for $P, E, m, n$. For chunk-parallel scans used in training, an additional block statistic $R^{KQ}$ appears only inside the concatenation operator, contributing $O(d^2)$ memory per chunk summary but not to the streaming path (Section 6.1, "Cost"). The decay mechanism with $\gamma \in (0, 1)$ applies analogously: $P_t^{KV} = \gamma P_{t-1}^{KV} + k_t v_t^\top$, and similarly for other summaries, preserving associativity of the scan operator.

The chunk-parallel scan for AHLA uses an augmented state $S = (R^{KQ}, P^{KV}, m^K, E, n)$ where $R^{KQ} = \sum_{i \in \text{segment}} k_i q_i^\top \in \mathbb{R}^{d \times d}$ is the segment-level key-query cross moment. This is used only during chunk concatenation to form cross terms and is not required by the serial streaming forward path. The undecayed associative concatenation is $(R_A, P_A, m_A, E_A, n_A) \oplus_{\text{AHLA}} (R_B, P_B, m_B, E_B, n_B) = (R_A + R_B, P_A + P_B, m_A + m_B, E_A + E_B + R_B P_A, n_A + n_B + R_B m_A)$ (Equation 6.2 in Section 6.2). The decayed version incorporates attenuation factors analogously to symmetric HLA.

Algorithm 2 in Section 6.3 provides pseudocode for AHLA streaming with causal mask and optional decay. The algorithm initializes $P=0_{d \times d_v}$, $m=0_d$, $E=0_{d \times d_v}$, $n=0_d$, then iterates through tokens updating summaries and computing outputs. The relation to $A A^\top V$ is that AHLA emphasizes a matrix power of $A$, weighting each value $v_j$ through a single pass $q_i^\top k_j$ routed by an intermediate key index $i$, whereas symmetric $A A^\top V$ aggregates via the metric $S^K$ and query summaries. Both are second-order, strictly causal, and stream with identical asymptotic costs but induce different inductive biases (Section 6.3, "Relation to $A A^\top V$").

#### Third-Order Linear Attention Extension

The paper extends the algebraic framework to third-order interactions, demonstrating that the prefix summary approach generalizes beyond second order. The third-order tensor attention mechanism uses the matrix $A A^\top A$ where $A = QK^\top \in \mathbb{R}^{n \times n}$. The $(t, j)$-entry is $[(A A^\top A)]_{t,j} = \sum_{u \leq n} \left( \sum_{i \leq n} (q_t^\top k_i)(q_u^\top k_i) \right) (q_u^\top k_j) = q_t^\top (K^\top K) \left( \sum_u q_u q_u^\top \right) k_j$, which immediately yields a streaming factorization through prefix moments (Section 7.1).

The unmasked factorization defines prefix summaries $S_t^K = \sum_{i \leq t} k_i k_i^\top \in \mathbb{R}^{d \times d}$, $S_t^Q = \sum_{i \leq t} q_i q_i^\top \in \mathbb{R}^{d \times d}$, $P_t^{KV} = \sum_{i \leq t} k_i v_i^\top \in \mathbb{R}^{d \times d_v}$, and $m_t^K = \sum_{i \leq t} k_i \in \mathbb{R}^d$. The default unnormalized third-order operator is $o_t^{(3)} = q_t^\top S_t^K S_t^Q P_t^{KV}$. An optional normalization divides by $q_t^\top S_t^K S_t^Q m_t^K + \varepsilon$ if desired (Section 7.1, "Unmasked factorization").

To impose strict causality at third order, the paper introduces six cross-summaries (three for numerator, three for denominator). The numerator corrections are: $G_t^{(1)} = \sum_{i \leq t} (k_i k_i^\top) S_{i-1}^Q P_{i-1}^{KV} \in \mathbb{R}^{d \times d_v}$, $G_t^{(2)} = \sum_{i \leq t} S_{i-1}^K (q_i q_i^\top) P_{i-1}^{KV} \in \mathbb{R}^{d \times d_v}$, and $G_t^{(3)} = \sum_{i \leq t} S_{i-1}^K S_{i-1}^Q (k_i v_i^\top) \in \mathbb{R}^{d \times d_v}$. The denominator corrections $h_t^{(1)}, h_t^{(2)}, h_t^{(3)}$ follow the same pattern but replace $P^{KV}$ with $m^K$ (Section 7.1, "Masked streaming summaries").

The masked unnormalized quantities are $\text{num}_t^{(3)\text{mask}} = q_t^\top (S_t^K S_t^Q P_t^{KV} - G_t^{(1)} - G_t^{(2)} - G_t^{(3)})$ and $\text{den}_t^{(3)\text{mask}} = q_t^\top (S_t^K S_t^Q m_t^K - h_t^{(1)} - h_t^{(2)} - h_t^{(3)})$. Theorem 7.1 (Masked streaming identity for third order) states that the strictly causal third-order output in default unnormalized form is $o_t^{(3)} = \text{num}_t^{(3)\text{mask}}$, with optional normalization dividing by $\text{den}_t^{(3)\text{mask}} + \varepsilon$ (Section 7.1).

The online updates for third-order HLA are more complex but follow the same streaming principle. The first-order updates are $S_t^K = S_{t-1}^K + k_t k_t^\top$, $S_t^Q = S_{t-1}^Q + q_t q_t^\top$, $P_t^{KV} = P_{t-1}^{KV} + k_t v_t^\top$, and $m_t^K = m_{t-1}^K + k_t$ (Equation 7.1). The cross-summary updates are $G_t^{(1)} = G_{t-1}^{(1)} + (k_t k_t^\top) S_{t-1}^Q P_{t-1}^{KV}$, $G_t^{(2)} = G_{t-1}^{(2)} + S_{t-1}^K (q_t q_t^\top) P_{t-1}^{KV}$, and $G_t^{(3)} = G_{t-1}^{(3)} + S_{t-1}^K S_{t-1}^Q (k_t v_t^\top)$ (Equation 7.2), with analogous updates for $h^{(1:3)}$ (Equation 7.3). Algorithm 3 in Section 7.2 provides explicit pseudocode for the masked third-order streaming kernel.

The paper notes that designing a chunk-parallel scan operator that exactly reproduces serial activations for third-order HLA requires additional segment-level summaries beyond $(S^K, S^Q, P^{KV}, m^K, G^{(1:3)}, h^{(1:3)})$, and leaves this composition to future work (Section 7.2, "Usage"). This is an important limitation: while the streaming inference is fully specified, the parallel training infrastructure for third-order is not yet developed in this paper.

#### Implementation Details and Complexity

The implementation considerations in Section 5 ground the theoretical development in practical constraints. For each token and each head at second order, the state consists of $S_t^K \in \mathbb{R}^{d \times d}$, $C_t^{QV} \in \mathbb{R}^{d \times d_v}$, $m_t^Q \in \mathbb{R}^d$, and for masked variants $G_t \in \mathbb{R}^{d \times d_v}$ and $h_t \in \mathbb{R}^d$. The compute evaluates $u_t = q_t^\top S_t^K$ (matrix-vector product, $O(d^2)$) and then $u_t C_t^{QV}$ (row-matrix product, $O(d d_v)$), with masked corrections $-q_t^\top G_t$. The denominator uses $u_t m_t^Q - q_t^\top h_t$. This avoids forming $S_t^K C_t^{QV}$ explicitly, and masked cross-terms use $k_t^\top X$ to avoid cubic cost (Section 5).

HLA only replaces the standard attention sublayer in the Transformer block, while the feed-forward sublayer and normalization sublayers remain unchanged. Drop-in replacement requires only swapping the kernel while keeping positional encodings and masking identical to the baseline. Multi-query keys/values (sharing $K, V$ across heads) reduce state from $O(h d^2)$ to $O(d^2 + h d d_v)$ while preserving the algebra, where $h$ is the number of heads. The summaries $(S, C, m, G, h)$ are per head, but with multi-query $(K, V)$ shared across heads, the key moment $S_t^K$ is shared and stored once per layer ($O(d^2)$), while $(C_t^{QV}, m_t^Q, G_t, h_t)$ remain per-head ($O(h d d_v + h d)$). This yields total memory of $O(d^2 + h d d_v)$ instead of $O(h d^2 + h d d_v)$ when each head maintains its own $S_t^K$ (Section 5.2).

For throughput optimization, the paper recommends maintaining $S_t^K$ in a packed symmetric layout, storing only the upper triangle ($\frac{1}{2} d(d+1)$ entries) to reduce bandwidth without changing the algebra. Within a chunk of width $w$, use an exclusive Blelloch scan to obtain prefixes in $O(\log w)$ span and constant extra memory per position; inter-chunk scans use the same operator across $B_c$ chunks (Section 5.2). The complexity summary from the Abstract and Section 1 is that second-order HLA maintains constant-size state and computes per-token outputs in linear time $O(d^2 + d d_v)$, independent of sequence length $n$.

#### Design Choices and Trade-offs

Several design choices in HLA warrant explicit explanation because they reflect important trade-offs. The paper defaults to unnormalized output rather than normalized because this "avoids length-dependent renormalization while preserving streaming updates and the same state as the normalized variant" (Section 3, "Unnormalized HLA"). Normalization is available as an option built from the same summaries, giving users flexibility without requiring separate implementations.

The choice to work directly with $S_t^K = \sum_{i \leq t} k_i k_i^\top$ rather than its inverse is deliberate. Section 8 (Related Work) notes that inverse-based methods "typically require heavier linear algebra" and that "working directly with $S_t^K$ avoids explicit matrix inversion and preserves streaming updates." This is a critical efficiency decision: matrix inversion is $O(d^3)$ and would break the $O(d^2)$ per-token budget, whereas using $S_t^K$ directly only requires matrix-vector products.

The paper deliberately focuses on algorithmic structure and implementation rather than empirical benchmarks. Section 1 states: "We deliberately focus on algorithmic structure and implementation." This positions HLA as a foundational mechanism that other work can build upon, rather than a task-specific optimization. The contributions are formal proofs (Theorem 3.1, Theorem 4.1, Theorem 6.1, Theorem 7.1) and explicit algorithms (Algorithm 1, 2, 3) rather than accuracy tables.

The extension to third order demonstrates the framework's generality but also reveals scaling limits. The state grows from five summaries at second order (symmetric HLA: $S^K, C^{QV}, m^Q, G, h$) to ten summaries at third order ($S^K, S^Q, P^{KV}, m^K, G^{(1:3)}, h^{(1:3)}$), and the per-token compute increases accordingly. The paper does not provide empirical guidance on whether third-order interactions are worth the additional cost in practice, noting only that the algebraic framework extends (Section 7).

One limitation the paper acknowledges is that chunk-parallel scan operators for third-order HLA are not fully developed. Section 7.2 states: "Designing a chunk-parallel scan operator that exactly reproduces serial activations requires additional segment-level summaries beyond $(S^K, S^Q, P^{KV}, m^K, G^{(1:3)}, h^{(1:3)})$; we leave this composition to future work." This is an important gap: without parallel training infrastructure, third-order HLA would be limited to research settings where serial training is acceptable, not production systems requiring GPU efficiency.

## 4. Key Insights and Innovations

This section identifies the fundamental conceptual advances that distinguish HLA from prior efficient attention mechanisms. Rather than re-explaining the mechanics covered in Section 3, we focus on **why** these innovations matter and what makes them qualitatively different from incremental improvements.

---

### 4.1 Exact Causal Masking at Higher Order Without Approximation

**The Innovation:** Prior linear attention methods enforce causality trivially—running sums naturally include only past tokens. However, this breaks down at second order because the interaction $(QK^\top)(QK^\top)^\top$ creates dependencies where token $t$ could indirectly "see" future tokens through the second-moment structure. The paper's key insight is that **exact causality can be preserved through algebraic correction terms** ($G_t$ and $h_t$) rather than approximation or truncation.

**What Makes This Different:**

| Approach | Causality at Second Order | Cost of Enforcement |
|----------|--------------------------|---------------------|
| Standard Attention | Exact (via mask matrix) | $O(n^2)$ |
| Linear Attention | Exact (trivial, first-order only) | $O(1)$ |
| Prior Higher-Order Methods | Approximate or requires inversion | $O(d^3)$ or heuristic |
| **HLA (This Work)** | **Exact via extended summaries** | **$O(1)$ streaming** |

The significance lies in Theorem 3.1's proof structure: rather than approximating the masked second-order operator, the paper derives an **exact algebraic identity** that decomposes the masked computation into prefix summaries plus correction terms. This is not an engineering trick but a structural property of the tensor algebra itself.

**Why This Matters:** Many efficient attention variants sacrifice theoretical guarantees for speed. HLA demonstrates that **exactness and efficiency are not mutually exclusive** at higher order. This matters for downstream reliability—approximate masking can introduce subtle bugs in autoregressive generation where a single token seeing future information corrupts the entire sequence.

---

### 4.2 Training-Inference Exactness Through Associative Scan Equivalence

**The Innovation:** Section 4 establishes that parallel training via associative scans produces **bitwise-identical activations** to serial recurrence (Theorem 4.1). This is stronger than typical "compatible with scans" claims—the paper proves the scan operator $\oplus_\gamma$ is mathematically equivalent to the serial update function $\Phi_\gamma$.

**What Makes This Different:**

Many recurrent-style efficient models face a **training-inference mismatch**:
- **Inference:** Uses streaming recurrence (efficient, $O(1)$ state)
- **Training:** Uses different parallel algorithm (approximate gradients, different numerical behavior)

HLA eliminates this gap entirely. The same algebraic structure supports both modes:
- **Serial mode:** $X_t = \Phi_\gamma(X_{t-1}, T_t)$
- **Parallel mode:** $P_t = \text{Scan}_\oplus(\{T_1, \ldots, T_{t-1}\})$, then $X_t = P_t \oplus T_t$

By Theorem 4.1, these are provably equal, not approximately similar.

**Why This Matters:** This eliminates a major deployment risk. Models trained with approximate parallel methods often exhibit **distribution shift** at inference time when switched to streaming mode. HLA's exactness guarantee means:
- No fine-tuning needed when switching between training and inference modes
- Gradients flow through the exact same computation graph used at inference
- Numerical stability properties are preserved across both regimes

This is a **fundamental architectural guarantee** rather than an empirical observation—it holds by construction, not by tuning.

---

### 4.3 Data-Dependent Metrics Without Matrix Inversion

**The Innovation:** Section 8 (Related Work) notes that prior fast-weight methods achieving higher-order interactions typically require **inverting second-moment matrices** (e.g., $(K^\top K)^{-1}$), which costs $O(d^3)$ and breaks streaming efficiency. HLA's insight is to **work directly with $S_t^K = \sum k_i k_i^\top$ as a learned kernel** rather than inverting it.

**What Makes This Different:**

The paper positions $S_t^K$ as a **data-adaptive metric on query space** (Section 3, "Connection with linear attention"). This is conceptually distinct from:
- **Fixed metrics** (standard linear attention uses identity or random features)
- **Inverted metrics** (fast-weight programmers that compute $(K^\top K)^{-1}$ for preconditioning)

By avoiding inversion, HLA maintains $O(d^2)$ per-token cost while still capturing second-order structure. The kernel $K_t(q, q') = q^\top S_t^K q'$ adapts based on observed keys without requiring expensive linear algebra.

**Why This Matters:** Matrix inversion is a **hard complexity barrier**—it cannot be parallelized across tokens and introduces numerical instability for ill-conditioned $K^\top K$. HLA sidesteps this entirely by recognizing that **the inverse is not necessary for expressive higher-order mixing**. This is a conceptual shift: instead of asking "how do we efficiently invert the second moment?", the paper asks "do we need to invert it at all?"

---

### 4.4 Generalizable Algebraic Framework (Beyond Second Order)

**The Innovation:** Section 7 demonstrates that the prefix-summary approach **extends systematically to third order** (and implicitly higher). The paper provides complete masked streaming formulas for third-order HLA with six cross-summaries ($G^{(1:3)}$, $h^{(1:3)}$), showing the algebraic pattern generalizes.

**What Makes This Different:**

Most efficient attention methods are **order-specific**:
- Linear attention: inherently first-order (sums of $\phi(k)v^\top$)
- SSMs: fixed recurrence order (typically first-order state evolution)
- Prior higher-order attempts: ad-hoc formulations without clear generalization path

HLA reveals a **recursive algebraic structure**: each increase in order adds predictable cross-summaries that correct for future-token leakage. The pattern in Section 7.1 shows:
- Second order: 2 correction summaries ($G, h$)
- Third order: 6 correction summaries ($G^{(1:3)}, h^{(1:3)}$)

This suggests a principled scaling law for state complexity rather than reinventing the mechanism at each order.

**Why This Matters:** This positions HLA as a **framework** rather than a single mechanism. Practitioners can choose the order based on their expressivity-efficiency trade-off:
- **Second order:** $O(d^2)$ state, suitable for most long-context tasks
- **Third order:** Higher expressivity for tasks requiring complex token interactions (e.g., reasoning chains)

The paper acknowledges that parallel training infrastructure for third-order is left to future work (Section 7.2, "Usage"), but the **streaming inference is fully specified**. This transparency about what is complete vs. provisional is itself a contribution—it maps the boundary of what is solved.

---

### 4.5 Multiple Second-Order Formulations With Different Inductive Biases

**The Innovation:** The paper introduces **two distinct second-order mechanisms**: symmetric HLA ($A A^\top V$) and asymmetric AHLA ($A A V$), both streaming at $O(d^2 + d d_v)$ cost but with different factorization patterns (Section 6).

**What Makes This Different:**

Prior work typically presents **one efficient attention variant** as "the solution." HLA demonstrates there are **multiple valid second-order formulations**:
- **Symmetric HLA:** Routes through key moment matrix $S_t^K$, emphasizing query-query interactions modulated by past keys
- **Asymmetric AHLA:** Routes through intermediate key indices, emphasizing sequential key-query-key chains

Both are strictly causal, both stream at the same asymptotic cost, but they induce **different inductive biases** (Section 6.3, "Relation to $A A^\top V$").

**Why This Matters:** This reveals that "higher-order linear attention" is not a single point in design space but a **family of mechanisms**. Different tasks may benefit from different factorization patterns:
- Tasks requiring symmetric token relationships (e.g., bidirectional understanding within causal constraints) may prefer symmetric HLA
- Tasks emphasizing sequential dependency chains may prefer AHLA

The paper does not empirically compare these variants (consistent with its focus on algorithmic structure), but providing **multiple valid options** gives practitioners flexibility that single-mechanism papers do not.

---

### Summary: Incremental vs. Fundamental

| Contribution | Incremental or Fundamental? | Reasoning |
|--------------|----------------------------|-----------|
| Exact masked streaming at second order | **Fundamental** | First method to achieve exact causality at higher order without approximation or $O(n^2)$ cost |
| Associative scan equivalence | **Fundamental** | Proves exact training-inference match, eliminating a common deployment gap |
| No matrix inversion | **Fundamental** | Conceptual shift: higher-order mixing does not require inverting second moments |
| Generalizable to third order | **Fundamental** | Reveals recursive algebraic structure, not a one-off trick |

The core theoretical contributions (exact masking, scan equivalence, inversion-free higher-order) are **fundamental innovations** that change what is theoretically possible in efficient attention. The multiple variants and third-order extension demonstrate the framework's generality but require empirical validation to assess practical significance.

## 6. Limitations and Trade-offs

This section critically examines the assumptions, constraints, and open questions surrounding HLA. While the paper establishes strong theoretical guarantees, several practical limitations warrant careful consideration before deployment.

---

### 6.1 Unresolved Parallel Training for Third-Order HLA

**The Limitation:** While the paper provides complete streaming inference formulas for third-order HLA (Section 7.1, Algorithm 3), the **chunk-parallel training infrastructure is explicitly incomplete**. Section 7.2 states:

> "Designing a chunk-parallel scan operator that exactly reproduces serial activations requires additional segment-level summaries beyond $(S^K, S^Q, P^{KV}, m^K, G^{(1:3)}, h^{(1:3)})$; we leave this composition to future work."

**Why This Matters:**

| Order | Streaming Inference | Parallel Training |
|-------|---------------------|-------------------|
| Second-order | ✅ Complete (Algorithm 1) | ✅ Complete (Section 4.2) |
| Third-order | ✅ Complete (Algorithm 3) | ❌ **Not developed** |

This creates a significant deployment gap:
- **Research settings:** Serial training may be acceptable for small-scale experiments
- **Production systems:** GPU-efficient parallel training is essential for long sequences
- **Scalability:** Without parallel training, third-order HLA cannot compete with second-order variants in practice

**The Underlying Challenge:** The number of cross-summaries grows combinatorially with order. Second-order requires 2 correction summaries ($G, h$), while third-order requires 6 ($G^{(1:3)}, h^{(1:3)}$). The segment-level statistics needed for associative scans likely grow similarly, making the concatenation operator increasingly complex.

**What Readers Should Know:** If you are considering third-order HLA for a real system, you must either:
1. Accept serial training limitations
2. Develop the missing scan operator yourself
3. Wait for follow-up work that completes this infrastructure

---

### 6.2 State Complexity Scales Quadratically with Feature Dimension

**The Limitation:** While HLA achieves $O(1)$ state per token **with respect to sequence length $n$**, the per-head state is $O(d^2)$ where $d$ is the query/key dimension. Section 5 states:

> "State: $S_t^K \in \mathbb{R}^{d \times d}$, $C_t^{QV} \in \mathbb{R}^{d \times d_v}$, $m_t^Q \in \mathbb{R}^d$ (and masked $G_t \in \mathbb{R}^{d \times d_v}$, $h_t \in \mathbb{R}^d$)."

**Concrete Memory Costs:**

| Component | Shape | Memory (float32) |
|-----------|-------|------------------|
| $S_t^K$ | $d \times d$ | $4d^2$ bytes |
| $C_t^{QV}$ | $d \times d_v$ | $4d d_v$ bytes |
| $G_t$ | $d \times d_v$ | $4d d_v$ bytes |
| $m_t^Q, h_t$ | $d$ each | $8d$ bytes |
| **Total per head** | — | **$\approx 4d^2 + 8d d_v$ bytes** |

For typical Transformer dimensions ($d = 256$, $d_v = 256$, $h = 32$ heads):
- Per-head state: $\approx 4(256^2) + 8(256)(256) = 262,144 + 524,288 = 786,432$ bytes ≈ **768 KB**
- Total state (32 heads): **≈ 24.6 MB** per layer

**Comparison to First-Order Linear Attention:**
- First-order maintains only $C_t^{QV}$ and scalar denominator: $O(d d_v)$ per head
- HLA adds $S_t^K$ at $O(d^2)$, which dominates when $d \approx d_v$

**The Trade-off:** The $O(d^2)$ state is the **price of higher-order expressivity**. Readers must decide whether the additional modeling capacity justifies:
- Increased GPU memory consumption during inference
- Reduced batch sizes for long-context training
- Potential bandwidth bottlenecks when loading state across layers

**Mitigation Mentioned in Paper:** Section 5.2 notes that **multi-query attention** (sharing $K, V$ across heads) reduces state from $O(h d^2)$ to $O(d^2 + h d d_v)$:

> "With multi-query $(K, V)$ shared across heads, the key moment $S_t^K$ is shared and stored once per layer ($O(d^2)$), while $(C_t^{QV}, m_t^Q, G_t, h_t)$ remain per-head."

This is an **optimization, not the default**—readers must explicitly configure multi-query to realize these savings.

### 6.3 No Empirical Guidance on When Higher-Order Helps

**The Limitation:** The paper establishes that HLA is **algebraically more expressive** than first-order linear attention (Section 3, "Connection with linear attention"), but provides **zero empirical evidence** that this expressivity translates to better task performance.

**What We Know Theoretically:**
- Second-order HLA implements kernel $K_t(q, q') = q^\top S_t^K q'$ where $S_t^K = \sum_{i \leq t} k_i k_i^\top$
- First-order linear attention uses $K(q, q') = q^\top q'$ (identity metric)
- HLA's metric **adapts based on observed keys**, while first-order is fixed

**What We Don't Know Empirically:**

| Question | Status |
|----------|--------|
| Does second-order improve perplexity on language modeling? | **Not tested** |
| Which tasks benefit from higher-order interactions? | **Not identified** |
| Is third-order worth the additional state cost? | **Not evaluated** |
| Does HLA close the gap with standard attention on long-context tasks? | **Not measured** |
| How does symmetric HLA compare to asymmetric AHLA in practice? | **Not compared** |

**The Risk:** Higher-order does not automatically mean better. Possible scenarios:
- **Diminishing returns:** First-order may capture most useful dependencies; second-order adds noise
- **Task dependence:** Reasoning tasks may benefit more than retrieval tasks (or vice versa)
- **Overfitting risk:** More parameters in state could overfit on smaller datasets

**What Readers Should Do:** Before adopting HLA:
1. Run controlled comparisons against first-order linear attention on your target tasks
2. Measure whether the $O(d^2)$ state cost yields measurable accuracy gains
3. Consider starting with second-order before exploring third-order

### 6.4 Numerical Stability at Extreme Sequence Lengths

**The Limitation:** The paper mentions numerical stability mechanisms but does not analyze their effectiveness at very long sequences. Section 3 notes:

> "$\varepsilon > 0$ is a small constant added for numerical stability."

Section 5.1 mentions optional ridge regularization:

> "Effective $S$: $S_t^{\text{eff}} \leftarrow S_t + \lambda I$ ▷ optional ridge for stability"

**Potential Stability Issues:**

1. **Unbounded Growth Without Decay:** The prefix summaries $S_t^K = \sum_{i \leq t} k_i k_i^\top$ grow monotonically. Without decay ($\gamma = 1$), entries can become arbitrarily large over millions of tokens, causing:
   - Floating-point overflow
   - Loss of precision in subtraction operations (e.g., $S_t^K C_t^{QV} - G_t$)
   - Gradient instability during training

2. **Decay Factor is Fixed:** Section 4.2 treats $\gamma \in (0, 1)$ as "a fixed exponential decay." The paper does not discuss:
   - How to choose $\gamma$ (e.g., 0.99 vs. 0.999)
   - Whether $\gamma$ should be learned or tuned per task
   - How decay interacts with the correction terms $G_t, h_t$

3. **Subtraction of Large Similar Terms:** The masked output computes $q_t^\top (S_t^K C_t^{QV} - G_t)$. If both terms are large and similar, **catastrophic cancellation** can occur, losing significant digits.

**What's Missing:** The paper does not provide:
- Recommended $\varepsilon$ and $\lambda$ values
- Maximum sequence length tested for stability
- Behavior analysis as $t \to \infty$ with and without decay

**Practical Guidance:** Readers implementing HLA should:
- **Always use decay** ($\gamma &lt; 1$) for sequences beyond ~10K tokens
- Monitor summary magnitudes during training
- Consider adaptive $\varepsilon$ based on denominator scale
- Test stability at target sequence lengths before deployment

### 6.5 Implementation Complexity and Error Surface

**The Limitation:** HLA's algebraic derivations are **significantly more complex** than first-order linear attention, increasing the risk of implementation errors.

**Complexity Comparison:**

| Component | First-Order Linear Attention | Second-Order HLA |
|-----------|------------------------------|------------------|
| Core summaries | 2 ($\sum \phi(k)v^\top$, $\sum \phi(k)$) | 5 ($S^K, C^{QV}, m^Q, G, h$) |
| Update equations | Simple additions | Matrix-vector products with ordering constraints |
| Masking | Trivial (running sums) | Requires correction terms ($G, h$) |
| Scan operator | Additive monoid | Semidirect product with cross-terms |
| Third-order extension | N/A | 10+ summaries, not fully parallelizable |

**Error-Prone Areas:**

1. **Update Ordering:** Section 3.1 emphasizes using $(k k^\top) X = k (k^\top X)$ to avoid cubic cost. Implementing this incorrectly (forming $k k^\top$ first) increases per-token cost from $O(d^2)$ to $O(d^3)$.

2. **Masking Corrections:** The terms $G_t$ and $h_t$ must use **previous-step** summaries ($C_{t-1}^{QV}$, $m_{t-1}^Q$), not current-step. Off-by-one errors break causality.

3. **Scan Concatenation:** The semidirect product in Equation 4.1 has specific cross-term ordering ($S_B C_A$, not $S_A C_B$). Reversing this produces incorrect activations.

4. **Decay Application:** Section 4.2 shows decay applies differently to different components ($\rho_B S_A + S_B$, not $\gamma(S_A + S_B)$). Incorrect decay breaks associativity.

**What This Means:** The **verification burden** is on the implementer. Unlike standard attention (widely implemented and tested), HLA requires:
- Careful code review against the paper's equations
- Unit tests verifying serial vs. parallel equivalence
- Causality tests ensuring no future-token leakage

**Paper's Stance:** Section 1 acknowledges this focus:

> "We deliberately focus on algorithmic structure and implementation."

This is honest but shifts responsibility to readers to validate correctness in their own codebases.

---

### 6.6 Drop-in Replacement Claim Requires Verification

**The Limitation:** Section 5.2 states HLA is a "drop-in replacement" for Transformer attention, but this claim is **architectural, not empirical**:

> "HLA only replaces the standard attention sublayer in the transformer block, while the feed-forward sublayer and normalization sublayers remain unchanged."

**What This Guarantees:**
- HLA accepts the same inputs ($Q, K, V$) as standard attention
- HLA produces outputs of the same shape ($n \times d_v$)
- No changes needed to feed-forward networks or layer normalization

**What This Does NOT Guarantee:**
- **Hyperparameter transfer:** Optimal learning rates, warmup schedules, or dropout rates may differ
- **Convergence behavior:** Training dynamics may change due to different gradient flow
- **Final performance:** Accuracy may be lower, equal, or higher—untested in this paper
- **Positional encoding compatibility:** Some positional encodings may interact differently with HLA's recurrence structure

**The Risk:** A "drop-in" replacement that requires extensive hyperparameter retuning is not truly drop-in. Readers should expect:
- At least a small hyperparameter search when integrating HLA
- Potential need to adjust model depth or width to match baseline performance
- Monitoring for training instability not present with standard attention

### 6.7 Summary: Key Trade-offs

| Trade-off | Benefit | Cost |
|-----------|---------|------|
| **Second-order vs. First-order** | Higher expressivity (data-adaptive metric) | $O(d^2)$ state vs. $O(d d_v)$ |
| **Exact causality** | No future-token leakage (proven) | Requires correction terms ($G, h$) |
| **Parallel training** | GPU-efficient via scans | Complex semidirect product operator |
| **Third-order extension** | Even higher expressivity | No parallel training; 10+ summaries |
| **Unnormalized default** | Avoids length-dependent renormalization | May require tuning for scale stability |
| **Fixed decay $\gamma$** | Maintains associativity | No adaptation to sequence characteristics |

**Bottom Line for Practitioners:**

HLA is **theoretically sound but empirically unproven**. The algorithmic contributions are rigorous and well-documented, but the practical value depends on factors this paper does not address:

1. **Start with second-order HLA**—third-order lacks parallel training infrastructure
2. **Use multi-query attention** to reduce $O(h d^2)$ state to $O(d^2 + h d d_v)$
3. **Enable decay** ($\gamma &lt; 1$) for sequences beyond ~10K tokens
4. **Run controlled comparisons** against first-order linear attention on your tasks
5. **Verify implementation correctness** through serial vs. parallel equivalence tests

The paper provides an excellent **foundation** for efficient higher-order sequence modeling, but readers should treat it as a starting point for empirical validation rather than a turnkey solution.

## 7. Implications and Future Directions

This paper does not merely propose a new layer; it redefines the theoretical boundaries of efficient sequence modeling. By demonstrating that higher-order interactions can be computed in linear time with exact causality, HLA challenges the prevailing assumption that expressivity must be sacrificed for efficiency. This section explores how HLA shifts the research landscape, identifies concrete opportunities for follow-up work, outlines practical deployment scenarios, and provides guidance for integrating this mechanism into real-world systems.

### 7.1 Redefining the Efficiency-Expressivity Frontier

Historically, efficient attention mechanisms have faced a rigid trade-off: achieve $O(n)$ complexity by reducing expressivity (e.g., first-order linear attention) or maintain expressivity at $O(n^2)$ cost (standard attention). HLA disrupts this dichotomy by proving that **higher-order interactions do not inherently require quadratic complexity**.

**Theoretical Shift:**
Prior work often assumed that capturing second-order statistics (like $K^\top K$) necessitated either:
1.  **Materializing $n \times n$ matrices:** Incurring $O(n^2)$ memory (Standard Attention).
2.  **Matrix Inversion:** Incurring $O(d^3)$ cost per token (某些 Fast Weight methods, Section 8).

HLA demonstrates a third path: maintaining the second-moment matrix $S_t^K = \sum_{i \leq t} k_i k_i^\top$ directly as a streaming state. As established in Section 3, this allows the model to compute a data-adaptive kernel $K_t(q, q') = q^\top S_t^K q'$ without inversion or quadratic sequence dependence. This shifts the Pareto frontier of sequence modeling:
*   **Before:** Efficiency $\implies$ First-Order Approximation.
*   **After:** Efficiency $\implies$ Exact Higher-Order Streaming.

**Impact on Architecture Design:**
This result suggests that modern architectures (like Mamba or Linear Transformers) may be under-parameterized in their interaction terms. If second-order statistics can be maintained at $O(d^2)$ state cost (Section 5), future architectures might default to higher-order mixing rather than treating it as a luxury. The paper's explicit construction of third-order summaries (Section 7.1) further implies that this frontier can be pushed even higher, limited primarily by state memory rather than algorithmic impossibility.

### 7.2 Critical Follow-Up Research Opportunities

While the algorithmic foundations are rigorous, several high-impact research directions remain open. The paper explicitly identifies some of these, while others emerge from the limitations analyzed in Section 6.

#### 7.2.1 Empirical Validation and Benchmarking
The most immediate gap is empirical performance. As noted in Section 5.1, this paper provides **no accuracy tables, perplexity scores, or downstream task results**. Follow-up work must answer:
*   **Does Higher-Order Translate to Accuracy?** Does the theoretical expressivity of second-order HLA yield measurable perplexity improvements over first-order linear attention on standard benchmarks (e.g., WikiText, C4)?
*   **Task Specificity:** Do certain tasks benefit more from higher-order interactions? For example, reasoning tasks requiring multi-hop dependency tracking might benefit more than simple retrieval tasks.
*   **Convergence Dynamics:** Does the additional complexity in the gradient path (via the semidirect product scan) affect training stability or convergence speed compared to standard attention?

#### 7.2.2 Parallel Training for Third-Order HLA
Section 7.2 explicitly states: *"Designing a chunk-parallel scan operator that exactly reproduces serial activations requires additional segment-level summaries... we leave this composition to future work."*
This is a critical engineering bottleneck. Without parallel training, third-order HLA is restricted to research settings with short sequences or serial training hardware.
*   **Research Goal:** Derive the associative concatenation operator for third-order summaries ($G^{(1:3)}, h^{(1:3)}$).
*   **Challenge:** The number of cross-terms likely grows combinatorially. Identifying a compact representation that preserves associativity without exploding memory is an open algebraic problem.

#### 7.2.3 Adaptive Decay and Order Selection
Currently, the decay factor $\gamma \in (0, 1)$ is fixed (Section 4.2), and the order (second vs. third) is a static architectural choice.
*   **Learnable Decay:** Could $\gamma$ be learned per head or per layer to adaptively control the effective context window? This would align HLA with selective state spaces (e.g., Mamba) while retaining higher-order statistics.
*   **Dynamic Order:** Could a model switch between first and second-order updates dynamically based on input complexity? This would allow $O(d d_v)$ compute for simple tokens and $O(d^2)$ for complex ones, optimizing the efficiency-expressivity trade-off per token.

#### 7.2.4 Hybrid Architectures
HLA is designed as a drop-in replacement for attention sublayers (Section 5.2). This enables hybrid designs:
*   **HLA + SSM:** Combine HLA's data-dependent mixing with SSM's structured state evolution for robust long-range modeling.
*   **HLA + Standard Attention:** Use standard attention for short local windows (where $O(n^2)$ is cheap) and HLA for global context, creating a hierarchical memory system.

### 7.3 Practical Deployment Scenarios

HLA is not a universal replacement for all attention mechanisms. Its value proposition is strongest in specific deployment contexts where its constraints align with system requirements.

#### 7.3.1 Long-Context Inference (Streaming)
**Best Fit:** Applications requiring context windows exceeding 100K tokens where standard attention is memory-prohibitive.
*   **Why:** HLA maintains $O(1)$ state per token independent of sequence length $n$ (Section 1).
*   **Use Cases:**
    *   **Document Analysis:** Processing full legal contracts or technical manuals where dependencies span thousands of tokens.
    *   **Conversation History:** Maintaining coherent dialogue over many turns without truncation.
    *   **Code Generation:** Tracking variable definitions and usage across entire files or repositories.

#### 7.3.2 Memory-Constrained Edge Devices
**Best Fit:** Deployment on hardware with limited SRAM or bandwidth.
*   **Why:** The state size is fixed at $O(d^2)$ per head (Section 5). With multi-query attention (sharing $K, V$ across heads), this reduces to $O(d^2 + h d d_v)$ total memory per layer (Section 5.2).
*   **Constraint:** Ensure $d$ is not excessively large. For $d=256$, $S_t^K$ requires ~256 KB per head (float32). For $d=1024$, this grows to ~4 MB per head.
*   **Optimization:** Use the packed symmetric layout mentioned in Section 5.2 to store only $\frac{1}{2}d(d+1)$ entries, reducing bandwidth by ~50%.

#### 7.3.3 When to Prefer Alternatives
HLA is not always the optimal choice. Practitioners should consider alternatives in these scenarios:
*   **Short Context (&lt; 4K tokens):** Standard attention is highly optimized on modern GPUs and may offer better throughput due to massive parallelism, despite $O(n^2)$ complexity.
*   **Strict Memory Budgets:** If $O(d^2)$ state is too large (e.g., very wide models), first-order linear attention ($O(d d_v)$ state) remains more memory-efficient.
*   **Bidirectional Tasks:** HLA is strictly causal (Section 3.1). For encoder-only tasks (e.g., classification, bidirectional embedding), standard attention or non-causal linear variants are more appropriate.

### 7.4 Integration and Reproduction Guidance

For researchers and engineers aiming to implement HLA, the following checklist ensures correctness and leverages the paper's theoretical guarantees.

#### 7.4.1 Implementation Checklist
1.  **Verify Serial-Parallel Equivalence:** Use Theorem 4.1 as a unit test. Implement both the serial recurrence (Algorithm 1, lines 3-17) and the parallel scan (Section 4.2). Compare outputs bitwise (within floating-point tolerance). If they diverge, the associative operator $\oplus$ is implemented incorrectly.
2.  **Enforce Causality Corrections:** Ensure the correction terms $G_t$ and $h_t$ use **previous-step** summaries ($C_{t-1}^{QV}$, $m_{t-1}^Q$), not current-step. As shown in Section 3.1, using current-step summaries breaks strict causality.
3.  **Optimize Matrix Operations:** Implement updates as matrix-vector products (e.g., $k_t (k_t^\top C_{t-1}^{QV})$) rather than forming full outer products first. This maintains $O(d^2)$ cost instead of $O(d^3)$ (Section 3.1, "Online updates").
4.  **Enable Decay for Stability:** For sequences beyond ~10K tokens, set decay $\gamma &lt; 1$ (e.g., 0.99 or 0.999). Without decay, prefix summaries may grow unbounded, leading to numerical instability (Section 4.3).
5.  **Use Multi-Query Attention:** To mitigate $O(d^2)$ state cost, share $K, V$ across heads where possible. This reduces total state memory from $O(h d^2)$ to $O(d^2 + h d d_v)$ (Section 5.2).

#### 7.4.2 Hyperparameter Recommendations
Since the paper does not provide empirical tuning guidelines (Section 5.1), start with these baseline settings derived from related linear attention work:
*   **Decay $\gamma$:** Start with $\gamma = 0.99$. Tune based on perplexity vs. context length.
*   **Stability $\varepsilon$:** Use $\varepsilon = 10^{-6}$ for normalization denominators (Equation 3.4).
*   **Ridge $\lambda$:** If $S_t^K$ becomes ill-conditioned, add $\lambda I$ with $\lambda \in [10^{-4}, 10^{-2}]$ (Algorithm 1, line 7).
*   **Chunk Size $w$:** Set based on GPU SRAM. Larger chunks improve parallelism but increase temporary memory during scans.

#### 7.4.3 Debugging Common Pitfalls
*   **Gradient Explosion:** If gradients explode during training, check the decay factor $\gamma$. Ensure the backward scan uses the adjoint operator $\oplus^*$ correctly (Section 4.2, "Backward for gradients").
*   **Future-Token Leakage:** Test with a simple copying task where token $t$ must predict token $t-1$. If accuracy exceeds chance on predicting token $t+1$, causality is broken (likely due to incorrect $G_t$ updates).
*   **Numerical Overflow:** Monitor the norm of $S_t^K$ during long sequences. If it grows monotonically without decay, switch to $\gamma &lt; 1$.

### 7.5 Summary: The Path Forward

Higher-order Linear Attention represents a maturation of efficient sequence modeling. It moves the field from "how do we approximate attention?" to "how do we express richer interactions efficiently?" The theoretical guarantees provided in this paper—exact causality (Theorem 3.1), parallel training equivalence (Theorem 4.1), and generalizable algebra (Section 7)—provide a solid foundation for this next phase.

However, the transition from theory to practice requires the community to undertake the empirical work this paper deliberately omits. The algorithmic structure is sound; the next step is to validate whether the additional expressivity justifies the $O(d^2)$ state cost in real-world applications. For practitioners, HLA offers a powerful tool for long-context streaming, provided they adhere to the implementation constraints regarding causality, stability, and memory management outlined above.
