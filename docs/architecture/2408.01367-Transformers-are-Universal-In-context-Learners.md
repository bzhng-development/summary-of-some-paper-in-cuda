## 1. Executive Summary

This paper proves that deep Transformers are universal approximators of continuous in-context mappings, capable of operating on an arbitrary (even infinite) number of tokens with a fixed embedding dimension and a fixed number of heads proportional to the output dimension $d'$ (specifically $H \leq d'$ and $d_{head}=1$). By modeling token contexts as probability distributions and utilizing the Wasserstein distance to define continuity, the authors establish this universality for both unmasked and masked causal attentions in Theorems 1 and 2. This result matters because it demonstrates that Transformers can handle arbitrarily long contexts without increasing model width, overcoming a key limitation in prior expressivity analyses that required embedding dimensions to grow with the number of tokens or precision.

## 2. Context and Motivation

### The Fundamental Question: Can Transformers Handle Arbitrary Context Lengths?

At its core, this paper addresses a deceptively simple question that has profound implications for both theory and practice: **Can a transformer with fixed architecture parameters approximate any continuous in-context mapping, regardless of how many tokens appear in the context?** 

To understand why this matters, consider how transformers operate in practice. When you prompt a language model with a document, or feed a vision transformer an image split into patches, the model must process a *context* — a set of previously seen tokens — to make predictions. The size of this context can vary dramatically: a short sentence might have 10 tokens, while a long document could have 10,000 or more. The theoretical question is whether a single transformer architecture can handle *all* these cases without needing to grow wider or deeper as the context expands.

This is not merely an academic concern. In real-world applications:
- **Long-context NLP**: Models must process documents, books, or even entire codebases without architectural changes
- **Vision transformers**: Image resolution determines patch count, which can vary significantly
- **Time series**: Sequence length depends on the observation window, which may be arbitrary

If transformers required their embedding dimension to scale with context length, this would impose severe memory and computational constraints, fundamentally limiting their practical utility for long sequences.

### The Gap in Prior Universality Results

The literature on neural network universality is extensive, but existing results for transformers contain a critical limitation that this paper identifies and resolves.

**Classical universality results** established that Multilayer Perceptrons (MLPs) with just two layers can approximate any continuous function to arbitrary precision [10, 19]. These results were later extended to various architectures including Convolutional Neural Networks (CNNs) with weight sharing [45], ResNets with skip connections [9, 37], and graph neural networks [25, 21, 41]. However, these classical results operate on *fixed-dimensional* input spaces — they do not address what happens when the *number* of input elements (tokens) can grow arbitrarily.

**Transformer-specific universality work** has made progress but with a significant caveat. The most detailed prior account, [43], proves transformer universality but requires that **the embedding dimension grows with the number of tokens**. This is explicitly stated in the Introduction:

> "[43] provides, to the best of our knowledge, the most detailed account of the universality of transformers. The authors rely on shallow transformers with only 2 heads but require that the transformers operate over an embedding dimension which grows with the number of tokens."

This requirement is problematic for two reasons:

1. **Practical infeasibility**: If you need to double your context length, you would need to double your embedding dimension, which quadratically increases memory requirements for attention matrices (since attention scales as $O(n^2 \cdot d)$ where $n$ is token count and $d$ is dimension).

2. **Theoretical incompleteness**: It does not explain why transformers in practice *do* handle variable context lengths with fixed dimensions. Real-world transformers use fixed embedding sizes (e.g., 768, 1024, 4096 dimensions) regardless of whether they process 100 or 100,000 tokens.

Other related works face similar limitations:
- **Sumformer** [3] and **stochastic deep networks** [11] also require embedding dimension growth with token count
- Work on **neural operators** [22, 15] extends universality to infinite-dimensional function spaces but does not directly address the transformer architecture's specific attention mechanism
- Research on transformers as **formal systems** [8, 28, 36, 14] studies expressivity on discrete token sets but does not handle the continuous, measure-theoretic limit

### Why the Measure-Theoretic Approach Matters

This paper's key innovation is reframing how we think about token contexts. Instead of viewing a context as a finite set of $n$ tokens $X = (x_i)_{i=1}^n \in \mathbb{R}^{d \times n}$, the authors represent it as a **probability distribution** $\mu \in \mathcal{P}(\mathbb{R}^d)$ over the token embedding space. A finite set of tokens becomes a discrete empirical measure:

$$\mu = \frac{1}{n} \sum_{i=1}^n \delta_{x_i} \in \mathcal{P}(\mathbb{R}^{d_{in}})$$

where $\delta_{x_i}$ is a Dirac mass at token $x_i$.

This reformulation, building on prior work [40, 33, 7], provides three critical advantages:

1. **Unified treatment of finite and infinite contexts**: The same mathematical object (a probability measure) describes both 100 tokens and a continuous distribution over tokens. This enables analysis of the "mean field" limit where $n \to \infty$.

2. **Proper notion of smoothness**: By using the **Wasserstein distance** $W_p$ between probability measures, the paper defines continuity for in-context mappings in a way that makes sense regardless of token count. Two contexts are "close" if their token distributions are close in optimal transport distance, not if they have the same number of tokens.

3. **Permutation equivariance**: In the unmasked setting, attention is permutation-equivariant (reordering tokens doesn't change the output). Probability measures naturally encode this symmetry since $\frac{1}{n}\sum_{i=1}^n \delta_{x_i} = \frac{1}{n}\sum_{i=1}^n \delta_{x_{\sigma(i)}}$ for any permutation $\sigma$.

### The Causal (Masked) Challenge

The paper distinguishes between two attention settings, each with distinct mathematical challenges:

**Unmasked attention** (used in vision transformers and encoder-only models) allows each token to attend to all other tokens. This setting is permutation-equivariant, making the measure-theoretic formulation natural.

**Masked causal attention** (used in language modeling and time series) restricts each token to attend only to preceding tokens. This breaks permutation equivariance and introduces a temporal structure. As the Introduction notes:

> "The masked setting requires some stronger regularity hypothesis on the contexts, namely that they are Wasserstein-Lipschitz with respect to time, which is needed to cope with the constraint of causality in the relevant mappings."

To handle this, the paper introduces a **space-time lifting** where tokens are pairs $(x_i, t_i)$ with $t_i \in [0, 1]$ representing temporal position. The masked attention then integrates only over tokens with $t_j \leq t_i$. This requires additional technical conditions (Lipschitz contexts, identifiability) that do not appear in the unmasked case.

### Positioning Relative to Existing Work

The following table summarizes how this paper's contributions compare to prior universality results:

| Work | Architecture | Embedding Dimension | Token Count | Setting |
|------|-------------|---------------------|-------------|---------|
| [43] | Shallow (2 heads) | Grows with tokens | Arbitrary | Unmasked |
| [3] Sumformer | Specialized | Grows with tokens | Arbitrary | Unmasked |
| [11] Stochastic | Deep | Grows with tokens | Arbitrary | General |
| **This work** | **Deep** | **Fixed** | **Arbitrary (even infinite)** | **Both** |

The key differentiator is the **fixed embedding dimension**. Theorem 1 (unmasked) and Theorem 2 (masked) both state that for any approximation precision $\varepsilon > 0$, there exists a transformer with:
- $d_{head}(\theta_\ell) = k(\theta_\ell) = 1$ (head dimension equals key/query dimension equals 1)
- $H(\theta_\ell) \leq d'$ (number of heads proportional to output dimension, not token count)
- $d_{in}(\theta_\ell) \leq d + 3d'$ (input dimension bounded independently of $\varepsilon$ and $n$)

This is a fundamentally stronger result than prior work because it shows that **depth compensates for width**. Rather than requiring wider layers to handle more tokens, the transformer can use more layers while keeping dimensions fixed.

### Why This Theoretical Result Matters

Beyond mathematical elegance, this universality result has concrete implications:

1. **Architecture design**: It suggests that for long-context applications, increasing depth may be more effective than increasing width, since width requirements do not scale with context length.

2. **Generalization guarantees**: If transformers can approximate any continuous in-context mapping uniformly over compact token domains, this provides theoretical grounding for why pre-trained models can adapt to contexts of varying lengths without retraining.

3. **Bridging theory and practice**: The result explains why empirical observations show transformers handling variable-length sequences with fixed dimensions — it is not merely a practical hack but a theoretically sound property of the architecture.

However, the authors are careful to note limitations. The proof is **not quantitative** — it establishes existence of approximating transformers but does not provide explicit bounds on how depth $L$ scales with precision $\varepsilon$. Additionally, the number of heads grows proportionally to output dimension $d'$, which the authors acknowledge as an area for improvement:

> "A weakness is that we have no explicit control over the dependency of the number of MLP parameters $\xi_\ell$ on $\varepsilon$... Obtaining a better balance between these two parameters is an interesting problem."

These caveats are important: while the paper establishes *that* transformers are universal in-context learners with fixed dimensions, it does not fully characterize *how efficiently* they achieve this approximation. This leaves open questions about the depth-precision tradeoff that are relevant for practical model design.

## 3. Technical Approach

### 3.1 Reader orientation (approachable technical breakdown)

This paper constructs a mathematical proof system showing that deep transformers can approximate any continuous function that maps token contexts to token predictions, regardless of how many tokens appear in the context. The solution works by reframing token sequences as probability distributions rather than finite lists, then proving that transformer layers can build up arbitrary continuous mappings through composition, much like how classical neural network universality proofs show MLPs can approximate functions through layered composition.

### 3.2 Big-picture architecture (diagram in words)

The technical framework consists of four interconnected components that work together to establish universality. First, the **Measure-Theoretic Token Representation** converts finite token sets into probability distributions, enabling analysis of arbitrary token counts including infinite limits. Second, the **In-Context Mapping Formalism** rewrites standard attention equations as functions that take both a context distribution and a query token as inputs, making the dependency structure explicit. Third, the **Layer Composition Algebra** defines how multiple transformer layers combine their in-context mappings through a specialized composition operator that tracks how contexts evolve through the network depth. Fourth, the **Approximation Machinery** uses the Stone-Weierstrass theorem to show that elementary attention building blocks can be combined to approximate any continuous target mapping, with separate treatments for unmasked (bidirectional) and masked (causal) attention patterns. Information flows from the measure-theoretic representation through the in-context mapping definition, then through the composition algebra, and finally through the approximation proof to establish the universality theorems.

### 3.3 Roadmap for the deep dive

- We begin with the measure-theoretic reformulation because it is the foundational insight that enables all subsequent analysis — without representing tokens as distributions, the arbitrary token count result would not be possible.
- We then explain the in-context mapping formalism for both unmasked and masked settings, since this rewriting of attention is what makes the mathematical analysis tractable and reveals the spatial mapping structure.
- We detail the composition algebra next because understanding how layers combine is essential for seeing how deep transformers build complex functions from simple attention operations.
- We present the unmasked universality proof (Theorem 1) before the masked case (Theorem 2) because the masked setting requires additional technical conditions that build on the unmasked foundation.
- We conclude with the architectural constraints and limitations because these specify exactly what the theorems guarantee and what remains open for future work.

### 3.4 Detailed, sentence-based technical breakdown

This is a theoretical computer science and machine learning paper that proves transformers are universal approximators of continuous in-context mappings using measure theory and functional analysis. The core idea is that by representing token contexts as probability distributions rather than finite sets, and by defining continuity through the Wasserstein distance between these distributions, one can prove that deep transformers approximate any continuous mapping uniformly over compact token domains without requiring the embedding dimension to grow with token count or precision.

**The Measure-Theoretic Token Representation**

The paper's first technical innovation is replacing the classical view of tokens as a finite matrix $X \in \mathbb{R}^{d_{in} \times n}$ with $n$ tokens of dimension $d_{in}$ with a probability measure $\mu \in \mathcal{P}(\mathbb{R}^{d_{in}})$ over the token embedding space. A finite set of $n$ tokens $(x_i)_{i=1}^n$ is encoded as a discrete empirical measure using the formula $\mu = \frac{1}{n} \sum_{i=1}^n \delta_{x_i}$, where $\delta_{x_i}$ denotes a Dirac mass (a point mass) at token $x_i$. This encoding is not merely notational convenience — it enables three critical capabilities. First, it provides a unified mathematical object that describes both finite token sets and continuous distributions, allowing analysis of the "mean field" limit where $n \to \infty$. Second, it enables a proper notion of smoothness through the Wasserstein distance $W_p$ between probability measures, which quantifies how "close" two contexts are regardless of whether they have the same number of tokens. Third, it naturally encodes permutation equivariance in the unmasked setting, since $\frac{1}{n}\sum_{i=1}^n \delta_{x_i} = \frac{1}{n}\sum_{i=1}^n \delta_{x_{\sigma(i)}}$ for any permutation $\sigma$, matching the symmetry properties of self-attention.

The paper works with $\Omega \subset \mathbb{R}^d$, a compact domain for token embeddings, and $\mathcal{P}(\Omega)$ denotes the space of probability measures on $\Omega$. The weak* topology on $\mathcal{P}(\Omega)$ defines convergence: a sequence $\mu_k$ converges to $\mu$ (written $\mu_k \rightharpoonup^* \mu$) if and only if $\int f(x) \, d\mu_k(x) \to \int f(x) \, d\mu(x)$ for all continuous functions $f \in C(\Omega)$. This topology can be metrized using the Wasserstein distance $W_p$ for $1 \leq p < +\infty$, defined as $W_p(\mu, \nu)^p := \min_{\pi \in \mathcal{P}(\Omega^2)} \left\{ \int \|x - y\|^p \, d\pi(x, y) : \pi_1 = \mu, \pi_2 = \nu \right\}$, where $\pi_i = (P_i)_\sharp \pi$ are the marginals of the coupling $\pi$. The key property is that $\mu_k \rightharpoonup^* \mu$ if and only if $W_p(\mu_k, \mu) \to 0$, which allows the paper to use optimal transport distance to define continuity for in-context mappings.

**The In-Context Mapping Formalism**

The paper's second technical innovation is rewriting the standard transformer attention equations as "in-context mappings" that make explicit how each output token depends on both the query token and the full context. For the classical unmasked setting with $n$ tokens $X = (x_i)_{i=1}^n \in \mathbb{R}^{d_{in} \times n}$, a single attention head maps these tokens through the equation $\text{Att}_\theta(X) := V X \, \text{SoftMax}(X^\top Q^\top K X / \sqrt{k}) \in \mathbb{R}^{d_{head} \times n}$, where the parameters are $\theta := (K, Q, V) \in \mathbb{R}^{k \times d_{in}} \times \mathbb{R}^{k \times d_{in}} \times \mathbb{R}^{d_{head} \times d_{in}}$ representing the Key, Query, and Value matrices, and $k$ is the key/query dimension. The SoftMax function operates row-wise with masking: for $Z \in \mathbb{R}^{n \times n}$, $\text{SoftMax}(Z)_{i,j} := \frac{M_{i,j} e^{Z_{i,j}}}{\sum_{\ell=1}^n M_{i,\ell} e^{Z_{i,\ell}}}$, where $M_{i,j} = 1$ for unmasked (bidirectional) attention and $M_{i,j} = \mathbf{1}_{j \leq i}$ for masked (causal) attention.

Multiple attention heads are combined through multi-head attention with a residual connection: $\text{MAtt}_\theta(X) := X + \sum_{h=1}^H W^h \text{Att}_{\theta_h}(X)$, where $W^h \in \mathbb{R}^{d_{in} \times d_{head}}$ are output projection matrices and $\theta_h := (K^h, Q^h, V^h)$ are the parameters for head $h$. The paper introduces notation for the various dimensions: $d_{in}(\theta)$ for token dimension, $d_{head}(\theta)$ for head dimension, $k(\theta)$ for key/query dimension, and $H(\theta)$ for the number of heads.

The critical reformulation comes in Equation (2), which rewrites the multi-head attention mapping $X \mapsto \text{MAtt}_\theta(X)$ as the application of an "in-context" function $G_\theta(X, \cdot)$ to each token individually: $x_i \mapsto G_\theta(X, x_i)$, so that $\text{MAtt}_\theta(X) = (G_\theta(X, x_i))_{i=1}^n$. For the unmasked case, the in-context mapping is defined for all $(X, x) \in \mathbb{R}^{d_{in} \times n} \times \mathbb{R}^{d_{in}}$ as:

$$G_\theta(X, x) := x + \sum_{h=1}^H W^h \sum_{j=1}^n \frac{\exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h x_j \rangle\right)}{\sum_{\ell=1}^n \exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h x_\ell \rangle\right)} V^h x_j$$

This reformulation highlights that transformers define spatial mappings where each token's transformation depends parametrically on its interactions with all other tokens in the context. The terminology "in-context" refers to the fact that $G_\theta(X, \cdot)$ depends on the tokens $X$ themselves, making it a parametric map that is modified for each token depending on its interactions with other tokens.

For the masked setting, due to the lack of permutation equivariance, the mapping must also track the index $i$ of each token. Equation (3) defines the masked in-context mapping as $G_\theta(X, x, i) := x + \sum_{h=1}^H W^h \sum_{j=1}^i \frac{\exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h x_j \rangle\right)}{\sum_{\ell=1}^i \exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h x_\ell \rangle\right)} V^h x_j$, where the summation only goes up to $i$ due to the causal mask $M_{i,j} = \mathbf{1}_{j \leq i}$.

**Measure-Theoretic In-Context Mappings**

The paper generalizes these finite-token definitions to arbitrary (possibly infinite) numbers of tokens by replacing sums with integrals over probability measures. For the unmasked setting, Equation (9) defines the measure-theoretic in-context mapping for all $(\mu, x) \in \mathcal{P}(\mathbb{R}^{d_{in}}) \times \mathbb{R}^{d_{in}}$ as:

$$\Gamma_\theta(\mu, x) := x + \sum_{h=1}^H W^h \int \frac{\exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h y \rangle\right)}{\int \exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h z \rangle\right) d\mu(z)} V^h y \, d\mu(y)$$

The discrete case is contained in this general definition in the sense that for any finite token set $X = (x_i)_{i=1}^n$, we have $G_\theta(X, x) = \Gamma_\theta\left(\frac{1}{n}\sum_{i=1}^n \delta_{x_i}, x\right)$. The paper uses a slight abuse of notation, writing $\Gamma_\theta(\mu, x) = \Gamma_\theta(\mu)(x)$, so that $\Gamma_\theta(\mu): \mathbb{R}^{d_{in}} \to \mathbb{R}^{d_{out}}$ defines a map between Euclidean spaces that depends on the context measure $\mu$.

Using this formulation, the attention map $X \mapsto \text{MAtt}_\theta(X)$ can be rewritten as displacing the tokens' positions, which corresponds to applying a push-forward to the measure. The push-forward operator $T_\sharp$ transforms a measure $\mu \in \mathcal{P}(\mathbb{R}^{d_{in}})$ through a map $T: \mathbb{R}^{d_{in}} \to \mathbb{R}^{d_{out}}$ to produce $T_\sharp \mu \in \mathcal{P}(\mathbb{R}^{d_{out}})$, defined by $\forall A \subset \mathbb{R}^{d_{out}}, \, T_\sharp \mu(A) := \mu(T^{-1}(A))$. For discrete measures, this operates by simply displacing their supports: $T_\sharp \left(\frac{1}{n}\sum_{i=1}^n \delta_{x_i}\right) := \frac{1}{n}\sum_{i=1}^n \delta_{T(x_i)}$. The attention map thus transforms the context measure as $\mu \in \mathcal{P}(\mathbb{R}^{d_{in}}) \mapsto \Gamma_\theta(\mu)_\sharp \mu \in \mathcal{P}(\mathbb{R}^{d_{out}})$.

For the masked setting, the paper introduces a **space-time lifting** to restore the ability to write in-context mappings using measures. Since masked attention is not permutation equivariant, the input tokens are represented as pairs $\{(x_i, t_i)\}_{i=1}^n$ where $t_i \in [0, 1]$ represents temporal position. For instance, assuming an upper bound $N$ on the number of tokens, one can use $t_i = i/N$, but it is also possible to assume that the $t_i$ are positioned arbitrarily in $[0, 1]$, which enables considering an arbitrarily large (and even infinite) number of tokens. The context is encoded as a space-time measure $\mu \in \mathcal{P}(\mathbb{R}^{d_{in}} \times [0, 1])$.

Equation (12) defines the masked measure-theoretic in-context map for all $(x, t) \in \mathbb{R}^{d_{in}} \times [0, 1]$ as:

$$\Gamma_\theta(\mu, x, t) := x + \sum_{h=1}^H W^h \int \frac{\exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h y \rangle\right) \mathbf{1}_{[0,t]}(r)}{\int \exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h z \rangle\right) \mathbf{1}_{[0,t]}(s) \, d\mu(z, s)} V^h y \, d\mu(y, r)$$

where $\mathbf{1}_{[0,t]}(s)$ is a masking function that equals 1 if $0 \leq s \leq t$ and 0 otherwise. For a finite number of tokens using a discrete measure $\mu = \frac{1}{n}\sum_{i=1}^n \delta_{(x_i, i/n)}$, one retrieves the initial definition in Equation (3) for the masked case.

**The Layer Composition Algebra**

A transformer (ignoring normalization layers) is defined as a composition of $L$ attention layers and Multi-Layer Perceptrons (MLPs): $\text{MLP}_{\xi_L} \circ \text{MAtt}_{\theta_L} \circ \cdots \circ \text{MLP}_{\xi_1} \circ \text{MAtt}_{\theta_1}$. The MLP functions process each token independently from one another: $\text{MLP}_\xi(X) = (F_\xi(x_i))_{i=1}^n$, meaning they are "context-free" mappings where $F_\xi(X, x) = F_\xi(x)$ does not depend on the context $X$, while the attention maps $G_\theta(X, \cdot)$ depend on the context.

On the level of in-context mappings, the composition of layers induces a new "in-context" composition rule, denoted by $\diamond$. For the unmasked case, Equation (5) defines $(G_2 \diamond G_1)(X, x) := G_2(X_1, G_1(X, x))$ where $X_1 := (G_1(X, x_i))_{i=1}^n$. For the masked case, Equation (6) defines $(G_2 \diamond G_1)(X, x, i) := G_2(X_1, G_1(X, x, i), i)$ where $X_1 := (G_1(X, x_i, i))_{i=1}^n$. This rule can be applied whether $G_1(X, \cdot)$ or $G_2(X, \cdot)$ depends on the context $X$ or not (such as for the $F_\xi$ mappings). Using this rule, the transformer's definition translates into a composition of in-context and context-free maps: $F_{\xi_L} \diamond G_{\theta_L} \diamond \cdots \diamond F_{\xi_1} \diamond G_{\theta_1}$.

The measure-theoretic setting generalizes this composition rule. For the unmasked case, Equation (10) defines $(\Gamma_2 \diamond \Gamma_1)(\mu, x) := \Gamma_2(\mu_1, \Gamma_1(\mu, x))$ where $\mu_1 := \Gamma_1(\mu)_\sharp \mu$, i.e., $(\Gamma_2 \diamond \Gamma_1)(\mu) = \Gamma_2(\mu_1) \circ \Gamma_1(\mu)$. Transformers operating over an arbitrary (possibly infinite) number of tokens are then obtained by replacing the original definition with $F_{\xi_L} \diamond \Gamma_{\theta_L} \diamond \cdots \diamond F_{\xi_1} \diamond \Gamma_{\theta_1}$. When restricted to finite discrete empirical measures, the finite and measure-theoretic definitions coincide.

For the masked setting, the composition rule is similar except that the time position of the token is kept unchanged while the push-forward acts in space. Equation (13) defines $(\Gamma_2 \diamond \Gamma_1)(\mu, x, t) := \Gamma_2(\mu_1, \Gamma_1(\mu, x, t), t)$ where $\mu_1 := (\Gamma_1(\mu), \text{Id}_{\mathbb{R}})_\sharp \mu$. Here, $(\Gamma_1(\mu), \text{Id}_{\mathbb{R}}): (x, t) \in \mathbb{R}^{d_{in}+1} \mapsto (\Gamma_1(\mu)(x, t), t) \in \mathbb{R}^{d_{in}+1}$, meaning the time coordinate is preserved while the spatial embedding is transformed.

**The Unmasked Universality Proof (Theorem 1)**

Theorem 1 states the main universality result for unmasked in-context mappings. Let $\Omega \subset \mathbb{R}^d$ be a compact set and let $\Lambda^\star: \mathcal{P}(\Omega) \times \Omega \to \mathbb{R}^{d'}$ be continuous, where $\mathcal{P}(\Omega)$ is endowed with the weak* topology. Then for all $\varepsilon > 0$, there exist $L$ layers and parameters $(\theta_\ell, \xi_\ell)_{\ell=1}^L$ such that for all $(\mu, x) \in \mathcal{P}(\Omega) \times \Omega$:

$$|F_{\xi_L} \diamond \Gamma_{\theta_L} \diamond \cdots \diamond F_{\xi_1} \diamond \Gamma_{\theta_1}(\mu, x) - \Lambda^\star(\mu, x)| \leq \varepsilon$$

with architectural constraints $d_{in}(\theta_\ell) \leq d + 3d'$, $d_{head}(\theta_\ell) = k(\theta_\ell) = 1$, and $H(\theta_\ell) \leq d'$.

The two strengths of this result are: (i) the approximating architecture performs the approximation independently of $n$ (it even works for an infinite number of tokens), and (ii) the number of heads and the embedding dimension do not depend on $\varepsilon$. A weakness is that there is no explicit control over the dependency of the number of MLP parameters $\xi_\ell$ on $\varepsilon$. Another limitation is that the number of heads grows proportionally to the output dimension while each head only outputs a scalar $d_{head}(\theta_\ell) = 1$.

The proof proceeds through three main steps. First, the authors define "elementary" in-context mappings that map $(x, \mu)$ to a real variable. Equation (14) defines $\gamma_\lambda(\mu, x) := \langle x, a \rangle + b + \int \frac{e^{c(\langle x, a \rangle + b)} (\langle y, a \rangle + b)}{\int e^{c(\langle x, a \rangle + b)} (\langle z, a \rangle + b) \, d\mu(z)} v (\langle a, y \rangle + b) \, d\mu(y)$, where $\lambda := (a, b, c, v) \in \mathbb{R}^d \times \mathbb{R} \times \mathbb{R} \times \mathbb{R}$. These elementary mappings are built by composing an affine scalar-valued MLP with a single-head attention (with skip connection) as in Equation (9), operating in 1-D. Specifically, defining $F_\xi(x) = \langle a, x \rangle + b \in \mathbb{R}$ as an affine MLP where $\xi = (a, b)$, we have $\gamma_\lambda(\mu, x) = (\Gamma_\theta \diamond F_\xi)(\mu, x)$, where $\theta = (k, q, v) \in \mathbb{R}^3$ (recalling that this attention operates in 1-D), and we let $c = qk$.

Second, the authors define $\mathcal{A}$, the algebra spanned by these elementary functions: $\mathcal{A} := \left\{ \mathcal{P}(\Omega) \times \Omega \ni (\mu, x) \mapsto \sum_{n=1}^N \gamma_{\lambda_{1,n}}(\mu, x) \odot \cdots \odot \gamma_{\lambda_{T,n}}(\mu, x) \in \mathbb{R} : N, T \in \mathbb{N} \right\}$, where $\odot$ denotes component-wise multiplication. Elements of this algebra are sums of products of elementary functions, which are often referred to as "cylindrical functions". Proposition 1 states that $\mathcal{A}$ is dense in the space of (weak* $\times \, \ell_2$)-continuous functions from $\mathcal{P}(\Omega) \times \Omega$ to $\mathbb{R}$.

The proof of Proposition 1 applies the Stone-Weierstrass theorem, which requires checking three key hypotheses. First, the functions $\gamma_\lambda$ are continuous because the denominator $\int e^{c(\langle x, a \rangle + b)} (\langle z, a \rangle + b) \, d\mu(z)$ in the elementary mapping is not always zero for any $\mu \in \mathcal{P}(\Omega)$ and $x \in \Omega$. Second, when setting $a = 0$, $b = v = 1$, one has that $\gamma_\lambda(\mu, x) = 1$ is the constant function. Third, the set $(\gamma_\lambda)_\lambda$ separates points, which is more challenging. To show this, the authors need to prove that if $\forall \lambda, \gamma_\lambda(\mu, x) = \gamma_\lambda(\mu', x')$, then $(\mu, x) = (\mu', x')$. First setting $v = 0$, this implies that $\langle x, a \rangle = \langle x', a \rangle$ for all $a \in \mathbb{R}^d$ and hence $x = x'$. Then, setting $b = 1 - \langle a, x \rangle$, the equality reads $L(\mu)(a, c) = L(\mu')(a, c)$ where the authors define a generalized Laplace-like transform $L(\mu)(a, c) := \int \frac{e^{c \langle a, y \rangle} \langle a, y \rangle}{\int e^{c \langle a, z \rangle} \, d\mu(z)} \, d\mu(y)$. Lemma 1 states that the map $\mu \mapsto L(\mu)$ defined in Equation (16) is injective, which allows the authors to conclude that $\mu = \mu'$.

Third, to approximate vector-valued in-context mappings, the authors use the previous algebra of cylindrical functions along each dimension. Lemma 2 states that for any $\varepsilon > 0$, there exist $T, N \in \mathbb{N}$ and $(\tilde{\theta}_{t,n}, \tilde{\xi}_{t,n})_{t \in [T], n \in [N]}$ such that for all $(\mu, x) \in \mathcal{P}(\Omega) \times \Omega$: $|G(\mu, x) - \Lambda^\star(\mu, x)| \leq \varepsilon$, where $G(\mu, x) := \sum_{n=1}^N (\Gamma_{\tilde{\theta}_{1,n}} \diamond F_{\tilde{\xi}_{1,n}})(\mu, x) \odot \cdots \odot (\Gamma_{\tilde{\theta}_{T,n}} \diamond F_{\tilde{\xi}_{T,n}})(\mu, x)$, with $d_{in}(\tilde{\theta}_{t,n}) = d'$, $d_{head}(\tilde{\theta}_{t,n}) = k(\tilde{\theta}_{t,n}) = 1$, and $H(\tilde{\theta}_{t,n}) = d'$.

Finally, Lemma 3 approximates $G$ in Equation (17) by a deep transformer of the form of Equation (7). To do that, the authors approximate the component-wise multiplication maps $(x, y) \in \mathbb{R}^{2d'} \mapsto x \odot y \in \mathbb{R}^{d'}$ in Equation (17) by some MLPs. This yields the final result: for any $\varepsilon > 0$, there exist $L$ and parameters $(\theta_\ell, \xi_\ell)_{\ell=1}^L$ such that for all $(\mu, x) \in \mathcal{P}(\Omega) \times \Omega$: $|G(\mu, x) - F_{\xi_L} \diamond \Gamma_{\theta_L} \diamond \cdots \diamond F_{\xi_1} \diamond \Gamma_{\theta_1}(\mu, x)| \leq \varepsilon$, with $d_{in}(\theta_\ell) \leq d + 3d'$, $d_{head}(\theta_\ell) = k(\theta_\ell) = 1$, and $H(\theta_\ell) \leq d'$.

**The Masked Universality Proof (Theorem 2)**

The masked setting requires additional technical conditions to cope with the causality constraint. Throughout Section 4, $\Omega \subset \mathbb{R}^d$ is a compact set, and the authors define $\tilde{\Omega} := \Omega \times [0, 1]$ as the space-time domain. The marginal with respect to only the time variable of some space-time measure $\mu \in \mathcal{P}(\tilde{\Omega})$ is denoted $\bar{\mu} := P_\sharp \mu$ where $P: \tilde{\Omega} \ni (x, t) \mapsto t \in [0, 1]$.

Definition 1 introduces **Lipschitz contexts**. A map $t \in [0, 1] \mapsto \mu(\cdot|t) \in \mathcal{P}(\Omega)$ is $C$-Lipschitz if for all $(s, t) \in [0, 1]^2$: $W_2(\mu(\cdot|s), \mu(\cdot|t)) \leq C|s - t|$. The set of space-time $C$-Lipschitz measures is $\text{Lip}_C(\tilde{\Omega}) := \{ \mu \in \mathcal{P}(\tilde{\Omega}) : \exists \mu(\cdot|t) \text{ s.t. } \mu(x, s) = \mu(x|s)\bar{\mu}(s) \text{ and } \mu(\cdot|t) \text{ is } C\text{-Lipschitz} \}$. The conditional measure $t \mapsto \mu(\cdot|t) \in \mathcal{P}(\Omega)$ is any valid disintegration of $\mu$ against the marginal $\bar{\mu}$, and must be $C$-Lipschitz. The authors also define for all $\sigma \in (0, 1)$: $\text{Lip}_C^\sigma(\tilde{\Omega}) := \{ \mu \in \text{Lip}_C(\tilde{\Omega}) : \bar{\mu}(\{0\}) \geq \sigma \}$.

These conditions are automatically satisfied by a discrete measure $\mu = \frac{1}{n}\sum_{i=1}^n \delta_{(x_i, t_i)}$ with distinct times $\delta := \min_{i \neq j} |t_i - t_j| > 0$ (using $C = \text{Radius}(\Omega)/\delta$). More generally, if $t \in [0, 1] \mapsto \phi(t) \in \Omega$ is $C$-Lipschitz and $\nu \in \mathcal{P}([0, 1])$, then $\mu = \phi_\sharp \nu \in \text{Lip}_C(\tilde{\Omega})$.

Definition 2 introduces **masked measures**. For $\mu \in \text{Lip}_C^\sigma(\tilde{\Omega})$, the masked probability measure $\mu_t \in \text{Lip}_C^\sigma(\tilde{\Omega})$ is defined as $\mu_t := \frac{\mathbf{1}_{[0,t]}}{\bar{\mu}([0, t])} \cdot \mu$. Thanks to $\mu \in \text{Lip}_C^\sigma(\tilde{\Omega})$, where the starting point of $\bar{\mu}$ is fixed as 0 (i.e., $0 \in \text{supp}(\bar{\mu})$), the masked probability measure $\mu_t$ is well-defined when $t \in (0, 1]$. The authors note that $\mu_t$ can be defined at $t = 0$ as the limit (in the weak* topology), and such a limit exists (see Lemma 10). Thus, the map $[0, 1] \ni t \mapsto \mu_t \in \text{Lip}_C^\sigma(\tilde{\Omega})$ is continuous (for the weak* topology).

Definition 3 introduces **causal identifiable maps**. A space-time in-context map $\Lambda$ is said to be causal if for all $(\mu, x, t) \in \text{Lip}_C^\sigma(\tilde{\Omega}) \times \tilde{\Omega}$: $\Lambda(\mu, x, t) = \Lambda(\mu_t, x, t)$. Such a map $\Lambda$ is said to be identifiable if for any context $\mu \in \text{Lip}_C^\sigma(\tilde{\Omega})$: $\mu_t = \mu_{t'} \implies \Lambda(\mu_t, \cdot, t) = \Lambda(\mu_{t'}, \cdot, t')$. By construction, the masked attention map $\Gamma_\theta$ defined in Equation (12) is a causal identifiable in-context map (see Lemma 11). Since the composition of such maps preserves both causality and identifiability (see Lemma 12), a deep transformer formed by composing causal identifiable in-context maps remains these properties.

Theorem 2 states the masked universality result. Let $\Lambda^\star$ be a continuous (where $\text{Lip}_C^\sigma(\tilde{\Omega})$ is endowed with the weak* topology) and causal identifiable in-context mapping. Then for all $\varepsilon > 0$, there exist $L$ and parameters $(\theta_\ell, \xi_\ell)_{\ell=1}^L$ such that for all $(\mu, x, t) \in \text{Lip}_C^\sigma(\tilde{\Omega}) \times \tilde{\Omega}$: $|F_{\xi_L} \diamond \Gamma_{\theta_L} \diamond \cdots \diamond F_{\xi_1} \diamond \Gamma_{\theta_1}(\mu, x, t) - \Lambda^\star(\mu, x, t)| \leq \varepsilon$, with $d_{in}(\theta_\ell) \leq d + 3d'$, $d_{head}(\theta_\ell) = k(\theta_\ell) = 1$, and $H(\theta_\ell) \leq d'$.

Remark 1 notes the sharpness of the identifiability and Lipschitz hypotheses. The masked approximation in Theorem 2 shares the same conclusion as the unmasked Theorem 1, but it requires stronger assumptions. In particular, the map $\Lambda^\star$ is assumed to be identifiable. This hypothesis is sharp and cannot be weakened: transformers define identifiable maps, and as proved in Lemma 13, identifiable maps can only approximate uniformly identifiable maps. Identifiability is also crucial since the proof technique involves recasting the approximation over $(\mu, x, t)$ as an approximation over a reduced space $(\mu_t, x)$. Another important assumption is that the authors restrict approximation to Lipschitz contexts. This limitation is essential for ensuring that the set of masked contexts $\mu_t$ is compact, which allows application of the Stone-Weierstrass theorem.

The proof of Theorem 2 introduces a **reduced mapping** in Definition 4. For $\Lambda: \mathcal{P}(\tilde{\Omega}) \times \tilde{\Omega} \to \mathbb{R}^{d'}$, the reduced map $\bar{\Lambda}$ takes two arguments $(\mu, x)$ as $\bar{\Lambda}(\mu, x) := \Lambda(\mu, x, e(\bar{\mu}))$, where $e(\bar{\mu})$ is the end point of $\text{supp}(\bar{\mu})$, that is, $e(\bar{\mu}) := \max\{r \in \text{supp}(\bar{\mu})\} \in [0, 1]$. The authors introduce the reduced space on which they consider the approximation: $\mathcal{X}_C^\sigma := \{ (\mu_t, x) : \mu \in \text{Lip}_C^\sigma(\tilde{\Omega}), x \in \Omega, t \in [0, 1] \}$.

Lemma 4 states that for a causal identifiable in-context mapping $\Lambda$, two properties hold: (i) for any $(\mu, x, t) \in \text{Lip}_C^\sigma(\tilde{\Omega}) \times \tilde{\Omega}$, $\Lambda(\mu, x, t) = \bar{\Lambda}(\mu_t, x)$, and (ii) if $\Lambda$ is continuous, then the reduced map $\mathcal{X}_C^\sigma \ni (\mu_t, x) \mapsto \bar{\Lambda}(\mu_t, x)$ is (weak* $\times \, \ell_2$)-continuous. As the target map $\Lambda^\star$ is a continuous and causal identifiable in-context mapping, by applying Lemma 4, $\Lambda^\star$ has form (i) and is continuous on $\mathcal{X}_C^\sigma$. Also, the masked attention map $\Gamma_\theta$ holds the same properties (see Lemma 11). Thus, it suffices to show Proposition 2, which states that for all $\varepsilon > 0$, there exist $L$ and parameters $(\theta_\ell, \xi_\ell)_{\ell=1}^L$ such that for all $(\mu_t, x) \in \mathcal{X}_C^\sigma$: $|F_{\xi_L} \diamond \bar{\Gamma}_{\theta_L} \diamond \cdots \diamond F_{\xi_1} \diamond \bar{\Gamma}_{\theta_1}(\mu_t, x) - \bar{\Lambda}^\star(\mu_t, x)| \leq \varepsilon$, with the same architectural constraints as Theorem 1.

The proof of Proposition 2 is basically the same as in the unmasked case, just replacing the arguments for $\mu \in \mathcal{P}(\Omega)$ with that for $\mu_t \in \text{Lip}_C^\sigma(\tilde{\Omega})$. For the application of the Stone-Weierstrass theorem, the authors need to check the compactness of $\mathcal{X}_C^\sigma$, which is not obvious. Lemma 5 states that $\mathcal{X}_C^\sigma$ is compact for the (weak* $\times \, \ell_2$) topology. The sketch of proof assumes that $\mu_n \in \text{Lip}_C^\sigma(\tilde{\Omega})$ and $(x_n, t_n) \in \tilde{\Omega}$. The authors denote $\mathcal{P}^\sigma([0, 1]) := \{ \nu \in \mathcal{P}([0, 1]) : \nu(\{0\}) \geq \sigma \}$, and $\bar{\mu}_n \in \mathcal{P}^\sigma([0, 1])$. As $\mathcal{P}^\sigma([0, 1])$ and $\tilde{\Omega}$ are compact, there exist $\bar{\mu} \in \mathcal{P}^\sigma([0, 1])$ and $(x, t) \in \tilde{\Omega}$ such that (if necessary, re-choosing a subsequence) $\bar{\mu}_n \rightharpoonup^* \bar{\mu}$ and $(x_n, t_n) \to (x, t)$. Since $[0, 1] \ni s \mapsto \mu_n(\cdot|s) \in \mathcal{P}(\Omega)$ is $C$-Lipschitz, applying the general Arzelà–Ascoli theorem, there exists $\mu(\cdot|s) \in \mathcal{P}(\Omega)$ such that (if necessary, re-choosing a subsequence) $\sup_{s \in [0, 1]} W_2(\mu_n(\cdot|s), \mu(\cdot|s)) \to 0$ as $n \to \infty$. Thus, the authors define $\mu := \mu(\cdot|s)\bar{\mu}(s)$ which belongs to $\text{Lip}_C^\sigma(\tilde{\Omega})$. Using this convergence and the fact that the start points of $\bar{\mu}_n, \bar{\mu} \in \mathcal{P}^\sigma([0, 1])$ are always fixed at $t = 0$, they can prove the convergence of the masked probability measures, i.e., $(\mu_n)_{t_n} \rightharpoonup^* \mu_t$ as $n \to \infty$.

**Architectural Constraints and Design Choices**

The paper makes several specific architectural choices that are crucial to the universality results. The head dimension is fixed at $d_{head}(\theta_\ell) = 1$, meaning each attention head outputs a scalar. The key/query dimension is also fixed at $k(\theta_\ell) = 1$. The number of heads is bounded by $H(\theta_\ell) \leq d'$, where $d'$ is the output dimension, meaning the number of heads is proportional to the output dimension rather than the token count or precision. The input dimension is bounded by $d_{in}(\theta_\ell) \leq d + 3d'$, which is independent of $\varepsilon$ and $n$.

The use of MLPs between multi-head attention layers is explicitly controlled. The MLPs approximate a real-valued squaring operator $\mathbb{R} \ni a \mapsto a^2$, so the authors expect this dependency to be well-behaved in common situations. However, the construction does not provide any a priori bound on how the magnitude of the tokens grows through the layers. The authors note that obtaining a better balance between the number of heads and the output dimension is an interesting problem for future work.

The proof relies on the approximation of the map along each dimension and the use of a commuting architecture (the transformer layers are multiplied together to obtain the output). This results in a growth of the number of heads proportional to the dimension. Lowering this dependency would require the development of new proof techniques beyond the use of the Stone-Weierstrass theorem.

**Key Design Choices and Alternatives**

The measure-theoretic approach was chosen over alternative formulations for several reasons. First, representing contexts as probability distributions rather than finite sets enables a unified treatment of finite and infinite token counts, which is essential for proving universality over arbitrary context lengths. Second, the Wasserstein distance provides a natural notion of continuity that respects the geometry of the token space, unlike simpler metrics that might not capture the semantic similarity between different token distributions. Third, the push-forward formulation of attention as a measure transformation makes the mathematical analysis tractable while preserving the essential structure of the attention mechanism.

The space-time lifting for the masked setting was necessary because causal attention breaks permutation equivariance. Without introducing temporal coordinates, there would be no way to represent the causal structure using probability measures. The Lipschitz condition on contexts ensures that the token distribution does not change too rapidly with time, which is needed to establish compactness of the reduced space $\mathcal{X}_C^\sigma$ for applying the Stone-Weierstrass theorem.

The choice to use the Stone-Weierstrass theorem rather than constructive approximation methods reflects the paper's focus on existence rather than quantitative bounds. The theorem guarantees that an approximating transformer exists but does not provide explicit bounds on how the depth $L$ scales with precision $\varepsilon$. The authors acknowledge this as a limitation and note that using the Wasserstein distance between token distributions could be a way to impose smoothness on the map to obtain quantitative bounds in future work.

## 4. Key Insights and Innovations

This section distills the paper's most significant conceptual breakthroughs. While Section 3 detailed the technical machinery, here we focus on *why* these innovations matter and what they reveal about transformer architecture that was previously unknown.

### 4.1 Fixed Embedding Dimension Independent of Context Length

**The Innovation:** This paper proves that transformer embedding dimensions need not scale with token count — a fundamental departure from all prior universality results.

**What Changed from Prior Work:**

| Prior Universality Results | This Paper |
|---------------------------|------------|
| Embedding dimension $d$ grows with token count $n$ [43] | $d_{in}(\theta_\ell) \leq d + 3d'$ independent of $n$ |
| Width must increase for more tokens | Depth compensates; width stays fixed |
| Finite token analysis only | Works for infinite token limits |

**Why This Is Significant:**

This is a **fundamental theoretical innovation**, not an incremental improvement. The practical implications are substantial:

- **Memory efficiency**: Attention matrices scale as $O(n^2 \cdot d)$. If $d$ had to grow with $n$, memory would scale super-linearly with context length. Fixed $d$ means quadratic scaling only in $n$, which is already challenging but manageable.

- **Architecture stability**: Real-world transformers use fixed embedding sizes (768, 1024, 4096 dimensions) regardless of context length. This paper explains *why this works theoretically* — it is not merely an engineering compromise but a mathematically sound property.

- **Depth over width**: The result suggests that for long-context applications, increasing depth $L$ may be more effective than increasing width. This provides theoretical grounding for architectural decisions in models designed for long sequences.

**The Key Mechanism:**

The proof achieves this through the measure-theoretic reformulation (Section 3.4). By representing contexts as probability distributions $\mu \in \mathcal{P}(\Omega)$ rather than finite matrices $X \in \mathbb{R}^{d \times n}$, the token count $n$ disappears from the mathematical object entirely. The attention integral in Equation (9):

$$\Gamma_\theta(\mu, x) := x + \sum_{h=1}^H W^h \int \frac{\exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h y \rangle\right)}{\int \exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h z \rangle\right) d\mu(z)} V^h y \, d\mu(y)$$

contains no reference to $n$. The discrete case is recovered by setting $\mu = \frac{1}{n}\sum_{i=1}^n \delta_{x_i}$, but the formula remains valid for any measure. This is why the embedding dimension bounds in Theorem 1 do not depend on token count.

**Caveat:** The number of heads $H(\theta_\ell) \leq d'$ grows proportionally to output dimension $d'$, not token count. The authors acknowledge this as an area for improvement — obtaining a better balance between heads and output dimension remains open.

---

### 4.2 Wasserstein Continuity as the Correct Notion of Smoothness

**The Innovation:** The paper identifies Wasserstein distance $W_p$ between probability measures as the appropriate metric for defining continuity of in-context mappings.

**What Makes This Novel:**

Prior transformer expressivity work typically analyzed functions on fixed-dimensional Euclidean spaces $\mathbb{R}^{d \times n}$. When contexts have different token counts, these spaces are not comparable. The Wasserstein distance solves this by:

- **Comparing different token counts**: $W_p(\mu, \nu)$ is defined for any two probability measures, regardless of whether they represent 100 tokens or 10,000 tokens.

- **Respecting token geometry**: Unlike simpler metrics (e.g., total variation distance), Wasserstein distance accounts for the *positions* of tokens in embedding space. Two contexts with similar token distributions but slightly shifted positions are "close" in Wasserstein distance.

- **Enabling the infinite limit**: The weak* topology metrized by $W_p$ allows analysis of the "mean field" limit where $n \to \infty$, connecting finite transformers to their continuous counterparts.

**Why This Matters:**

This is a **conceptual innovation** that reframes how we think about transformer smoothness. Consider two contexts:

- Context A: 1000 tokens clustered around position $x_0$
- Context B: 1000 tokens clustered around position $x_0 + \epsilon$

In Euclidean space with fixed $n$, these are $O(\sqrt{n}\epsilon)$ apart. In Wasserstein space, they are $O(\epsilon)$ apart — the distance does not grow with token count. This matches the intuition that these contexts are "similar" regardless of how many tokens they contain.

**The Technical Payoff:**

The Wasserstein continuity assumption enables the Stone-Weierstrass theorem application in Proposition 1. The compactness of $\mathcal{P}(\Omega)$ under weak* topology (for compact $\Omega$) is essential for the density argument. Without this measure-theoretic formulation, the universality proof would not extend to arbitrary token counts.

**Limitation:** The paper does not provide quantitative bounds on how approximation error scales with Wasserstein distance. As the authors note in the Conclusion:

> "A limitation of our method is that it is not quantitative. Using, for instance, the Wasserstein distance between token distributions could be a way to impose smoothness on the map to obtain quantitative bounds."

This leaves open the question of *how many layers* are needed for a given precision — an important direction for future work.

---

### 4.3 Identifiability as a Fundamental Constraint on Causal Transformers

**The Innovation:** The paper discovers that masked (causal) transformers can only approximate **identifiable** in-context mappings — a constraint that does not exist in the unmasked setting.

**What Is Identifiability?**

From Definition 3, a map $\Lambda$ is identifiable if:

$$\mu_t = \mu_{t'} \implies \Lambda(\mu_t, \cdot, t) = \Lambda(\mu_{t'}, \cdot, t')$$

In words: if two masked contexts contain the same tokens (up to time $t$ and $t'$ respectively), the transformer must produce the same output regardless of the absolute time position.

**Why This Is a Fundamental Discovery:**

This is a **theoretical constraint** that was previously unknown. Lemma 13 proves that identifiable maps can only approximate uniformly identifiable maps:

> "Identifiability is also crucial since our proof technique involves recasting the approximation over $(\mu, x, t)$ as an approximation over a reduced space $(\mu_t, x)$."

**Practical Implications:**

This reveals a subtle limitation of causal transformers:

- **Cannot distinguish equivalent histories**: If a language model sees the same sequence of tokens at different positions in a document, it must produce equivalent representations (up to the masking constraint). This is actually desirable for many applications but was not previously characterized as a mathematical necessity.

- **Stronger assumptions than unmasked**: Theorem 2 requires both Lipschitz contexts *and* identifiability, while Theorem 1 only requires continuity. This means masked transformers are theoretically less expressive than unmasked ones for the same architecture — a tradeoff for causality.

- **Architecture design**: If you need to approximate non-identifiable causal mappings, standard transformer architectures cannot do it. You would need architectural modifications that break the identifiability constraint.

**Why This Was Hidden Before:**

Prior universality analyses did not model the temporal structure of masked attention carefully enough to discover this constraint. The space-time lifting and masked measure formulation (Section 2.3) were necessary to expose the identifiability requirement.

---

### 4.4 Depth Compensates for Width in Transformer Expressivity

**The Innovation:** The paper demonstrates that transformer depth $L$ can substitute for width (embedding dimension, number of heads) in achieving universality.

**The Tradeoff Characterization:**

From Theorem 1 and Theorem 2, the architectural bounds are:

- $d_{head}(\theta_\ell) = k(\theta_\ell) = 1$ (minimal head dimension)
- $H(\theta_\ell) \leq d'$ (heads proportional to output, not precision)
- $d_{in}(\theta_\ell) \leq d + 3d'$ (input dimension bounded independently of $\varepsilon$ and $n$)
- $L$ is *unbounded* and depends on $\varepsilon$

**What This Reveals:**

This is a **structural insight** about transformer architecture. The proof constructs the approximating transformer through iterative composition of elementary mappings (Equation 14). Each layer adds expressivity, and enough layers can compensate for narrow widths.

**Comparison to Classical Results:**

Classical MLP universality [10, 19] also shows that depth can compensate for width, but with different tradeoffs:

| Architecture | Width Requirement | Depth Requirement |
|-------------|-------------------|-------------------|
| Classical MLP [10, 19] | Grows with input dimension | 2 layers sufficient |
| Transformers [43] | Grows with token count | Shallow (2 heads) |
| **This Paper** | **Fixed** | **Grows with precision** |

**Practical Significance:**

This provides theoretical grounding for empirical observations:

- **Deep, narrow models** can handle long contexts as well as shallow, wide models (in terms of expressivity, not efficiency)
- **Scaling laws**: The result suggests that for long-context applications, depth scaling may be more important than width scaling
- **Memory constraints**: When memory limits width, increasing depth is a theoretically valid alternative

**The Open Question:**

The paper does not quantify *how* depth scales with precision. As noted in Remark 3 (Appendix B.3):

> "The key is to approximate and capture the mentioned multiplicity by MLPs, for which quantitative estimates have been studied... However, the depth and width of MLPs depend on the bound of input variables... Thus, providing the quantitative estimate for the MLP, $\Phi$, is challenging."

This is a critical gap: we know deep transformers *can* approximate any continuous in-context mapping, but we do not know *how deep* they need to be for a given precision. This remains an important direction for future theoretical work.

---

### 4.5 Unified Framework for Finite and Infinite Contexts

**The Innovation:** The measure-theoretic formulation provides a single mathematical framework that handles both finite token sets and continuous token distributions.

**Why This Is Conceptually Important:**

This is a **unifying innovation** that bridges discrete and continuous analysis. The same equations (Equation 9 for unmasked, Equation 12 for masked) describe:

- A transformer processing 100 tokens
- A transformer processing 10,000 tokens
- The "mean field" limit as $n \to \infty$

**The Mathematical Elegance:**

The push-forward formulation $\mu \mapsto \Gamma_\theta(\mu)_\sharp \mu$ treats the transformer as a dynamical system on probability measures. Each layer transforms the context distribution, and the composition of layers corresponds to iterating this transformation. This connects transformer analysis to:

- **Optimal transport theory** (via Wasserstein distance)
- **Mean field theory** (via the infinite token limit)
- **Dynamical systems** (via the layer-to-layer evolution)

**Practical Relevance:**

While the infinite token limit may seem abstract, it has concrete implications:

- **Generalization guarantees**: If a transformer works for arbitrary measures, it generalizes to any token distribution encountered at test time (within the compact domain $\Omega$)
- **Robustness analysis**: The continuity in Wasserstein distance implies that small perturbations in the token distribution produce small changes in output — a form of robustness
- **Theoretical tools**: The measure-theoretic framework enables applying tools from functional analysis and optimal transport to transformer theory

**Limitation:**

The compact domain assumption $\Omega \subset \mathbb{R}^d$ is restrictive. Real token embeddings may not lie in a known compact set, and the bounds $d_{in}(\theta_\ell) \leq d + 3d'$ depend on the dimension of this compact set. Extending the results to unbounded domains remains open.

## 5. Experimental Analysis

### 5.1 Evaluation Methodology: Theoretical Validation vs. Empirical Benchmarks

It is critical to clarify at the outset that this paper is a **theoretical work**. It does not contain empirical experiments, datasets, or performance benchmarks in the traditional machine learning sense (e.g., accuracy on GLUE, perplexity on WikiText, or image classification scores). There are no empirical tables or figures reporting numerical performance on real-world tasks.

Instead, the "evaluation" consists of **mathematical proofs** that validate the hypotheses regarding transformer expressivity. The methodology relies on functional analysis and measure theory rather than gradient descent or validation sets.

*   **Datasets**: None. The analysis operates on abstract compact domains $\Omega \subset \mathbb{R}^d$ and probability measures $\mu \in \mathcal{P}(\Omega)$. The results hold for *any* token distribution within these domains, effectively generalizing across all possible "datasets" that fit the compactness assumption.
*   **Metrics**: The primary metric is the **uniform approximation error** $\varepsilon > 0$. The goal is to show that for any continuous target mapping $\Lambda^\star$, there exists a transformer architecture such that the supremum distance between the transformer output and $\Lambda^\star$ is less than $\varepsilon$.
*   **Baselines**: The paper compares its theoretical bounds against prior universality results cited in the Introduction, specifically:
    *   **[43] Yun et al.**: Requires embedding dimension to grow with token count.
    *   **[3] Sumformer**: Requires embedding dimension to grow with token count.
    *   **[11] Stochastic Deep Networks**: Requires embedding dimension to grow with token count.
*   **Experimental Setup**: The "setup" is the mathematical framework defined in Sections 2 and 3:
    *   **Token Representation**: Probability measures $\mu$ rather than finite matrices $X$.
    *   **Continuity**: Defined via the Wasserstein distance $W_p$ and weak* topology.
    *   **Architecture**: Deep transformers with specific constraints on head dimension, key dimension, and number of heads.

### 5.2 Quantitative Results: Architectural Bounds

While there are no empirical accuracy numbers, the paper provides precise **quantitative bounds** on the architectural parameters required to achieve universality. These bounds are the "results" of the theoretical analysis.

**Theorem 1 (Unmasked Setting)** and **Theorem 2 (Masked Setting)** establish the following specific constraints for an approximating transformer with $L$ layers:

*   **Head Dimension**: Fixed at $d_{head}(\theta_\ell) = 1$. Each attention head outputs a scalar.
*   **Key/Query Dimension**: Fixed at $k(\theta_\ell) = 1$.
*   **Number of Heads**: Bounded by $H(\theta_\ell) \leq d'$, where $d'$ is the output dimension. Crucially, this does **not** depend on the number of tokens $n$ or the precision $\varepsilon$.
*   **Input Dimension**: Bounded by $d_{in}(\theta_\ell) \leq d + 3d'$, where $d$ is the token embedding dimension. This is independent of token count $n$.
*   **Depth**: The number of layers $L$ exists but is **not explicitly bounded** in terms of $\varepsilon$. The proof establishes existence ("there exist $L$") but does not provide a formula for how $L$ scales with precision.

> "The two strengths of this result are (i) the approximating architecture performs the approximation independently of $n$ (it even works for an infinite number of tokens), and (ii) the number of heads and the embedding dimension do not depend on $\varepsilon$." (Section 3.1)

These bounds are significant because they contradict prior theoretical results (e.g., [43]) which suggested that width (embedding dimension) must scale with context length. Here, the paper demonstrates that **fixed width** is sufficient if depth is allowed to vary.

### 5.3 Comparison to Prior Theoretical Baselines

The paper positions its results against existing universality theorems in the Introduction (Section 1). The comparison highlights the novelty of the fixed-dimensional architecture.

| Feature | Prior Work [43, 3, 11] | This Paper (Theorems 1 & 2) |
| :--- | :--- | :--- |
| **Embedding Dimension** | Grows with token count $n$ | **Fixed** ($d_{in} \leq d + 3d'$) |
| **Token Count** | Arbitrary (finite) | **Arbitrary (even infinite)** |
| **Attention Heads** | Varies (e.g., 2 heads in [43]) | **Proportional to output** ($H \leq d'$) |
| **Setting** | Mostly Unmasked | **Both Unmasked and Masked** |

The key differentiator is the **fixed embedding dimension**. As noted in Section 1:

> "[43] provides, to the best of our knowledge, the most detailed account of the universality of transformers. The authors rely on shallow transformers with only 2 heads but require that the transformers operate over an embedding dimension which grows with the number of tokens."

This paper resolves that limitation by showing that depth compensates for width. However, this comes with a trade-off: the prior work [43] uses shallow transformers (2 heads), whereas this work requires **deep** transformers (arbitrary $L$) to achieve the fixed-width property.

### 5.4 Limitations and Open Questions

The theoretical "experiments" reveal several limitations that are explicitly acknowledged by the authors. These are critical for understanding the scope of the claims.

*   **Non-Quantitative Depth Bounds**: The proof does not specify how many layers $L$ are needed for a given precision $\varepsilon$.
    > "A weakness is that we have no explicit control over the dependency of the number of MLP parameters $\xi_\ell$ on $\varepsilon$." (Section 3.1)
    This means while we know a solution *exists*, we do not know if it requires 10 layers or 10,000 layers for a specific task. This limits practical utility for architecture design.

*   **Head Dimension Trade-off**: While the embedding dimension is fixed, the number of heads scales with the output dimension $d'$.
    > "Another limitation of our proof technique is that the number of heads grows proportionally to the output dimension while each head only outputs a scalar $d_{head}(\theta_\ell) = 1$." (Section 3.1)
    The authors note that obtaining a better balance between heads and output dimension is an open problem.

*   **Masked Setting Constraints**: The masked universality (Theorem 2) requires stronger assumptions than the unmasked case.
    *   **Lipschitz Contexts**: The token distribution must change smoothly over time (Definition 1).
    *   **Identifiability**: The target mapping must be "identifiable" (Definition 3), meaning equivalent masked histories must produce equivalent outputs.
    > "The masked setting requires some stronger regularity hypothesis on the contexts, namely that they are Wasserstein-Lipschitz with respect to time..." (Section 1.1)
    This implies that causal transformers are theoretically less expressive than unmasked ones unless these constraints are met.

*   **Absence of Empirical Verification**: There are no experiments verifying whether practical transformers (trained with SGD on real data) actually converge to these theoretical bounds. The paper proves *capacity* (what the architecture *can* represent), not *learnability* (whether gradient descent can find the parameters).

### 5.5 Assessment of Claims

The paper's claims are **mathematically rigorous** within the stated assumptions. The proofs rely on standard tools (Stone-Weierstrass theorem, Arzelà–Ascoli theorem) and are detailed in the Appendix (Sections B and C).

*   **Claim**: "Transformers are universal in-context learners with fixed embedding dimension."
    *   **Support**: **Strong**. Theorem 1 and Theorem 2 provide constructive proofs (via Lemma 2 and Lemma 3) that such architectures exist.
*   **Claim**: "Works for arbitrary (even infinite) number of tokens."
    *   **Support**: **Strong**. The measure-theoretic formulation (Section 2.2) explicitly handles probability measures, which encompass infinite supports.
*   **Claim**: "Applies to both masked and unmasked settings."
    *   **Support**: **Conditional**. Supported for masked settings only under Lipschitz and identifiability constraints (Theorem 2), which are stricter than the unmasked case (Theorem 1).

**Conclusion on Experiments**: Since this is a theory paper, the "experimental analysis" is confined to the logical validity of the proofs and the tightness of the derived bounds. The results convincingly support the claim that fixed-width deep transformers are theoretically capable of handling arbitrary context lengths, but they leave open the practical question of how efficiently (in terms of depth and training dynamics) this capability can be realized.

## 6. Limitations and Trade-offs

This section critically examines the assumptions, constraints, and open questions that temper the paper's universality claims. While the theoretical results are mathematically rigorous, several important limitations affect their practical interpretation and scope.

### 6.1 Theoretical Assumptions That Restrict Applicability

**Compact Domain Requirement**

The entire analysis rests on the assumption that token embeddings lie within a **compact domain** $\Omega \subset \mathbb{R}^d$. This is stated explicitly in Theorem 1:

> "Let $\Omega \subset \mathbb{R}^d$ be a compact set and $\Lambda^\star: \mathcal{P}(\Omega) \times \Omega \to \mathbb{R}^{d'}$ be continuous..."

This assumption is mathematically necessary for applying the Stone-Weierstrass theorem (which requires compactness of the domain) and for ensuring the Wasserstein distance metrizes the weak* topology. However, it creates a gap between theory and practice:

- **Real token embeddings are unbounded**: In practice, transformer token embeddings can take arbitrary values in $\mathbb{R}^d$, especially during training before normalization stabilizes them. The compactness assumption means the universality guarantee does not formally apply to all possible token configurations.

- **No explicit bound specification**: The paper does not specify what compact set $\Omega$ should be used or how the architectural bounds ($d_{in} \leq d + 3d'$) depend on the size of $\Omega$. This makes it difficult to translate the theoretical bounds into practical architecture design.

**Continuity Requirements on Target Mappings**

The universality theorems only apply to **continuous** in-context mappings. Theorem 1 requires $\Lambda^\star$ to be continuous with respect to the weak* topology on $\mathcal{P}(\Omega)$. This excludes:

- **Discontinuous functions**: Any target mapping with discontinuities (e.g., threshold-based decisions, discrete classification boundaries) falls outside the scope of the guarantee.

- **Functions with high-frequency variations**: While technically continuous, mappings that oscillate rapidly may require prohibitively large depth $L$ to approximate, though the paper provides no quantitative bound on this relationship.

The authors acknowledge this limitation in Section 3.1:

> "The main hypothesis of Theorem 1 is that the underlying map, $\Lambda^\star$, is a smooth (at least continuous) map for the weak* topology over measures... However, this setting might not be a proper one for conducting further quantitative studies."

**Masked Setting: Stronger Regularity Constraints**

The masked (causal) universality result (Theorem 2) requires significantly stronger assumptions than the unmasked case (Theorem 1). Two additional constraints are imposed:

1. **Lipschitz Contexts** (Definition 1): The token distribution must change smoothly over time:
   $$W_2(\mu(\cdot|s), \mu(\cdot|t)) \leq C|s - t| \quad \forall (s, t) \in [0, 1]^2$$
   
   This excludes contexts where the token distribution changes abruptly. As noted in Section 4.1:
   > "This limitation is essential for ensuring that the set of masked contexts $\mu_t$ is compact, which allows us to apply the Stone-Weierstrass theorem."

2. **Identifiability** (Definition 3): The target mapping must satisfy:
   $$\mu_t = \mu_{t'} \implies \Lambda(\mu_t, \cdot, t) = \Lambda(\mu_{t'}, \cdot, t')$$
   
   Lemma 13 proves this constraint is **sharp** — identifiable maps can only approximate uniformly identifiable maps. This means:
   - Causal transformers cannot approximate non-identifiable mappings, regardless of depth or width.
   - This is a fundamental expressivity limitation specific to the masked setting.

The paper explicitly notes this asymmetry in Section 1.1:

> "The masked setting requires some stronger regularity hypothesis on the contexts, namely that they are Wasserstein-Lipschitz with respect to time, which is needed to cope with the constraint of causality in the relevant mappings."

**Time Marginal Constraint**

Remark 2 in Section 4.1 identifies an additional technical constraint:

> "We impose that contexts have Dirac masses at 0, namely $\bar{\mu}(\{0\}) \geq \sigma$. While this is not restrictive for discrete measures, this prevents for instance measures with density with respect to Lebesgues."

This means the theory does not directly apply to contexts where time is continuously distributed (e.g., tokens arriving according to a continuous temporal process). The authors note this can be relaxed with "minor modifications," but the base theorem does not cover this case.

### 6.2 Architectural Trade-offs: What Is Gained vs. What Is Sacrificed

**Depth Compensates for Width — But How Much Depth?**

The paper's key innovation is showing that **fixed embedding dimension** suffices for universality, with depth $L$ absorbing the complexity. However, this introduces a critical trade-off:

| Parameter | Prior Work [43] | This Paper |
|-----------|-----------------|------------|
| Embedding Dimension | Grows with $n$ | **Fixed** ($d + 3d'$) |
| Depth | Shallow (2 heads) | **Unbounded** (depends on $\varepsilon$) |
| Heads | 2 | **Proportional to $d'$** ($H \leq d'$) |

The problem is that the paper provides **no quantitative bound** on how depth scales with approximation precision. Section 3.1 explicitly states:

> "A weakness is that we have no explicit control over the dependency of the number of MLP parameters $\xi_\ell$ on $\varepsilon$."

This has practical implications:

- **Unknown depth requirements**: For a target precision $\varepsilon = 0.01$, the theory does not tell us whether $L = 10$, $L = 100$, or $L = 10,000$ layers are needed.

- **Training feasibility**: Even if a deep transformer *can* represent a function, gradient descent may not find the parameters efficiently. The paper proves *capacity*, not *learnability*.

**Head Dimension vs. Number of Heads**

The architectural constraints in Theorems 1 and 2 specify:
- $d_{head}(\theta_\ell) = k(\theta_\ell) = 1$ (each head outputs a scalar)
- $H(\theta_\ell) \leq d'$ (number of heads proportional to output dimension)

This is an unusual design compared to practical transformers, which typically use:
- $d_{head} \in \{64, 128, 256\}$ (larger head dimensions)
- $H \in \{8, 12, 16, 32\}$ (fixed number of heads, independent of output dimension)

The authors acknowledge this mismatch in Section 3.1:

> "Another limitation of our proof technique is that the number of heads grows proportionally to the output dimension while each head only outputs a scalar $d_{head}(\theta_\ell) = 1$. Obtaining a better balance between these two parameters is an interesting problem."

This suggests the proven architecture may be **inefficient** compared to practical designs. The theoretical guarantee does not imply that standard multi-head attention configurations (with larger $d_{head}$) achieve the same universality with fixed dimensions.

**MLP Approximation of Multiplication**

The proof requires MLPs to approximate component-wise multiplication $(x, y) \mapsto x \odot y$. Lemma 3 and Lemma 8 (Appendix B.3) construct this approximation, but Remark 3 notes:

> "The key is to approximate and capture the mentioned multiplicity by MLPs... However, the depth and width of MLPs depend on the bound of input variables... Thus, providing the quantitative estimate for the MLP, $\Phi$, is challenging."

This introduces two issues:

1. **Token magnitude growth**: The construction does not bound how token magnitudes grow through layers. If values explode, the MLP approximation may fail.

2. **MLP size uncertainty**: The number of MLP parameters $\xi_\ell$ needed depends on $\varepsilon$, but no explicit bound is provided. This means the total model size (attention + MLP) is not fully characterized.

### 6.3 Practical Gaps: Theory vs. Implementation

**No Learnability Guarantees**

The paper proves that approximating transformers *exist*, but does not address whether they can be *found* through training:

- **Optimization landscape**: The proof is constructive in a mathematical sense (showing existence via Stone-Weierstrass), but does not provide a gradient-based construction. Standard training with SGD may not converge to the theoretical solution.

- **Sample complexity**: The theory does not specify how many training examples are needed to learn an $\varepsilon$-approximation. This is critical for understanding data efficiency.

- **Generalization**: The universality result is about approximation capacity on a compact domain, not generalization to unseen token distributions.

The Conclusion explicitly acknowledges this gap:

> "Our proof relies on the approximation of the map along each dimension and the use of a commuting architecture... This results in a growth of the number of heads proportional to the dimension. Lowering this dependency would require the development of new proof techniques beyond the use of the Stone-Weierstrass theorem."

**No Empirical Validation**

As noted in Section 5, the paper contains no experiments. This means:

- **Practical feasibility is untested**: We do not know if transformers trained on real tasks actually exhibit the fixed-dimension universality behavior.

- **Depth-precision relationship is unknown**: Without experiments, we cannot estimate how depth $L$ scales with task complexity in practice.

- **Architectural recommendations are theoretical**: The suggestion that "depth compensates for width" is not empirically validated for long-context applications.

**Normalization Layers Omitted**

Section 2 explicitly states:

> "For the sake of simplicity, we omit normalization in the following analysis."

Layer normalization is a standard component of practical transformers. Its omission means:

- The theoretical results apply to a **simplified architecture** that differs from implementations.

- Normalization may affect the continuity properties assumed in the proofs (e.g., by bounding token magnitudes).

- The interaction between normalization and the measure-theoretic formulation is unexplored.

### 6.4 Open Questions and Future Directions

The paper identifies several unresolved issues that limit the completeness of the theory:

**Quantitative Bounds on Depth**

The most significant open question is characterizing how depth $L$ scales with precision $\varepsilon$. The Conclusion states:

> "A limitation of our method is that it is not quantitative. Using, for instance, the Wasserstein distance between token distributions could be a way to impose smoothness on the map to obtain quantitative bounds."

Without such bounds, the theory cannot guide practical architecture design (e.g., "how many layers do I need for 1% error?").

**Head-Output Dimension Balance**

The proportional relationship $H \leq d'$ is identified as suboptimal:

> "Obtaining a better balance between these two parameters is an interesting problem." (Section 3.1)

Future work could explore whether alternative proof techniques yield architectures with fixed $H$ independent of $d'$.

**Extension to Unbounded Domains**

The compact domain assumption $\Omega \subset \mathbb{R}^d$ is restrictive. Extending the results to unbounded domains (e.g., $\mathbb{R}^d$ with appropriate decay conditions) would broaden applicability.

**Learnability Analysis**

The paper opens the door for studying whether gradient descent can efficiently find the approximating transformers. This would bridge the gap between capacity (what transformers *can* represent) and learnability (what transformers *can be trained* to represent).

**Positional Encoding Impact**

The Introduction mentions that "another line of work studies the impact of positional encoding on their expressivity [26]," but this paper does not analyze how positional encodings interact with the measure-theoretic formulation. For the masked setting, time is encoded via the space-time lifting $(x, t)$, but this differs from standard positional encoding schemes.

### 6.5 Summary Assessment

The following table summarizes the key limitations and their severity:

| Limitation | Severity | Impact on Claims |
|------------|----------|------------------|
| Compact domain assumption | Moderate | Restricts token embedding space |
| No quantitative depth bounds | High | Limits practical architecture design |
| Masked setting constraints (Lipschitz, identifiability) | Moderate | Reduces expressivity for causal transformers |
| Head dimension trade-off ($H \propto d'$) | Moderate | Architecture differs from practice |
| No learnability guarantees | High | Theory does not address training |
| Normalization layers omitted | Low-Moderate | Simplified architecture vs. practice |
| No empirical validation | Moderate | Practical feasibility untested |

**Bottom Line**: The paper establishes a **strong theoretical foundation** for transformer universality with fixed dimensions, but the results are **existence proofs** rather than **constructive guidelines**. The limitations are primarily in **quantification** (how much depth? how many parameters?) and **practicality** (can this be trained? does it work on real data?). These gaps do not invalidate the theoretical contributions but indicate significant work remains to translate the theory into practical architecture design principles.

## 7. Implications and Future Directions

This paper does more than prove a theorem; it resolves a fundamental tension between transformer theory and practice. Prior universality results suggested that to handle more tokens, models must grow wider (embedding dimension increases). Yet, real-world transformers handle variable context lengths with fixed dimensions. This work bridges that gap, shifting the theoretical landscape from "width scales with context" to "depth scales with precision."

This section outlines how this result changes our understanding of transformer expressivity, what specific research questions it opens, and how practitioners should interpret these findings when designing architectures for long-context applications.

### 7.1 Shifting the Theoretical Landscape

The primary impact of this work is the **normalization of the measure-theoretic view** for transformer analysis. By representing contexts as probability measures $\mu \in \mathcal{P}(\Omega)$ rather than finite matrices $X \in \mathbb{R}^{d \times n}$, the paper removes the token count $n$ from the expressivity equations entirely.

**Resolution of the Dimension Paradox:**
Before this work, there was a disconnect:
*   **Practice:** Transformers use fixed embedding dimensions (e.g., $d=4096$) for contexts ranging from $n=100$ to $n=100,000$ tokens.
*   **Theory (e.g., [43]):** Universality required $d$ to grow with $n$.

Theorem 1 and Theorem 2 resolve this by proving that **fixed width is theoretically sufficient**. The key mechanism is the measure-theoretic attention integral in Equation (9):

$$\Gamma_\theta(\mu, x) := x + \sum_{h=1}^H W^h \int \frac{\exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h y \rangle\right)}{\int \exp\left(\frac{1}{\sqrt{k}} \langle Q^h x, K^h z \rangle\right) d\mu(z)} V^h y \, d\mu(y)$$

Because this integral is defined for any measure $\mu$, the architecture does not need to "know" $n$ in advance. This shifts the burden of expressivity from **width** (embedding dimension) to **depth** (number of layers $L$).

**New Standard for Continuity:**
The paper establishes **Wasserstein continuity** as the correct notion of smoothness for in-context mappings. Prior work often assumed Euclidean continuity on fixed-dimensional spaces. This work demonstrates that robustness to token perturbations should be measured via the Wasserstein distance $W_p(\mu, \nu)$, which respects the geometry of the token embedding space regardless of token count. Future expressivity analyses will likely need to adopt this metric to remain relevant for variable-length sequences.

### 7.2 Enabled Research Directions

The proofs in this paper are existential (showing that a solution exists) rather than constructive (showing how to find it efficiently). This distinction opens several critical avenues for future research.

**1. Quantitative Depth Bounds**
The most immediate open question is quantifying the depth-precision tradeoff. Theorem 1 states "there exist $L$" but does not specify how $L$ scales with $\varepsilon$.
*   **Research Goal:** Derive explicit bounds $L(\varepsilon, d, d')$.
*   **Why It Matters:** Without this, we cannot know if achieving $\varepsilon=0.01$ requires 10 layers or 10,000 layers. As noted in Section 3.1:
    > "A weakness is that we have no explicit control over the dependency of the number of MLP parameters $\xi_\ell$ on $\varepsilon$."
*   **Approach:** Future work could leverage quantitative versions of the Stone-Weierstrass theorem or analyze the spectral properties of the attention operator to bound convergence rates.

**2. Learnability and Optimization Dynamics**
This paper proves **capacity** (what the architecture *can* represent), not **learnability** (what gradient descent *can find*).
*   **Research Goal:** Connect the universality result to optimization landscapes. Does standard SGD converge to the universal approximators constructed in the proof?
*   **Why It Matters:** A function might be representable by a deep transformer but lie in a region of the loss landscape that is inaccessible to gradient-based optimization.
*   **Approach:** Analyze the Neural Tangent Kernel (NTK) or mean-field dynamics of the measure-theoretic transformer formulation. The push-forward structure $\mu \mapsto \Gamma_\theta(\mu)_\sharp \mu$ (Section 2.2) suggests a dynamical systems perspective that could link expressivity to training trajectories.

**3. Relaxing Regularity Constraints**
The masked setting (Theorem 2) requires **Lipschitz contexts** (Definition 1) and **identifiability** (Definition 3).
*   **Research Goal:** Determine if universality holds without these constraints, or prove they are fundamental limits.
*   **Why It Matters:** Real-world data may not satisfy Lipschitz continuity in time (e.g., sudden topic shifts in text). If these constraints are fundamental, it implies causal transformers have inherent blind spots.
*   **Approach:** Investigate whether architectural modifications (e.g., specific positional encodings or memory mechanisms) can bypass the identifiability constraint proved in Lemma 13.

**4. Efficient Head-Output Balancing**
The proof requires the number of heads to scale with output dimension ($H \leq d'$), with each head outputting a scalar ($d_{head}=1$).
*   **Research Goal:** Prove universality with fixed $H$ independent of $d'$, or with larger $d_{head}$.
*   **Why It Matters:** Practical transformers use $H \in \{8, 16, 32\}$ regardless of output dimension. The current theoretical bound suggests an inefficiency.
*   **Approach:** Develop new proof techniques beyond Stone-Weierstrass that allow for richer head interactions, as suggested in the Conclusion:
    > "Lowering this dependency would require the development of new proof techniques beyond the use of the Stone-Weierstrass theorem."

### 7.3 Practical Applications and Downstream Use Cases

While this is a theoretical work, the insights provide concrete guidance for architecture design and system analysis.

**Long-Context Architecture Design**
When designing models for documents, codebases, or high-resolution images:
*   **Guidance:** Prioritize **depth over width** for context handling. If you need to handle longer sequences, increasing layers $L$ is theoretically more sound than increasing embedding dimension $d$, since $d$ does not need to scale with $n$.
*   **Caveat:** Ensure the model remains trainable. Very deep models may suffer from vanishing gradients, requiring residual connections (which are included in the paper's $\text{MAtt}_\theta$ definition in Equation 1) and normalization (omitted in the proof but standard in practice).

**Robustness Analysis via Wasserstein Distance**
The continuity result implies that small perturbations in the token distribution should lead to small changes in output.
*   **Use Case:** Use Wasserstein distance $W_p$ to measure **distributional shift** between training and test contexts.
*   **Application:** If $W_p(\mu_{\text{train}}, \mu_{\text{test}})$ is small, the universality result suggests the model should generalize well, provided the target mapping is continuous. This offers a theoretical metric for out-of-distribution detection in sequence modeling.

**Causal Modeling Constraints**
The identifiability constraint (Section 4.3) reveals a limitation of causal transformers.
*   **Use Case:** When modeling time series or language, be aware that causal transformers **cannot distinguish** between histories that result in the same masked measure $\mu_t$.
*   **Application:** If your task requires distinguishing between two sequences that have the same token distribution up to time $t$ but different absolute time positions, a standard causal transformer is theoretically incapable of learning this mapping without architectural modifications (e.g., explicit time embeddings that break identifiability).

**Mean-Field Simulation**
The measure-theoretic formulation allows simulating transformers as dynamical systems on distributions.
*   **Use Case:** Analyze token clustering or collapse without simulating individual tokens.
*   **Application:** In large-scale systems where simulating $n=10^6$ tokens is expensive, one could approximate the token distribution as a continuous measure $\mu$ and simulate the evolution $\mu \mapsto \Gamma_\theta(\mu)_\sharp \mu$ directly. This connects to prior work on clustering dynamics [16, 17] cited in the Introduction.

### 7.4 Reproduction and Integration Guidance

Since this paper is theoretical, "reproduction" involves validating the architectural constraints or applying the theoretical lens to empirical work.

**When to Prefer This Framework**
*   **Variable-Length Contexts:** Use this theoretical lens when analyzing systems where context length varies significantly (e.g., document QA, variable-resolution vision). The fixed-dimension guarantee is most relevant here.
*   **Expressivity Analysis:** When comparing architectures (e.g., Transformer vs. State Space Models), use the measure-theoretic formulation to compare how each handles the limit $n \to \infty$.
*   **Causal Constraints:** When debugging causal models that fail to learn certain temporal patterns, check if the target mapping violates the **identifiability** condition (Definition 3).

**Integration Steps for Practitioners**
1.  **Architecture Check:** Verify your model uses residual connections (Equation 1) and sufficient depth. The theory relies on deep composition (Equation 7) to compensate for fixed width.
2.  **Normalization:** The paper omits normalization for simplicity (Section 2). In practice, integrate LayerNorm to ensure token magnitudes remain bounded, satisfying the compact domain assumption $\Omega \subset \mathbb{R}^d$ required by Theorem 1.
3.  **Monitoring Continuity:** During training, monitor the Wasserstein distance between batch-wise token distributions. Large jumps may indicate violation of the Lipschitz context assumption (Definition 1), potentially destabilizing causal attention.

**Limitations to Keep in Mind**
*   **No Training Guarantee:** Implementing the architecture does not guarantee SGD will find the universal approximator. The paper proves existence, not optimization convergence (Section 6.3).
*   **Compact Domain:** Ensure input embeddings are normalized or bounded. The theory assumes $\Omega$ is compact; unbounded embeddings violate the topological assumptions required for the Stone-Weierstrass theorem application.
*   **Head Configuration:** The proof uses $d_{head}=1$. Practical models use $d_{head} \geq 64$. While the theory suggests fixed width is possible, it does not prove that standard multi-head configurations ($d_{head} > 1$) achieve the same efficiency. Treat the head dimension constraint as a theoretical lower bound rather than a prescriptive design rule.

**Summary Recommendation**
Adopt the **measure-theoretic perspective** for thinking about context length. When asked "Can this model handle longer sequences?", the answer based on this paper is "Yes, provided it is deep enough and the target mapping is continuous," rather than "No, unless we widen the model." This shifts the engineering focus from memory-intensive width scaling to manageable depth scaling, aligning theoretical guarantees with practical constraints.
