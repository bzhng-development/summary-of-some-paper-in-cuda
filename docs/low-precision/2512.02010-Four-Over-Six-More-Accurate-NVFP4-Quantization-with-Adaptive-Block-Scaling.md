## 1. Executive Summary

This paper introduces **Four Over Six (4/6)**, an adaptive block scaling technique for **NVFP4** quantization that reduces error by dynamically choosing to scale 16-value blocks to a maximum of 4 instead of the standard 6, specifically targeting the high quantization error found in near-maximal values (around 5) inherent to the non-uniform step sizes of the **FP4 E2M1** format. By implementing this selection via Mean Squared Error (MSE) minimization with less than **15% computational overhead** on **NVIDIA Blackwell GPUs**, the method significantly improves model fidelity: it brings pre-training loss **22.3% closer to BF16** baselines for the **Nemotron 3 Nano 30B-A3B** architecture and consistently lowers perplexity across **Llama 3** and **Qwen 3** models when combined with post-training quantization methods like **AWQ** and **SmoothQuant**.

## 2. Context and Motivation

### The Drive Toward Lower Precision
The trajectory of Large Language Model (LLM) development is defined by a relentless increase in model size and training data volume. As models grow, the computational cost of training and inference becomes a primary bottleneck. Historically, the field has responded by progressively lowering numerical precision to reduce memory footprint and increase arithmetic throughput. The standard shifted from 32-bit floating point (`FP32`) to 16-bit (`FP16`), and currently, `BF16` (Bfloat16) is the dominant format for training stability. Recent research has pushed further into 8-bit formats like `FP8`, but the next logical frontier is **4-bit floating point (`FP4`)**.

The motivation for adopting `FP4` is twofold:
1.  **Memory Efficiency:** Reducing precision from 16 bits to 4 bits theoretically offers a $4\times$ reduction in memory bandwidth requirements and storage.
2.  **Compute Throughput:** Emerging hardware, specifically **NVIDIA Blackwell GPUs**, includes native tensor core support for `FP4` matrix multiplications. These operations are projected to be **4–6$\times$ faster** than their `BF16` counterparts.

However, simply switching to `FP4` is not viable. The `FP4` format (specifically `E2M1`, or 2 exponent bits and 1 mantissa bit) is extremely coarse, offering only **16 distinct representable values**: $\pm\{0, 0.5, 1, 1.5, 2, 3, 4, 6\}$. Quantizing high-precision tensors directly to this sparse set introduces catastrophic quantization error, destroying model accuracy.

### The Limitations of Block-Scaled Quantization
To mitigate the sparsity of `FP4`, the community adopted **block-scaled quantization**. Instead of applying a single global scale to an entire tensor, the tensor is divided into small blocks (e.g., 16 or 32 elements), and each block is assigned its own high-precision scale factor.
*   **MXFP4:** Uses a block size of 32 and an 8-bit integer scale (`E8M0`).
*   **NVFP4:** Uses a block size of 16 and an 8-bit floating-point scale (`FP8 E4M3`).

In this scheme, a block of values $X$ is quantized by first determining a scale factor $\Delta$ based on the maximum absolute value in the block ($X_{max}$), dividing the values by $\Delta$, rounding them to the nearest `FP4` value, and storing the result. Mathematically, for a block scaled to the maximum `FP4` value ($M_{FP4}=6$), the scale is typically derived as:

$$ \Delta = \frac{X_{max}}{6} $$

While block scaling expands the dynamic range significantly, it does not solve the fundamental issue of **non-uniform step sizes** inherent to floating-point formats. Unlike integer formats where the distance between representable values is constant, `FP4` has variable gaps:
*   Between 0 and 2, the step size is **0.5**.
*   Between 2 and 4, the step size is **1.0**.
*   Between 4 and 6, the step size is **2.0**.

This non-uniformity creates a specific vulnerability: **near-maximal values**. When a block is scaled such that its largest element maps to 6, the second-largest representable value is 4. This creates a massive "dead zone" between 66.6% and 100% of the block's dynamic range where no values can be represented. Any true value falling in this range (e.g., a value that should map to 5.0) must be rounded down to 4 or up to 6, introducing significant error. As shown in **Figure 1a** of the paper, this results in a spike in quantization error for values close to the block maximum.

### The Gap in Current Training Recipes
Current state-of-the-art methods for training with `NVFP4` attempt to patch these accuracy issues with heavy-handed algorithmic interventions, which introduce their own problems:
1.  **Computational Overhead:** Techniques like the **Random Hadamard Transform (RHT)** and **Stochastic Rounding (SR)** are required to smooth out outliers and reduce bias. However, these operations add latency. If the overhead becomes too high, the theoretical speedup of `FP4` matrix multiplication is negated, making `FP8` (which is more accurate and only $2\times$ slower than `BF16`) a more practical choice.
2.  **"Healing" Phases:** Many recipes require switching back to high-precision (`BF16` or `FP32`) weights and activations near the end of training to recover lost accuracy. This defeats the goal of **native `FP4` training**, resulting in a model that is not truly end-to-end low-precision and complicating the deployment pipeline.
3.  **Post-Training Quantization (PTQ) Limits:** Even for inference-only quantization, standard `NVFP4` fails to recover full `BF16` performance on large models (see **Table 3**), limiting its adoption for production systems where accuracy is paramount.

The core gap, therefore, is a lack of a **lightweight, algorithmic improvement** that addresses the specific structural weakness of `FP4` (the large step size near the maximum) without requiring expensive transforms or high-precision fallbacks.

### Positioning of Four Over Six (4/6)
This paper positions **Four Over Six (4/6)** as a fundamental modification to the `NVFP4` quantization algorithm itself, rather than an auxiliary training trick.

The key insight driving 4/6 is that the "dead zone" error described above is not inevitable. By adaptively choosing to scale certain blocks to a maximum of **4** instead of **6**, the distribution of representable values changes.
*   **Standard Scaling ($M=6$):** Representable relative values are $\{0, \dots, 4/6, 6/6\}$. The gap before the max is $2/6$ (33%).
*   **Adaptive Scaling ($M=4$):** Representable relative values become $\{0, \dots, 3/4, 4/4\}$. The gap before the max shrinks to $1/4$ (25%).

As illustrated in **Figure 1b** and **Table 1**, scaling a block to 4 allows the format to represent values at 75% of the maximum magnitude exactly (using the `FP4` value 3), whereas scaling to 6 forces a rounding error for values in that region.

Unlike prior works that apply global fixes or expensive rotations, 4/6 operates **locally and adaptively**:
*   It evaluates each 16-element block individually.
*   It computes the quantization error for both scaling options ($M=4$ and $M=6$).
*   It selects the scale that minimizes the Mean Squared Error (MSE) for that specific block.

This approach directly targets the root cause of `NVFP4` degradation—the rounding of near-maximal values—while maintaining compatibility with native hardware instructions on Blackwell GPUs. By solving the precision problem at the quantization definition level, 4/6 aims to enable efficient, native `FP4` training and inference without the need for "healing" phases or prohibitive computational overhead.

## 3. Technical Approach

This section details the mechanism of **Four Over Six (4/6)**, a modification to the NVFP4 quantization algorithm that adaptively selects block scaling factors to minimize quantization error. Unlike prior methods that apply a fixed scaling rule or rely on expensive pre-processing transforms, 4/6 operates as a lightweight, online decision process within the quantization kernel itself, choosing between two distinct scaling regimes for every block of data.

### 3.1 Reader orientation (approachable technical breakdown)
The system is a specialized quantization routine that runs on NVIDIA Blackwell GPUs, acting as a smart filter that decides whether to compress a small chunk of neural network data using a "wide" range (up to 6) or a "narrow" range (up to 4). It solves the problem of large rounding errors near the maximum values of a data block by dynamically switching to a narrower scale that offers finer resolution where the standard format has a dangerous gap.

### 3.2 Big-picture architecture (diagram in words)
The 4/6 architecture integrates directly into the data flow of a neural network layer, sitting between the high-precision tensor storage and the low-precision matrix multiplication units.
*   **Input Buffer:** Receives a block of 16 high-precision values (e.g., activations or gradients in BF16/FP32).
*   **Dual-Path Quantizer:** Simultaneously processes the block through two parallel logic paths: one calculating the quantization parameters for a maximum scale of 6 ($M=6$) and another for a maximum scale of 4 ($M=4$).
*   **Error Estimator:** Dequantizes the results from both paths back to high precision and computes the Mean Squared Error (MSE) against the original input values for each path.
*   **Selector Switch:** Compares the two error metrics and outputs a single binary flag (4 or 6) along with the corresponding quantized values and scale factor.
*   **Output Buffer:** Stores the chosen 4-bit values, the 8-bit scale factor, and the selection flag for use in subsequent matrix multiplications.

### 3.3 Roadmap for the deep dive
*   First, we define the mathematical foundation of standard NVFP4 quantization to establish the baseline from which 4/6 deviates.
*   Second, we explain the core "dual-scale" mechanism, detailing how the algorithm generates two candidate representations for every data block.
*   Third, we describe the selection criterion (MSE minimization) and the specific rules used to break ties or handle edge cases.
*   Fourth, we address the critical modification to the global tensor scaling factor required to prevent numerical overflow when mixing scales.
*   Finally, we detail the hardware-level implementation on NVIDIA Blackwell GPUs, explaining how the method achieves low latency using register-file operations.

### 3.4 Detailed, sentence-based technical breakdown

#### Foundation: Standard NVFP4 Quantization
To understand the innovation of 4/6, one must first understand the rigid structure of the standard **NVFP4** format it modifies. NVFP4 is a block-scaled quantization scheme where a high-precision tensor $X$ is divided into blocks of $m=16$ elements. Each block $i$ is assigned a scale factor $\Delta_i$ stored in **FP8 E4M3** format, and the entire tensor shares a global scale $\alpha$ stored in **FP32**.

In the standard approach, the scaling is determined solely by the maximum absolute value in the block, denoted as $X_{max}^{(i)}$. The goal is to map this maximum value to the largest representable number in the **FP4 E2M1** format, which is $M_{FP4} = 6$. The block scale $\Delta_i$ is calculated as:

$$ \Delta_i = \frac{\max(|X_{16i \dots 16(i+1)}|)}{\alpha \cdot M_{FP4}} $$

Once the scale is set, each element $x$ in the block is normalized by dividing by $\alpha \Delta_i$ and then rounded to the nearest representable FP4 value. The set of positive representable values in FP4 E2M1 is $\{0, 0.5, 1, 1.5, 2, 3, 4, 6\}$. The rounding function $\lceil \cdot \rfloor$ maps the normalized value to this discrete set. As established in the motivation, this creates a large gap between 4 and 6; any normalized value between roughly 4.6 and 6.0 is forced to round to either 4 or 6, introducing significant error.

#### The Dual-Scale Mechanism
The core innovation of **Four Over Six** is the rejection of the assumption that every block must be scaled to the maximum possible range ($M=6$). Instead, the algorithm hypothesizes that for many blocks, scaling the maximum value to $M=4$ yields a lower total quantization error, despite the reduced dynamic range.

The system implements this by performing a "what-if" analysis for every single block during the quantization process. For a given block of input values $X$, the algorithm executes two distinct quantization passes in parallel:
1.  **Pass A (Standard):** Computes the scale $\Delta^{(6)}$ assuming the block maximum maps to 6. It generates a set of quantized values $\bar{X}^{(6)}$.
2.  **Pass B (Adaptive):** Computes a new scale $\Delta^{(4)}$ assuming the block maximum maps to 4. It generates a second set of quantized values $\bar{X}^{(4)}$.

Mathematically, the alternative scale $\Delta^{(4)}$ is derived by replacing $M_{FP4}$ with 4 in the scaling equation:

$$ \Delta^{(4)}_i = \frac{\max(|X_{16i \dots 16(i+1)}|)}{\alpha \cdot 4} $$

Because the denominator is smaller (4 instead of 6), the resulting scale factor $\Delta^{(4)}$ is larger than $\Delta^{(6)}$ by a factor of $1.5$ ($\frac{6}{4}$). This larger scale means that the step sizes in the quantized domain effectively become finer relative to the original input values. Specifically, the FP4 value `3` now represents 75% of the block maximum, whereas in the standard scheme, the value `4` represents only 66.6% of the maximum. This shift allows the $M=4$ scheme to accurately represent values that would otherwise fall into the large error gap of the $M=6$ scheme.

**Table 2** in the paper provides a concrete numerical example of this mechanism. Consider a block with values $[10, 20, 30, 40]$.
*   **Under $M=6$:** The scale is determined by 40 mapping to 6. The value 30 normalizes to $4.62$, which rounds down to the FP4 value `4`. When dequantized, this becomes $26$, resulting in a squared error of $(30-26)^2 = 16$ for that single element.
*   **Under $M=4$:** The scale is determined by 40 mapping to 4. The value 30 normalizes to exactly $3.0$, which maps to the FP4 value `3`. When dequantized, this returns exactly $30$, resulting in zero error for that element.

The algorithm performs this calculation for all 16 elements in the block, generating two complete candidate representations.

#### Selection Criterion: Mean Squared Error Minimization
Once the two candidate quantizations ($\bar{X}^{(6)}$ and $\bar{X}^{(4)}$) are generated, the system must decide which one to store. The paper evaluates three potential selection rules: minimizing the maximum absolute error (L-infinity norm), minimizing the sum of absolute errors (L1 norm), and minimizing the Mean Squared Error (MSE).

The authors determine that **MSE minimization** is the most effective criterion for overall model performance. The error for each candidate is computed by dequantizing the FP4 values back to high precision and comparing them to the original input $X$. The MSE for a block under scale $M$ is defined as:

$$ E^{(M)} = \frac{1}{n} \sum_{j=1}^{n} \left( (\bar{X}^{(M)}_j \cdot \Delta^{(M)} \cdot \alpha) - X_j \right)^2 $$

where $n=16$ is the block size. The algorithm then applies a simple conditional selection:

$$ \text{Choose } M=4 \quad \text{if} \quad E^{(4)} < E^{(6)} $$
$$ \text{Choose } M=6 \quad \text{otherwise} $$

**Table 4** demonstrates that while all three rules offer improvements over standard NVFP4, the MSE rule consistently yields the lowest perplexity across Llama 3 and Qwen 3 models. This suggests that minimizing the average squared deviation is more critical for maintaining the statistical properties of weights and gradients than strictly bounding the worst-case single-element error.

The output of this stage is a tuple containing the chosen quantized values, the chosen scale factor, and a single bit indicating which scale was selected (4 or 6). This bit is crucial for the dequantization process during matrix multiplication, as the hardware must know whether to interpret the scale factor relative to a max value of 4 or 6.

#### Global Tensor Scale Adjustment
A subtle but critical design choice in 4/6 involves the computation of the global tensor scale factor $\alpha$. In standard NVFP4, $\alpha$ is calculated to ensure that the largest value in the entire tensor, when multiplied by the maximum block scale and the maximum FP4 value (6), does not overflow the FP8 scale format.

However, 4/6 introduces a constraint: to select the $M=4$ scale for a block, the required block scale factor $\Delta^{(4)}$ must be representable in **FP8 E4M3**. Since $\Delta^{(4)} = 1.5 \times \Delta^{(6)}$, there is a risk that for blocks containing the tensor's absolute maximum values, the required scale for the $M=4$ option would exceed the maximum representable value in FP8 E4M3 (which is 448).

Specifically, if the standard calculation allows a block scale up to 448 for the $M=6$ case, the corresponding $M=4$ scale would need to be $448 \times 1.5 = 672$. Since 672 cannot be represented in FP8 E4M3, the algorithm would be unable to choose the $M=4$ option for these critical blocks, forcing them back to $M=6$ and losing the potential accuracy gain.

To resolve this, the authors modify Equation 1 for the global tensor scale $\alpha$. Instead of using the maximum FP8 value ($M_{FP8} = 448$) in the denominator, they cap the effective range at **256**. The value 256 is chosen because $256 \times 1.5 = 384$, and 384 is a representable value in FP8 E4M3. By restricting the global scale such that the maximum required block scale for the $M=6$ case is 256, the system guarantees that the corresponding $M=4$ scale (384) will always fit within the FP8 format. This ensures that the adaptive selection logic remains valid for all blocks, including those with the largest magnitudes in the tensor.

#### Hardware Implementation and Efficiency
The theoretical benefits of 4/6 would be nullified if the dual-path computation introduced excessive latency. The authors emphasize that 4/6 is designed for efficient execution on **NVIDIA Blackwell GPUs**, which support specific PTX (Parallel Thread Execution) instructions for low-precision formats.

The implementation utilizes the `cvt` (convert) family of instructions to perform the quantization from high precision to packed FP4 and the subsequent dequantization back to FP16 (required for error calculation). The entire 4/6 logic—computing both scales, quantizing both ways, dequantizing, calculating MSE, and selecting the winner—is implemented within a single **CUDA kernel**.

Crucially, all intermediate values (the two sets of quantized values, the two sets of dequantized values, and the error accumulators) are kept entirely in the GPU's **register file**. This avoids costly reads and writes to global memory or shared memory, which would otherwise dominate the runtime. Because the block size is small (16 elements), the register pressure is manageable, and the operations are highly parallelizable.

The paper reports that this online adaptive process adds less than **15% overhead** to the standard NVFP4 quantization operation. This overhead is considered minimal compared to the alternative "healing" phases or the computational cost of Random Hadamard Transforms used in other recipes. Furthermore, the authors note that the resulting quantized model is **natively NVFP4**; once the quantization decision is made and stored, the subsequent matrix multiplications (GEMM) proceed at full hardware speed without any additional modification, as the decoder simply reads the extra selection bit to interpret the scale correctly.

In the context of training (as shown in **Figure 3**), this kernel is inserted into the forward and backward passes. During the forward pass, activations are quantized with 4/6 before the GEMM. During the backward pass, gradients are quantized with 4/6 (often combined with Stochastic Rounding) before being used for weight updates. The weights themselves are stored in FP32 but quantized to NVFP4 with 4/6 on-the-fly or offline for the weight gradient calculation, ensuring that all three operands (weights, activations, gradients) benefit from the reduced quantization error.

## 4. Key Insights and Innovations

The **Four Over Six (4/6)** method introduces a paradigm shift in how low-precision floating-point quantization is approached, moving from rigid, static scaling rules to dynamic, error-aware adaptation. While prior work focused on mitigating outliers through data transformation or accepting fixed quantization gaps, 4/6 fundamentally alters the definition of the quantization block itself. The following insights distinguish this work as a foundational innovation rather than an incremental tweak.

### 1. Adaptive Block Scaling as a First-Class Primitive
Prior block-scaled formats like **MXFP4** and standard **NVFP4** operate on a "one-size-fits-all" principle: every block is scaled to maximize the dynamic range of the specific low-precision format (mapping the block maximum to the format's maximum value, e.g., 6 for FP4). This assumes that maximizing range is always optimal. **4/6 challenges this assumption**, demonstrating that for a significant portion of blocks, sacrificing range for resolution yields lower total error.

*   **Differentiation:** Unlike **SmoothQuant** or **AWQ**, which apply global or per-channel smoothing factors to shift outliers before quantization, 4/6 operates at the **block level (16 elements)** and makes a discrete binary decision for *each* block independently. It does not transform the data distribution; it transforms the *quantization grid* to better fit the local data distribution.
*   **Significance:** This converts the quantization step from a deterministic mapping into an optimization problem solved locally. By allowing the scale factor to represent a maximum of 4 instead of 6, the method effectively "zooms in" on the upper quartile of the data distribution where standard FP4 has its largest blind spot (the gap between 4 and 6). As shown in **Table 3**, blindly applying $M=4$ globally degrades performance due to overflow, while blindly applying $M=6$ suffers from rounding errors. 4/6 is the first method to successfully navigate this trade-off dynamically, unlocking accuracy that neither static strategy can achieve alone.

### 2. Targeting the "Near-Maximal" Error Regime
A critical theoretical contribution of this work is the identification and isolation of **near-maximal values** as the primary source of performance degradation in FP4 training, distinct from the more commonly cited problem of outliers.

*   **Differentiation:** Most quantization literature focuses on **outliers** (extreme values that force large scale factors, compressing the rest of the data into noise). Techniques like **Random Hadamard Transform (RHT)** are designed specifically to spread these outliers across channels. However, **Figure 2b** reveals a counter-intuitive finding: keeping values below 4 in high precision recovers almost all performance, while quantizing values between 4 and 6 causes a catastrophic drop. This indicates that the **non-uniform step size** of FP4 (a jump of 2.0 between 4 and 6) is a more damaging structural flaw than the presence of outliers, which block scaling already handles well.
*   **Significance:** This insight redirects optimization efforts. Instead of spending compute budget on complex transforms to hide outliers (which adds latency), 4/6 directly patches the structural hole in the FP4 format. It proves that the "dead zone" between 66% and 100% of the block range is the true bottleneck for model fidelity. By specifically targeting this regime, 4/6 achieves **22.3% closer loss to BF16** during pre-training (**Figure 4**) without needing the heavy "healing" phases required by previous FP4 recipes.

### 3. Native Hardware Compatibility with Minimal Overhead
Perhaps the most pragmatic innovation is the demonstration that adaptive quantization can be implemented with **negligible latency overhead** on existing hardware, avoiding the need for custom silicon or software-emulated arithmetic that plagues many low-precision proposals.

*   **Differentiation:** Many advanced quantization schemes rely on software simulation of non-standard formats or expensive pre-processing steps (like RHT) that break the efficiency of native tensor cores. In contrast, 4/6 leverages the specific capabilities of **NVIDIA Blackwell GPUs**. It uses standard `cvt` (convert) instructions and keeps all dual-path calculations within the **register file**, avoiding slow global memory access.
*   **Significance:** The paper reports an overhead of **less than 15%** for the quantization operation itself. Crucially, this overhead is incurred only during the *quantization* phase (forward pass for activations, backward pass for gradients). The resulting model is **natively NVFP4**; the stored weights and activations use the standard hardware format, distinguished only by a single metadata bit per block. This means the subsequent matrix multiplications (GEMM) run at full **4–6$\times$ speedup** over BF16 with **zero modification** to the compute kernels. This preserves the primary economic incentive for using FP4 (throughput) while fixing its accuracy deficit, a balance previous methods failed to strike.

### 4. The "Overflow-Safe" Global Scale Modification
A subtle but essential innovation is the modification of the global tensor scale factor $\alpha$ to guarantee the feasibility of the adaptive choice.

*   **Differentiation:** In standard quantization, the global scale is set tightly against the maximum representable value of the scale format (FP8 E4M3 max = 448). 4/6 identifies that this tight packing prevents the $M=4$ option from being viable for blocks containing the tensor's absolute maximum values, because the required scale factor ($1.5\times$ larger) would overflow the FP8 format.
*   **Significance:** By intentionally under-utilizing the FP8 range (capping the effective max at **256** instead of 448, as detailed in **Section 3.1**), the authors create "headroom" that ensures the $M=4$ option is *always* a valid candidate, even for the largest values in the network. This design choice prevents the adaptive mechanism from silently failing on critical layers, ensuring the theoretical benefits of dual-scale selection are realized in practice across the entire model depth. It represents a systems-level awareness that algorithmic flexibility requires careful memory layout planning.

## 5. Experimental Analysis

This section dissects the empirical evidence provided in the paper to validate the **Four Over Six (4/6)** method. The authors employ a rigorous two-pronged evaluation strategy: **pre-training experiments** to test stability and convergence during model creation, and **post-training quantization (PTQ)** experiments to assess accuracy recovery on established models. The analysis relies on specific metrics—training loss curves, word perplexity, and downstream task accuracy—to demonstrate that 4/6 consistently narrows the performance gap between ultra-low-precision `NVFP4` and high-precision `BF16` baselines.

### 5.1 Evaluation Methodology

The experimental design is structured to isolate the impact of adaptive block scaling across different model scales, architectures, and training phases.

**Datasets and Metrics:**
*   **Pre-training Data:** Models are trained on a curated distribution of **1 trillion tokens** comprising high-quality and synthetic data, following the setup in reference [24].
*   **Perplexity Metrics:** For PTQ evaluation, the primary metric is **word-level perplexity** on two standard benchmarks: **WikiText-2** and **C4**. Lower perplexity indicates better probability distribution modeling.
*   **Downstream Tasks:** To ensure perplexity improvements translate to real-world utility, models are evaluated on four reasoning and comprehension tasks: **BoolQ** (boolean questions), **ARC-Easy** and **ARC-Challenge** (science questions), and **HellaSwag** (commonsense inference). Results are reported as **normalized accuracy** to account for tokenization differences.

**Model Architectures:**
*   **Pre-training:** The authors utilize the **Nemotron 3 Nano 30B-A3B** architecture. This is a hybrid Mixture-of-Experts (MoE) model with **30 billion total parameters**, of which only **3 billion are active** per forward pass. It consists of 52 blocks: 6 Self-Attention, 23 MoE, and 23 Mamba-2 blocks.
*   **Post-Training:** Evaluation covers a wide range of open-weight models to test generalizability: **Llama 3** (1B, 8B, 70B) and **Qwen 3** (1.7B, 8B, 32B).

**Baselines and Comparators:**
The experiments compare 4/6 against three tiers of baselines:
1.  **High-Precision Baseline:** Models kept entirely in **BF16** (or FP32 for weights), representing the "gold standard" of accuracy.
2.  **Standard NVFP4:** The state-of-the-art block-scaled quantization using a fixed maximum scale of $M=6$.
3.  **Alternative Static Scales:** A variant where *all* blocks are forced to scale to $M=4$ (demonstrating why adaptive selection is necessary).
4.  **Advanced PTQ Methods:** For post-training, 4/6 is combined with leading algorithms: **GPTQ**, **AWQ**, and **SmoothQuant**, as well as simple Round-to-Nearest (RTN).

**Hardware and Training Setup:**
Pre-training experiments were conducted on **384 NVIDIA B200 GPUs** (180GB memory each), leveraging the native `FP4` tensor core support of the Blackwell architecture. The training recipe follows the state-of-the-art `NVFP4` protocol:
*   **Optimizer:** AdamW with $\beta_1=0.9$, $\beta_2=0.95$, weight decay $0.1$.
*   **Schedule:** Warmup-Stable-Decay (WSD) with a peak learning rate of $10^{-3}$ decaying to $10^{-5}$.
*   **Precision Mix:** While matrix multiplications (GEMM) are performed in `NVFP4`, sensitive components (Attention, normalization, non-linearities, and the output head) are kept in high precision (`BF16`/`FP32`). Gradients use **Stochastic Rounding (SR)**, and weight gradients utilize a **Random Hadamard Transform (RHT)**.

### 5.2 Pre-Training Performance: Closing the Gap to BF16

The primary claim of the paper is that 4/6 enables more stable and accurate training in `FP4` compared to existing recipes. The results in **Figure 4** provide direct evidence for this.

**Quantitative Results:**
*   **Training Loss Convergence:** As shown in **Figure 4**, the training loss curve for **NVFP4 (4/6)** tracks significantly closer to the **BF16** baseline than the standard **NVFP4** curve.
*   **Gap Reduction:** The authors quantify this improvement precisely: adding 4/6 brings the training loss **22.3% closer to BF16 performance** compared to the standard NVFP4 recipe.
*   **Stability:** The curve for standard NVFP4 shows a persistent divergence from the BF16 baseline throughout the 1 trillion token training run, indicating that standard quantization error accumulates or prevents the model from finding optimal minima. In contrast, the 4/6 curve maintains a tight bound near the high-precision limit.

**Interpretation:**
This result is critical because it validates the hypothesis from Section 2.2: the error introduced by rounding near-maximal values (the "dead zone" between 4 and 6) is not just a minor noise source but a fundamental barrier to convergence. By adaptively removing this barrier, 4/6 allows the optimizer to behave much more like it would in high precision. Crucially, this is achieved *without* the "healing" phase (switching back to BF16 at the end of training) required by other methods, proving that **native FP4 training** is viable with the right scaling strategy.

### 5.3 Post-Training Quantization: Perplexity Analysis

For inference, the goal is to quantize pre-trained models with minimal accuracy loss. The paper presents extensive perplexity results in **Table 3**, **Table 4**, and **Table 5**.

**The Necessity of Adaptivity (Table 3):**
Before claiming 4/6 works, the authors prove that a static switch to $M=4$ is detrimental. **Table 3** compares `NVFP4 (M=6)` against a hypothetical `NVFP4 (M=4)` where *every* block is scaled to 4.
*   **Llama 3 8B (WikiText-2):** Standard NVFP4 achieves **8.43** perplexity. The static $M=4$ variant degrades to **8.63**.
*   **Qwen 3 32B (WikiText-2):** Standard NVFP4 is **9.85**, while static $M=4$ worsens to **10.57**.
*   **Conclusion:** Blindly reducing the range causes overflow for large values, increasing error. This confirms that the *adaptive* selection mechanism is the key innovation, not the $M=4$ scale itself.

**Optimizing the Selection Rule (Table 4):**
The authors ablate the decision criterion used to choose between $M=4$ and $M=6$. They test three rules: Max Error (L-infinity), L1 Norm, and Mean Squared Error (MSE).
*   **Result:** **MSE minimization** consistently yields the best perplexity.
*   **Example (Llama 3 8B, WikiText-2):**
    *   Standard RTN: **8.43**
    *   + 4/6 (Max Error): **8.36**
    *   + 4/6 (L1): **8.33**
    *   **+ 4/6 (MSE): 8.30** (Best)
*   This suggests that minimizing the *average* squared deviation is more important for language modeling than strictly bounding the single worst error, likely because large outliers are already handled well by the block scaling, while the bulk of the distribution benefits from average error reduction.

**Integration with PTQ Methods (Table 5):**
The most compelling evidence for 4/6's utility is its ability to boost existing state-of-the-art PTQ methods. **Table 5** details the performance when 4/6 is layered on top of RTN, GPTQ, AWQ, and SmoothQuant.

*   **AWQ + 4/6:** This combination delivers the strongest overall performance.
    *   **Llama 3 8B (WikiText-2):** Improves from **8.33** (AWQ) to **8.24** (AWQ + 4/6).
    *   **Qwen 3 8B (WikiText-2):** Improves from **12.68** to **12.57**.
    *   **Aggregate Gain:** The paper states that AWQ + 4/6 brings perplexity **19.9% closer to BF16** compared to standard AWQ.
*   **SmoothQuant + 4/6:** Shows consistent, albeit smaller, gains.
    *   **Llama 3 8B:** Improves from **8.38** to **8.32**.
    *   **Aggregate Gain:** Brings performance **5.3% closer to BF16**.
*   **GPTQ + 4/6:** Results are mixed.
    *   **Llama 3 8B:** Improves slightly from **8.33** to **8.30**.
    *   **Qwen 3 8B:** Degrades slightly from **12.50** to **12.65**.
    *   **Analysis:** The authors note that their implementation uses the standard GPTQ optimization loop but swaps in the 4/6 linear layer for measurement. They hypothesize that fully integrating 4/6 into the GPTQ *optimization process* (rather than just the measurement) would yield better results. Currently, the mismatch between the Hessian approximation in GPTQ and the dynamic scaling of 4/6 may cause suboptimal weight updates in some cases.
*   **RTN + 4/6:** Even without advanced calibration, simple Round-to-Nearest benefits significantly.
    *   **Llama 3 8B:** Drops from **8.43** to **8.30**, a substantial recovery given the simplicity of the method.

### 5.4 Downstream Task Accuracy

Perplexity is a proxy metric; the ultimate test is task performance. **Table 6** (Llama 3) and **Table 7** (Qwen 3) report normalized accuracy on BoolQ, ARC, and HellaSwag.

**Key Findings:**
*   **Consistent Improvement:** In nearly every case, adding 4/6 improves or matches the baseline PTQ method's accuracy.
*   **Llama 3 8B Average Accuracy:**
    *   **RTN:** Increases from **72.0%** to **72.2%**.
    *   **AWQ:** Increases from **72.2%** to **73.1%** (a **0.9%** absolute gain, which is significant at this performance level).
    *   **SmoothQuant:** Increases from **72.3%** to **72.9%**.
*   **Qwen 3 8B Average Accuracy:**
    *   **RTN:** Increases from **72.3%** to **72.9%**.
    *   **AWQ:** Increases from **73.4%** to **73.5%**.
    *   **GPTQ:** Increases from **73.4%** to **73.9%**.
*   **Specific Task Wins:** On **BoolQ** for Llama 3 8B with AWQ, accuracy jumps from **81.3%** to **80.4%**? Wait, checking **Table 6**:
    *   AWQ Base: **81.3%**
    *   AWQ + 4/6: **80.4%** (This is a decrease).
    *   *Correction:* Looking closely at **Table 6**, the AWQ + 4/6 row for Llama 3 8B shows **80.4%** on BoolQ, which is lower than the base **81.3%**. However, it gains on Arc-E (**78.4** $\to$ **80.2**) and Arc-C (**51.7** $\to$ **53.6**). The *average* still increases (**72.2** $\to$ **73.1**). This highlights a trade-off: 4/6 may shift error distribution, helping reasoning tasks (ARC) while occasionally dipping on specific yes/no queries (BoolQ), but the net effect is positive.
*   **HellaSwag Stability:** HellaSwag scores remain remarkably stable or improve. For Llama 3 8B AWQ, it moves from **77.5%** to **78.2%**.

### 5.5 Critical Assessment and Limitations

Do the experiments convincingly support the claims? **Yes, with minor caveats.**

**Strengths of the Evidence:**
1.  **Pre-training Validity:** The **22.3%** gap reduction in **Figure 4** is a strong, unambiguous result. It demonstrates that 4/6 solves a fundamental convergence issue in low-precision training that previous methods only patched with expensive "healing."
2.  **Ablation Rigor:** **Table 3** effectively kills the counter-argument that "just using a smaller scale is better." It proves the *adaptivity* is essential. **Table 4** justifies the choice of MSE over other norms with concrete numbers.
3.  **Generalizability:** The method works across model families (Llama, Qwen), sizes (1B to 70B), and architectures (Transformer, MoE, Mamba), suggesting the solution addresses a universal property of FP4 quantization rather than overfitting to a specific model.

**Limitations and Mixed Results:**
1.  **GPTQ Integration:** The degradation seen in **Table 5** for Qwen 3 with GPTQ (+4/6) and the mixed BoolQ results in **Table 6** indicate that 4/6 is not a "drop-in" magic bullet for *every* existing algorithm without modification. The authors explicitly acknowledge this in Section 4.2, noting that modifying the GPTQ *optimization objective* to account for dual-scale selection is future work. Currently, it acts as a better quantizer, but not necessarily a better optimizer for second-order methods.
2.  **Overhead Context:** While **&lt;15%** overhead is low, it is not zero. In latency-critical inference scenarios where every microsecond counts, this overhead during the quantization/calibration phase (for PTQ) or the forward/backward pass (for training) must be weighed against the accuracy gains. However, since the *inference* GEMM itself remains unchanged and fast, the training overhead is the primary cost, which the 22.3% accuracy gain likely justifies.
3.  **Missing Large-Scale GPTQ Data:** **Table 5** omits results for **Llama-3.1-70B** with GPTQ due to memory constraints. Given that larger models are often more sensitive to quantization, the absence of this specific data point leaves a small gap in the "largest model" validation for the GPTQ combination, though the 32B Qwen results provide a reasonable proxy.

**Conclusion on Experiments:**
The experimental analysis is robust. The authors successfully demonstrate that **Four Over Six** is a superior quantization primitive. It transforms `NVFP4` from a format that requires heavy algorithmic crutches (RHT, healing) into a viable standalone format for training, and it acts as a powerful "force multiplier" for post-training methods like AWQ. The specific numerical gains—particularly the **8.30** vs **8.43** perplexity on Llama 3 8B and the **22.3%** pre-training gap closure—provide concrete proof that addressing the non-uniform step sizes of FP4 via adaptive scaling is the correct path forward for 4-bit LLMs.

## 6. Limitations and Trade-offs

While **Four Over Six (4/6)** demonstrates significant improvements in quantization accuracy and training stability, it is not a universal solution for all low-precision scenarios. The method introduces specific architectural dependencies, incurs non-zero computational costs, and faces theoretical boundaries regarding format compatibility. Understanding these limitations is crucial for determining when 4/6 is the appropriate tool versus when alternative strategies (such as integer quantization or different floating-point formats) might be superior.

### 6.1 Dependency on Scale Factor Precision
The most fundamental constraint of 4/6 is its reliance on the precision of the block scale factor format. The mechanism works by toggling between two scaling regimes: mapping the block maximum to $M=6$ or $M=4$. Switching to the $M=4$ regime requires the block scale factor $\Delta$ to increase by a factor of $1.5$ (since $\Delta \propto 1/M$).

*   **The Precision Bottleneck:** This $1.5\times$ scaling requires the scale factor format to have sufficient resolution to represent the intermediate values between the standard $M=6$ scales. **NVFP4** uses **FP8 E4M3** for scales, which has enough granularity to represent both the original scale and the $1.5\times$ larger scale without excessive rounding error.
*   **Incompatibility with MXFP4:** The paper explicitly states in **Section 5.2** that 4/6 **cannot be applied to MXFP4**. MXFP4 uses **FP8 E8M0** (an 8-bit integer-like format with only exponent bits) for scaling. In E8M0, representable values are powers of 2 (e.g., 1, 2, 4, 8...). There is no representable value between $2^k$ and $2^{k+1}$ that corresponds to a $1.5\times$ increase. Attempting to scale an MXFP4 block by $1.5$ would force the scale to round to the nearest power of 2, effectively nullifying the adaptive benefit or introducing massive new errors.
*   **Implication:** This limits the portability of 4/6. It is strictly tied to hardware and formats that utilize floating-point scale factors with mantissa bits (like E4M3). As the field explores other microscaling formats, 4/6 will only be viable if the scale format offers fine-grained resolution, not just dynamic range.

### 6.2 Computational Overhead and Latency
Although the authors characterize the overhead as "minimal," it is not zero. The core mechanism requires performing **two full quantization passes** (one for $M=6$, one for $M=4$), dequantizing both results, computing the Mean Squared Error (MSE) for each, and executing a comparison logic for every single block of 16 elements.

*   **Quantified Overhead:** The paper reports an overhead of **less than 15%** to the NVFP4 quantization operation (**Section 3.2**). While this is achieved by keeping all intermediate data in the GPU register file to avoid memory bottlenecks, it represents a tangible increase in instruction count and register pressure.
*   **Training vs. Inference Impact:**
    *   **Training:** During pre-training, this 15% overhead is applied to every forward and backward pass for activations and gradients. For massive training runs spanning months, this accumulates to significant compute time. The trade-off is whether the **22.3% reduction in loss gap** (Figure 4) justifies the extended training duration. For many research groups, the answer is yes, as it avoids the need for a separate "healing" phase. However, for groups with strict time-to-solution constraints, this constant tax might be prohibitive compared to simpler, albeit less accurate, static scaling.
    *   **Inference (PTQ):** For Post-Training Quantization, the overhead is a one-time cost during the calibration/quantization phase. Since inference itself runs at native NVFP4 speeds (zero overhead during the actual matrix multiplication), this trade-off is generally favorable. However, for applications requiring **online quantization** (e.g., quantizing activations on-the-fly in a dynamic graph without pre-calibration), the 15% latency penalty per token could degrade real-time throughput.

### 6.3 Diminishing Returns with Higher Precision
The utility of adaptive block scaling is inversely proportional to the precision of the underlying value format. The "dead zone" error that 4/6 solves is a specific artifact of the extremely coarse **FP4 E2M1** format, which has a step size of 2.0 between values 4 and 6.

*   **The Precision Ceiling:** As the value format becomes more precise (e.g., moving to **FP6** or **FP8** for values), the step sizes become smaller and more uniform. In an FP8 format, the gap between the largest and second-largest representable values is negligible compared to FP4. Consequently, the benefit of dynamically shrinking the scale to "zoom in" on the top end of the range diminishes rapidly.
*   **Design Choice:** The authors note in **Section 5.2** that "benefits fade quickly as the precision used to store values increases." This implies that 4/6 is a specialized patch for the unique sparsity of 4-bit floating point. It is not a generalizable technique for all low-precision quantization; if hardware evolves to natively support efficient FP6 or denser FP8 formats, the complexity of 4/6 may no longer be justified.

### 6.4 Integration Challenges with Second-Order Methods
While 4/6 improves simple quantization methods (RTN) and activation-aware methods (AWQ, SmoothQuant), its interaction with second-order optimization methods like **GPTQ** reveals friction.

*   **The Mismatch:** GPTQ relies on approximating the Hessian matrix (second-order derivatives) to determine optimal weight updates that minimize reconstruction error. This process assumes a static quantization function. However, 4/6 introduces a **non-linear, data-dependent decision boundary**: the quantization function for a specific weight changes dynamically based on whether the local block error is lower with $M=4$ or $M=6$.
*   **Observed Degradation:** In **Table 5**, combining 4/6 with GPTQ yields mixed results. For **Qwen 3 8B**, the perplexity actually worsens slightly (from **12.50** to **12.65** on WikiText-2). The authors hypothesize in **Section 4.2** that this is because the current implementation applies 4/6 only during the *measurement* phase, not within the GPTQ *optimization loop* itself.
*   **Open Question:** To fully unlock the potential of 4/6 with GPTQ, the optimization algorithm would need to be rewritten to account for the discrete switch between scaling regimes. Until such an integration is developed, 4/6 cannot be blindly assumed to improve all existing PTQ pipelines, particularly those relying on complex second-order statistics.

### 6.5 Global Scale Headroom Trade-off
A subtle but necessary trade-off involves the **global tensor scale factor** ($\alpha$). To ensure that the $M=4$ option is always mathematically viable (i.e., the required scale factor doesn't overflow the FP8 format), the authors deliberately restrict the maximum usable value of the FP8 scale.

*   **The Constraint:** Standard NVFP4 utilizes the full range of FP8 E4M3 up to **448**. 4/6 caps the effective maximum at **256** (**Section 3.1**). This is because $256 \times 1.5 = 384$, which is representable, whereas $448 \times 1.5 = 672$, which overflows.
*   **The Cost:** By capping the global scale at 256, the method sacrifices approximately **43% of the dynamic range** available to the scale factors ($448/256 = 1.75$).
*   **Risk Scenario:** In scenarios where a tensor contains extreme outliers that require the full range of the FP8 format to prevent saturation, 4/6 forces a compression of the entire tensor to fit within the reduced headroom. While the paper argues in **Section 5.1** that block scaling already handles outliers well, there remains a theoretical risk that in models with pathological outlier distributions, this reduced headroom could force smaller values into higher quantization noise floors to accommodate the largest values, potentially offsetting the gains from the adaptive scaling.

### 6.6 Summary of Trade-offs

| Feature | Benefit of 4/6 | Cost / Limitation |
| :--- | :--- | :--- |
| **Accuracy** | Reduces quantization error for near-maximal values; closes 22.3% of BF16 gap in training. | Requires specific FP8 E4M3 scale format; incompatible with MXFP4 (E8M0). |
| **Compute** | Enables native FP4 training without "healing" phases. | Adds **&lt;15%** overhead to quantization ops; increases register pressure. |
| **Compatibility** | Works seamlessly with AWQ, SmoothQuant, and RTN. | Mixed results with GPTQ; requires algorithmic re-engineering for second-order methods. |
| **Dynamic Range** | Improves resolution in the critical 66%-100% range of blocks. | Sacrifices ~43% of global scale headroom (cap at 256 vs 448) to prevent overflow. |
| **Scalability** | Effective for FP4; benefits fade for higher precisions (FP6/FP8). | Not a future-proof solution for formats denser than 4-bit. |

In conclusion, **Four Over Six** is a highly effective, targeted optimization for the specific structural weaknesses of **NVFP4** on **Blackwell GPUs**. However, it is not a free lunch: it demands specific hardware features (FP8 E4M3 scales), accepts a modest compute tax, and requires careful integration when paired with advanced optimization-based quantization methods. Its success relies on the bet that the accuracy gains from fixing the "dead zone" outweigh the loss of global dynamic range and the added computational complexity.

## 7. Implications and Future Directions

The introduction of **Four Over Six (4/6)** represents a pivotal shift in the trajectory of low-precision deep learning, moving the field away from treating quantization error as an unavoidable penalty to be mitigated by expensive external transforms, and toward treating it as a solvable structural problem within the numerical format itself. By demonstrating that a simple, adaptive change to block scaling can recover **22.3% of the performance gap** to high-precision baselines during pre-training (**Figure 4**) and consistently boost post-training accuracy, this work fundamentally alters the viability of **native FP4 training**.

### 7.1 Reshaping the Low-Precision Landscape
Historically, the path to 4-bit training has been blocked by the "accuracy wall" of coarse formats like **FP4 E2M1**. Previous solutions attempted to climb over this wall using heavy algorithmic crutches: **Random Hadamard Transforms (RHT)** to smooth outliers, **Stochastic Rounding (SR)** to reduce bias, and **"healing" phases** where models revert to high precision at the end of training. These approaches often negated the theoretical speed benefits of FP4 or resulted in non-native models that were difficult to deploy.

**4/6 changes this landscape by lowering the wall itself.**
*   **From Mitigation to Correction:** Instead of hiding the non-uniform step sizes of FP4 through data rotation, 4/6 directly corrects the representation error caused by the large gap between values 4 and 6. This proves that **algorithmic intelligence at the quantization kernel level** can be more effective than global data transformations.
*   **Viability of Native Training:** The results suggest that **native end-to-end FP4 training** is achievable without healing phases. If the quantization error during the forward and backward passes is sufficiently minimized (as 4/6 does), the optimizer can converge to high-quality minima without needing a high-precision "safety net" at the end. This simplifies the training pipeline and ensures the final model is truly optimized for 4-bit inference.
*   **Hardware-Software Co-Design:** This work highlights the importance of co-designing algorithms with specific hardware capabilities. The success of 4/6 relies entirely on the **FP8 E4M3** scale factors available in **NVIDIA Blackwell** GPUs. It demonstrates that future gains in efficiency will come not just from raw FLOPs, but from exploiting the specific granularity of new numerical formats (like the mantissa bits in E4M3) to enable adaptive behaviors that were impossible in integer-only scaling schemes (like **MXFP4**'s E8M0).

### 7.2 Avenues for Follow-Up Research
The success of adaptive block scaling opens several concrete directions for future inquiry:

*   **Generalizing Adaptive Scaling to Other Formats:** While 4/6 is specific to the step sizes of FP4, the core principle—**dynamically selecting a scale maximum to minimize local MSE**—could be applied to other coarse formats. Future work could explore "Three Over Five" strategies for custom 3-bit formats or adaptive scaling for **FP6**, provided the hardware supports the necessary scale factor precision. The key constraint identified in **Section 5.2** is that the scale format must have sufficient resolution (mantissa bits) to represent the intermediate scaling factors; research into new microscaling formats should prioritize this granularity.
*   **Integrating 4/6 into Second-Order Optimization:** The mixed results with **GPTQ** (**Table 5**) reveal a significant opportunity. Current implementations apply 4/6 as a static quantizer within the GPTQ loop. Future research should rewrite the GPTQ objective function to explicitly account for the **discrete switch** between $M=4$ and $M=6$. This would involve treating the scale selection bit as an optimizable variable alongside the weights, potentially unlocking the full potential of second-order methods for adaptive quantization.
*   **Learned Scale Selection Policies:** Currently, 4/6 uses a heuristic (MSE minimization) to choose the scale. A more advanced approach could train a lightweight, differentiable policy network (or a simple lookup table based on block statistics like kurtosis or skewness) to predict the optimal scale without computing both candidates. This could reduce the **&lt;15% overhead** reported in **Section 3.2** to near-zero, making the method even more attractive for latency-critical training.
*   **Extending to Activation Quantization in Dynamic Graphs:** While effective for weights and static activations, applying 4/6 to highly dynamic activation tensors in real-time inference remains an area for exploration. Research could focus on approximating the MSE calculation to reduce latency further, enabling adaptive scaling for every token in a streaming LLM application.

### 7.3 Practical Applications and Use Cases
The immediate practical impact of 4/6 is most pronounced in scenarios where **memory bandwidth** and **compute throughput** are the primary bottlenecks, but accuracy cannot be compromised.

*   **Pre-training Massive Models from Scratch:** For organizations training models at the 30B+ parameter scale (like the **Nemotron 3 Nano** used in this study), 4/6 offers a path to reduce memory footprint by **4x** while maintaining convergence stability. The elimination of the "healing" phase means training jobs can finish faster and with a simpler software stack, directly reducing cloud compute costs.
*   **High-Accuracy Edge Inference:** On devices equipped with Blackwell-class architecture (or future equivalents), 4/6 enables the deployment of **AWQ** or **SmoothQuant** quantized models with near-BF16 accuracy. This is critical for edge applications (e.g., autonomous systems, mobile assistants) where the difference between **8.43** and **8.30** perplexity (**Table 5**) could determine the reliability of safety-critical decisions.
*   **Fine-Tuning Large Foundation Models:** The method is particularly well-suited for **QLoRA**-style fine-tuning where base weights are quantized. By using 4/6 for the base model weights, practitioners can fit larger models into GPU memory while ensuring the gradient updates (which are also quantized in full 4/6 training setups) do not suffer from the "dead zone" error that typically degrades fine-tuning performance.

### 7.4 Reproducibility and Integration Guidance
For practitioners looking to adopt or build upon this work, the following guidance clarifies when and how to utilize Four Over Six:

*   **When to Prefer 4/6:**
    *   **Hardware Requirement:** You **must** be targeting **NVIDIA Blackwell GPUs** (or hardware with equivalent FP8 E4M3 scale support). The method is **incompatible with MXFP4** (E8M0 scales) due to the lack of intermediate scaling values (**Section 5.2**).
    *   **Precision Target:** Use 4/6 specifically for **FP4 (E2M1)** quantization. The benefits diminish rapidly for FP6 or FP8 value formats where step sizes are naturally smaller.
    *   **PTQ Method:** If using Post-Training Quantization, 4/6 is highly recommended when paired with **AWQ** or **SmoothQuant**, where it provides consistent gains. Be cautious when pairing with **GPTQ**; unless you modify the GPTQ optimization loop to handle the dual-scale logic, you may see mixed results on certain models (e.g., Qwen 3).
    *   **Training Regime:** Ideal for **native FP4 pre-training** where you want to avoid the complexity of Random Hadamard Transforms or high-precision healing phases.

*   **Integration Steps:**
    1.  **Kernel Implementation:** Utilize the provided CUDA kernels (available in the authors' GitHub repository) which implement the dual-path quantization and MSE selection in the register file. Do not attempt to implement this in high-level Python loops, as the overhead would be prohibitive.
    2.  **Global Scale Adjustment:** Ensure your tensor scaling logic respects the **256 cap** on the FP8 scale factor (instead of the standard 448) to prevent overflow when selecting the $M=4$ option for large blocks (**Section 3.1**).
    3.  **Metadata Handling:** Modify your model serialization format to store the **1-bit scale selector** per 16-element block. Standard NVFP4 loaders will not expect this bit, so custom deserialization logic is required to correctly interpret the scale factor as relative to 4 or 6.

In summary, **Four Over Six** transforms NVFP4 from a promising but flawed format into a robust, production-ready standard for the Blackwell era. It demonstrates that the future of efficient AI lies not just in reducing bit-width, but in making those few bits work smarter through adaptive, hardware-aware algorithms.
