## 1. Executive Summary

This paper introduces REASONING GYM (RG), a library of over 100 procedurally generated reasoning environments designed to overcome the data scarcity and memorization bottlenecks inherent in fixed datasets used for Reinforcement Learning with Verifiable Rewards (RLVR). By enabling unlimited training data with adjustable complexity across domains like algebra, logic, and games, RG facilitates curriculum learning and prevents overfitting, demonstrated by Qwen2.5-3B-Instruct models achieving significant gains on external benchmarks such as MATH (+9.7%) and Big-Bench Hard (+7.66%) after RLVR training. The framework establishes that procedural generation supports robust skill transfer, where training on algorithmic tasks improved algebra performance by 29.1%, validating RG as a scalable infrastructure for advancing reasoning capabilities in large language models like o3-mini and DeepSeek-R1.

## 2. Context and Motivation

### The Data Scarcity Bottleneck in Reasoning Research

The central problem this paper addresses is a **fundamental scalability bottleneck** in training reasoning-capable language models. As Section 1 explains, the recent leaps in LLM reasoning abilities—exemplified by models like OpenAI-o1, DeepSeek-R1, and QwQ-32B—are powered by Reinforcement Learning with Verifiable Rewards (RLVR). This training paradigm leverages outcome-based feedback to unlock open-ended reasoning processes with diverse solution paths. However, RLVR's success "hinges critically on the availability of high-quality training data."

The problem is structural: current approaches depend on one of two data sources, both problematic:

1. **Expensive human-curated question-answer pairs** (e.g., GSM8K, MATH) — These are finite, costly to produce, and quickly exhausted as models improve.
2. **Internet-scraped content** — This is "neither sustainable nor reliable in the long term" and often contains errors that can corrupt training signals.

As reasoning models continue to advance, this data scarcity "threatens to become an increasingly severe constraint on further progress." The paper identifies this as a **dataset ceiling** problem: models eventually memorize fixed benchmarks rather than developing genuine reasoning capabilities, making it impossible to distinguish true improvement from dataset familiarity.

### Why This Problem Matters

This bottleneck has both theoretical and practical significance:

**Theoretical Impact:** Without unlimited, controllable training data, researchers cannot systematically study how reasoning capabilities develop. Fixed datasets prevent curriculum learning (progressively increasing difficulty), making it impossible to probe whether models are learning robust strategies or simply recognizing solution templates. The paper's zero-shot evaluation reveals that "current models have shallow competencies"—performance drops dramatically when task difficulty increases (Figure 3b shows declines of −71.9% in code and −33.8% in graphs for o3-mini). This suggests models may be "learning to recognize and apply solution templates rather than developing robust reasoning strategies."

**Practical Impact:** The data scarcity directly limits model improvement. If training data is exhausted or memorized, further RLVR training yields diminishing returns. This constrains the development of more capable reasoning systems for real-world applications requiring systematic problem-solving, from mathematical reasoning to code generation to logical inference.

### Prior Approaches and Their Limitations

Section 6 (Related Work) catalogs existing approaches, which fall into several categories:

**Fixed Reasoning Benchmarks:** Datasets like GSM8K, MATH, OlympiadBench, BIG-Bench, and GPQA provide standardized evaluation but suffer from critical limitations:
- They are **fixed in size**, creating hard ceilings on training data volume.
- Models can **memorize** them rather than learning generalizable reasoning.
- Internet-scraped variants are often **erroneous**, introducing noise into training signals.

**Procedurally Generated Benchmarks:** Some prior work uses procedural generation in specific domains:
- Games (e.g., NetHack Learning Environment, Alchemy benchmark)
- Puzzles (e.g., Sudoku variants, logic puzzle generators)
- Generative model-based synthesis

However, these are typically **domain-specific** rather than comprehensive libraries spanning multiple reasoning types.

**RLVR Training Environments:** Several systems use verifiable rewards for training:
- **Tülu 3** builds RL corpus from prompts with objectively checkable answers (GSM8K, MATH) but still relies on fixed datasets.
- **DeepSeek-R1** starts with manually curated examples then expands via rejection sampling to ~600k verified trajectories—still finite.
- **TextArena** provides 57+ text-based games but focuses on game environments specifically.
- **Logic-RL** procedurally generates logic puzzles but is domain-limited.
- **Absolute Zero Reasoner** self-evolves curriculum using code execution but focuses on code reasoning.

**Closest Parallel Efforts:** The paper acknowledges three libraries most similar to RG:
- **KORGym** focuses on games (50+ games in textual or visual formats) with multi-turn interactions.
- **Reasoning Core** provides procedurally generated RLVR environments across formal domains.
- **GEM** offers a standardized framework for RL environments with a built-in suite.

### How REASONING GYM Positions Itself

REASONING GYM distinguishes itself through three core design principles (Section 2):

| Principle | What It Means | Why It Matters |
|-----------|---------------|----------------|
| **(P1) Algorithmic Verifiability** | Every task admits automatic verification with no human judgment | Enables reliable RLVR training while eliminating subjective evaluation |
| **(P2) Large Solution Spaces** | Tasks designed with expansive solution spaces | Rewards generalizable strategies above overfitting, mitigating reward hacking |
| **(P3) Parametric Difficulty Control** | Configurable parameters systematically control problem characteristics | Enables dynamic curricula via precise difficulty adjustment |

The key innovation is **comprehensiveness combined with procedural generation**. Unlike prior work that focuses on single domains (logic puzzles, games, or math), RG provides "over 100 algorithmically verifiable tasks that can generate unlimited training instances with controllable difficulty and structural variation" across diverse domains: symbolic algebra, discrete algorithms, spatial geometry, formal logic, pattern recognition, and constraint-based puzzles.

Table 6 in the Appendix shows the full taxonomy: 6 algebra tasks, 32 algorithmic tasks, 11 arithmetic tasks, 14 cognition tasks (including ARC), 2 code tasks, 18 games, 4 geometry tasks, 6 graph tasks, 2 induction tasks, and 7 logic tasks. This breadth allows researchers to "target specific abilities during training or evaluation while still drawing from a rich, procedurally generated mix of challenges."

Furthermore, RG exposes the **ground-truth data-generating process**. As Section 6 notes, this differs from fixed benchmarks by "(1) the ability to create unlimited training examples with controllable characteristics and (2) exposing the ground-truth data-generating process." This transparency enables researchers to understand exactly how problems are constructed, facilitating deeper analysis of model behavior and failure modes.

The paper positions RG not as a replacement for human-curated benchmarks but as **complementary infrastructure**: RG handles scalable training and difficulty-progressive evaluation, while external benchmarks (MATH, GSM8K, Big-Bench Hard, MMLU-Pro) serve as held-out tests of generalization. Section 4.3 demonstrates this workflow, showing that RG-trained models improve on these external benchmarks, validating that skills transfer beyond the procedural training environment.

## 3. Technical Approach

This paper presents a **system infrastructure contribution** — a library of procedurally generated reasoning environments designed to enable scalable Reinforcement Learning with Verifiable Rewards (RLVR) training for language models.

### 3.1 Reader orientation (approachable technical breakdown)

REASONING GYM is a software library that automatically generates unlimited reasoning problems (like math puzzles, logic games, and algorithmic challenges) with adjustable difficulty levels and built-in answer verification. It solves the data scarcity problem in reasoning model training by replacing fixed datasets with procedural generators that can produce infinitely many unique training examples, where each problem's complexity can be precisely controlled through configurable parameters.

### 3.2 Big-picture architecture (diagram in words)

The REASONING GYM system consists of four major components that work together in a pipeline:

1. **Task Generators** (100+ procedural algorithms): Each generator accepts configuration parameters (difficulty, structure, style) and outputs a unique problem instance with its ground-truth solution. For example, the `spiral_matrix` generator takes matrix dimensions (`min_n=2`, `max_n=10`) and produces a matrix traversal problem.

2. **Verifier Module** (automatic reward computation): For each generated problem, the verifier compares the model's output against the ground-truth solution and computes a binary accuracy reward (1.0 for correct, 0.0 for incorrect) plus auxiliary formatting rewards (worth 0.2).

3. **RLVR Training Loop** (GRPO algorithm): The training infrastructure samples problems from generators, collects model responses, computes verifiable rewards, and updates model weights using Group Relative Policy Optimization with a learning rate of $1 \times 10^{-6}$ and KL coefficient of $0.001$.

4. **Curriculum Controller** (optional difficulty progression): When enabled, this component monitors performance and increases task difficulty when the model exceeds 70% accuracy over 20 consecutive training steps, enabling adaptive curriculum learning.

The information flow proceeds as follows: First, the task generator produces a problem instance with known solution. Second, the language model generates a response. Third, the verifier compares response to ground truth and computes rewards. Fourth, the RL algorithm updates model parameters based on reward signals. This cycle repeats for 500–800 training steps depending on the experiment.

### 3.3 Roadmap for the deep dive

- **Procedural Generation Mechanism**: We first explain how task generators work, including the three parameter types (difficulty, structural, stylistic) and how they enable unlimited unique problem instances without memorization risk.
- **Verification and Reward Computation**: We then detail the automatic verification system that enables RLVR training, including the reward structure (accuracy + formatting) and why algorithmic verifiability matters.
- **RLVR Training Configuration**: We explain the GRPO algorithm setup, hyperparameters (learning rate, batch size, KL penalty), and training infrastructure (verl library, 4×A6000 GPUs).
- **Curriculum Learning Implementation**: We describe how difficulty progression works, including the 70% threshold over 20 steps that triggers difficulty increases.
- **Evaluation Protocol**: We cover how performance is measured (Acc@3, 50 holdout examples) and how transfer is tested across domains and external benchmarks.

### 3.4 Detailed, sentence-based technical breakdown

#### Procedural Generation Mechanism

REASONING GYM implements over 100 task generators, each representing a distinct reasoning challenge that can produce unlimited unique problem instances through parameterized algorithms rather than fixed question-answer pairs. The system organizes these generators into ten high-level categories: algebra (6 tasks), algorithms (32 tasks), arithmetic (11 tasks), cognition including ARC (14 tasks), code (2 tasks), games (18 tasks), geometry (4 tasks), graphs (6 tasks), induction (2 tasks), and logic (7 tasks), as documented in Table 6 of the Appendix. Each generator accepts three types of configuration parameters that control different aspects of problem generation. **Difficulty Parameters** directly control complexity metrics such as node counts for graph problems, polynomial degrees for algebra tasks, or word lengths for language tasks — for example, the `polynomial_equations` generator uses `min_degree=1` and `max_degree=3` for easy settings but `min_degree=2` and `max_degree=3` for hard settings (Appendix A.3). **Structural Parameters** determine fundamental problem properties like dimensionality, constraint types, or proof depth — the `binary_matrix` task uses `p_zero=0.25` to control the density of zeros in the input matrix, affecting solution complexity without changing the problem type. **Stylistic Parameters** vary presentation without affecting difficulty, such as variable names, number formats, or problem framing — this ensures models cannot exploit superficial patterns while maintaining consistent cognitive demands.

The procedural approach eliminates memorization concerns through combinatorial explosion of possible problem instances. For the `rubiks_cube` task, the configuration specifies `cube_size=3` with `min_scramble_steps=3` and `max_scramble_steps=10` for easy settings, but `min_scramble_steps=25` and `max_scramble_steps=50` for hard settings (Appendix A.3), creating virtually infinite unique cube configurations. The `count_primes` task ranges from `min_n=1, max_n=10000` for easy problems to `min_n=10000, max_n=50000` for hard problems, ensuring each training instance presents novel numerical challenges. This contrasts with fixed datasets where models can memorize specific problem-solution pairs rather than learning generalizable reasoning strategies. Each generator exposes its ground-truth data-generating process, meaning researchers can inspect exactly how problems are constructed — for instance, the `spiral_matrix` generator specifies clockwise traversal starting from the top-left corner, moving right, then down, then left, then up, repeating for inner elements until every entry is visited (Appendix A.2.2). This transparency enables systematic analysis of model failure modes by correlating errors with specific parameter configurations.

#### Verification and Reward Computation

Every task in REASONING GYM implements algorithmic verifiability (Design Principle P1), meaning each problem admits automatic verification requiring no human judgment to determine correctness. The verification system computes a total reward $R$ as the sum of an accuracy component $R_{\text{acc}}$ and an auxiliary formatting component $R_{\text{format}}$:

$$R = R_{\text{acc}} + R_{\text{format}}$$

where $R_{\text{acc}} \in \{0, 1\}$ indicates whether the model's answer matches the ground-truth solution, and $R_{\text{format}} = 0.2$ rewards proper output formatting. This reward structure serves two purposes: the accuracy component provides the primary learning signal for reasoning capability, while the formatting component ensures models learn to produce outputs in the expected structure (e.g., "F-1 G+1 A-1" for Rush Hour moves or space-separated integers for matrix problems). The training reward plots in Figures 4, 5, and 6 represent this total reward, but evaluation tables report only the accuracy component rescaled to a 0–100% range for interpretability.

The verification mechanism differs by task category but always produces deterministic, unambiguous rewards. For mathematical tasks like `polynomial_multiplication`, the verifier symbolically evaluates the model's answer and compares it to the computed ground truth. For game tasks like `rush_hour`, the verifier simulates the proposed moves and checks if the red car reaches the exit. For logic tasks like `knights_knaves`, the verifier evaluates the truth assignments against the constraint satisfaction requirements. This automatic verification enables reliable RLVR training while eliminating the subjective evaluation that plagues human-graded benchmarks. The large solution spaces (Design Principle P2) mitigate reward hacking by ensuring tasks have expansive solution spaces that reward generalizable strategies above overfitting — for example, the `spell_backward` task requires reversing arbitrary words rather than memorizing specific word-reversal pairs.

#### RLVR Training Configuration

The paper uses Group Relative Policy Optimization (GRPO) as the reinforcement learning algorithm for all experiments, implemented through the verl open-source library. The base model for all experiments is Qwen2.5-3B-Instruct, a 3 billion parameter instruction-tuned language model. Training runs are conducted on a 4×A6000 GPU node rented from Runpod, with total compute requirements of approximately 1500 A6000 hours across all experiments. The training configuration specifies a learning rate of $1 \times 10^{-6}$ with no warmup (lr_warmup_steps_ratio: 0.0) and constant warmup style. The batch size is set to 32 for training and 64 for validation, with a maximum prompt length of 4096 tokens and maximum response length of 2048 tokens.

GRPO requires sampling multiple responses per prompt to compute relative advantages, and the configuration specifies $n=8$ samples per prompt. The algorithm uses a KL penalty to prevent the policy from diverging too far from the reference model, with kl_loss_coef set to $0.001$ and kl_loss_type set to "low_var_kl". The entropy coefficient is set to $0.001$ to maintain some exploration during training. Gradient clipping is applied at 1.0 to prevent unstable updates, and the clip ratio is set to 0.2 following standard PPO conventions. The training uses FSDP (Fully Sharded Data Parallel) strategy with gradient checkpointing enabled to fit the model in GPU memory. The total training steps vary by experiment: intra-domain and cross-domain transfer experiments run for 500 steps, while the RG-Math model for external benchmark evaluation trains for 800 GRPO steps on a composite of algebra, arithmetic, and geometry tasks.

The reward and critic models required by verl are present in the configuration but have no effect when using GRPO due to the algorithm's design — GRPO computes advantages from the sampled responses directly without requiring separate value function estimation. The actor_rollout_ref configuration uses hybrid_engine=True with vllm for efficient rollout generation, temperature set to 1.0, top_p set to 1, and gpu_memory_utilization set to 0.7. The tensor_model_parallel_size is set to 4 to distribute the model across all four A6000 GPUs. Training configurations are stored as YAML files specifying dataset weights (all set to 1.0 for uniform sampling), difficulty parameters, and curriculum settings (Appendix A.6 shows the complete configuration).

#### Curriculum Learning Implementation

The curriculum learning experiments (Section 5) implement a simple but effective difficulty progression mechanism that continuously increases task complexity based on model performance. The system trains under two conditions for comparison: (1) curriculum learning, which starts with the easiest difficulty level and progressively increases difficulty when performance exceeds 70% accuracy over 20 training steps, and (2) fixed difficulty, which samples uniformly from all difficulty levels throughout training. The curriculum controller monitors performance using a success_threshold of 0.70 and a failure_threshold of 0.10, with automatic curriculum updating enabled after every 30 steps (update_steps: 30). The `last_k` parameter is set to 20, meaning the system evaluates the last 20 training steps to determine if the threshold has been met.

When the curriculum condition is satisfied, the system increases the difficulty parameter for that task — for `spell_backward`, this means increasing word length from 4 to 6 to 8 to 10; for `mini_sudoku`, this means increasing empty cells from 4–6 to 6–8 to 8–10 to 10–12; for `count_primes`, this means increasing the number range from 100–500 to 100–1000 to 100–5000. Figure 6 shows that difficulty increases are often preceded by sharp drops in reward, reflecting the model encountering harder examples after mastering easier ones. The curriculum model's lower terminal reward in some environments reflects its exclusive exposure to maximum difficulty examples by the end of training, while the non-curriculum model samples across the full difficulty distribution. Both models train for a single epoch, and evaluation uses 50 holdout examples from each difficulty level to measure performance across the difficulty spectrum.

Table 5 demonstrates that curriculum-trained models achieve superior performance to non-curriculum equivalents in all environments and difficulty levels, with margins of +40.67% for word length 4 on Spell Backwards, +13.33% for 8–10 empty cells on Mini Sudoku, and +26.67% for number range 100–500 on Count Primes. However, the paper notes that curriculum learning's value "may be more limited in environments where progression paths are ambiguous or difficult to formalize" — for Count Primes, the curriculum model never advanced past the initial difficulty level yet still decisively outperformed the non-curriculum approach, suggesting that even attempted curriculum progression provides benefits.

#### Evaluation Protocol

The paper employs a rigorous evaluation protocol to measure both in-distribution performance and generalization capabilities. For intra-domain and cross-domain transfer experiments, each evaluation involves three independent runs on identical evaluation sets of 50 problems, providing robust estimates of transfer effects with reported standard errors. Performance is measured using Acc@3, which represents the accuracy across the three independent runs. For zero-shot evaluation of frontier models, the paper evaluates on both easy and hard task configurations, with Appendix A.3 detailing the specific parameter configurations for each setting — for example, `complex_arithmetic` uses `min_real=-10, max_real=10` for easy but `min_real=-100, max_real=100` for hard settings.

External benchmark evaluation uses standardized protocols from the Language Model Evaluation Harness to ensure fair comparison. GSM8K is evaluated with 8-shot chain-of-thought prompting, MATH with 0-shot chain-of-thought, Big-Bench Hard with 3-shot chain-of-thought, and MMLU-Pro with standard protocols. The RG-Math model (trained on algebra, arithmetic, and geometry composites) and RG-Algorithmic model (trained on algorithmic tasks) are evaluated on these benchmarks to test cross-benchmark transfer. Table 3 shows that RG-Math achieves 76.7% on GSM8K (+0.5% over baseline), 58.2% on MATH (+9.7% over baseline), and 16.34% on Big-Bench Hard (+7.66% over baseline). Table 4 shows improvements across MMLU-Pro categories, with the RG-Math model achieving +5.62% on Math, +4.40% on Computer Science, and +6.02% on Psychology tasks.

The evaluation design isolates transfer effects by ensuring models never see data from evaluation domains during training — for cross-domain transfer, models trained on one category (e.g., algorithmic tasks) are evaluated on held-out tasks from different domains (e.g., algebra, geometry). This design tests whether RLVR instills general reasoning capabilities that transcend specific problem types, rather than domain-specific heuristics. The consistency of improvements across diverse difficulty levels and domains indicates that RLVR develops robust reasoning strategies rather than task-specific solutions.

## 4. Key Insights and Innovations

This section distills the paper's most significant contributions — not just what the authors built, but what we *learn* from using it. These insights reshape how we understand reasoning development in language models and point toward new research directions.

### 4.1 The Difficulty Cliff: Exposing Shallow Competencies

**What We Learned:** When task difficulty increases, frontier models don't degrade gracefully — they collapse. Figure 3b reveals performance drops of **−71.9% in code**, **−33.8% in graphs**, and **−27.9% in algorithms** for o3-mini when transitioning from easy to hard configurations. DeepSeek-R1 shows similar patterns (−61.8% in code, −29.6% in graphs).

**Why This Is Surprising:** Prior benchmark evaluations typically report aggregate scores that mask this fragility. A model scoring 63.5% on RG's hard configurations (Figure 3a) appears competent, but the cliff phenomenon reveals this competence is **brittle** — it depends on specific difficulty parameters rather than robust reasoning strategies.

**What Makes This Different from Prior Work:**

| Prior Benchmark Approach | REASONING GYM Discovery |
|-------------------------|------------------------|
| Fixed difficulty per task | Parametric difficulty reveals performance gradients |
| Single accuracy score | Difficulty-transition analysis exposes competency boundaries |
| Assumes monotonic improvement | Reveals sharp cliffs where capabilities break |

**Significance:** This finding challenges the assumption that high benchmark scores indicate genuine reasoning ability. The paper suggests models may be "learning to recognize and apply solution templates rather than developing robust reasoning strategies" (Section 3.2). This has profound implications:

- **For Evaluation:** Single-difficulty benchmarks are insufficient. Researchers must test across difficulty gradients to distinguish template-matching from genuine reasoning.
- **For Training:** If models exploit shallow patterns, RLVR training must explicitly target difficulty progression to force deeper strategy learning.
- **For Interpretation:** A 60% accuracy score means something very different when you know the model drops to 10% on slightly harder variants of the same task.

**Critical Caveat:** The difficulty cliff magnitude varies by domain. Arithmetic shows minimal degradation (some models maintain near-ceiling performance), while code and graphs show catastrophic drops. This suggests the cliff is not uniform — some reasoning capabilities are more fragile than others. The paper does not fully explain *why* certain domains exhibit steeper cliffs, leaving this as an open research question.

### 4.2 Cross-Domain Transfer: Evidence for General Reasoning Primitives

**What We Learned:** Training on algorithmic tasks improves performance on *unrelated* mathematical domains. Table 2 shows that models trained solely on algorithmic RG tasks achieve **+29.1% on algebra** and **+22.3% on geometry** — domains they never saw during training. Logic training improves cognition (+13.3%) and graph reasoning (+9.1%). Games training, despite poor in-domain performance (3.3%), transfers to algebra (+21.8%) and cognition (+13.1%).

**Why This Is Surprising:** Standard transfer learning theory suggests skills transfer primarily within related domains. The observation that constraint-satisfaction skills from games transfer to algebra challenges this assumption. Even more striking: the Games category shows the worst *in-domain* learning (Figure 5 shows early plateauing), yet produces substantial *cross-domain* benefits.

**What Makes This Different from Prior Work:**

| Prior RLVR Studies | REASONING GYM Discovery |
|-------------------|------------------------|
| Intra-domain transfer only (e.g., math→math) | Cross-domain transfer across fundamentally different reasoning types |
| Transfer assumed but rarely measured | Systematic evaluation across 10 domain pairs with held-out test sets |
| Domain-specific heuristics expected | Evidence for generalizable reasoning primitives |

**Significance:** This finding validates a core hypothesis underlying RLVR research: that reinforcement learning can develop **general reasoning capabilities** rather than narrow task-specific competencies. The implications are substantial:

- **For Training Efficiency:** Researchers need not curate training data for every target domain. Algorithmic training alone may bootstrap mathematical reasoning.
- **For Model Architecture:** If transfer occurs, reasoning may rely on shared computational primitives across domains — suggesting architectural inductive biases could accelerate learning.
- **For Evaluation:** Cross-domain transfer becomes a litmus test for genuine reasoning vs. memorization. Models that cannot transfer are likely exploiting domain-specific shortcuts.

**Critical Caveat:** Transfer is not universal. ARC tasks show *negative* transfer from algorithmic training (−2.3%), and Games training slightly harms Logic performance (−0.8%). The paper does not explain these negative transfers, which suggests some task combinations may interfere rather than complement. Understanding when transfer helps vs. hurts remains an open question.

---

### 4.3 Procedural Generation as Anti-Memorization Infrastructure

**What We Learned:** RG's procedural approach eliminates the "dataset ceiling" problem inherent to fixed benchmarks. Each task generator produces virtually infinite unique instances through combinatorial parameter variation — the `rubiks_cube` generator alone can produce $3^{50}$ scramble configurations with its hard settings (Appendix A.3 specifies 25–50 scramble steps).

**Why This Is Surprising:** Prior procedurally generated benchmarks (NetHack, Alchemy, Sudoku variants) focus on single domains. RG demonstrates that procedural generation scales across **100+ distinct reasoning types** while maintaining verifiable rewards — a systems engineering achievement that enables new research paradigms.

**What Makes This Different from Prior Work:**

| Prior Approach | REASONING GYM Innovation |
|---------------|-------------------------|
| Fixed datasets (GSM8K, MATH) | Unlimited generation with known ground truth |
| Domain-specific generators (Logic-RL, KORGym) | Comprehensive library spanning algebra, logic, games, cognition |
| Opaque data generation | Exposed data-generating process for analysis |
| Memorization risk | Combinatorial explosion prevents instance repetition |

**Significance:** This transforms how researchers can study reasoning development:

- **For Curriculum Learning:** Researchers can now implement true difficulty progression (Section 5) rather than approximating it with fixed dataset subsets.
- **For Failure Analysis:** Since the ground-truth generation process is exposed, researchers can correlate errors with specific parameter configurations — impossible with scraped or human-curated data.
- **For Long-Horizon Training:** RLVR experiments no longer terminate due to data exhaustion. The paper's 800-step RG-Math training (Section 4.3) could extend indefinitely with increasing difficulty.

**Critical Caveat:** Procedural generation has inherent limitations. Section 7 acknowledges that "some reasoning domains, particularly those requiring extensive domain knowledge or creativity, are difficult to capture with procedural generators." Tasks requiring unstructured answers, cultural knowledge, or open-ended creativity remain outside RG's scope. The library excels at verifiable reasoning but cannot replace human-curated data for all use cases.

### 4.4 Curriculum Learning Outperforms Fixed-Difficulty Training

**What We Learned:** Adaptive difficulty progression consistently beats uniform difficulty sampling. Table 5 shows curriculum-trained models outperforming non-curriculum equivalents across all tested environments: **+40.67%** for word length 4 on Spell Backwards, **+13.33%** for 8–10 empty cells on Mini Sudoku, **+26.67%** for number range 100–500 on Count Primes.

**Why This Is Surprising:** The Count Primes experiment reveals something counterintuitive: the curriculum model *never advanced past the initial difficulty level* (Figure 6 shows no vertical bars indicating difficulty increases), yet still decisively outperformed the non-curriculum model. This suggests that **attempting curriculum progression provides benefits even when progression fails** — possibly because the training dynamics differ when the model knows harder examples await.

**What Makes This Different from Prior Work:**

| Prior Curriculum Approaches | REASONING GYM Implementation |
|----------------------------|-----------------------------|
| Manual difficulty ordering | Automatic threshold-based progression (70% over 20 steps) |
| Fixed curriculum schedules | Dynamic adaptation based on real-time performance |
| Evaluated on single tasks | Systematic comparison across multiple reasoning domains |

**Significance:** This validates curriculum learning as more than a training optimization — it appears to fundamentally change *what* the model learns:

- **For RLVR Training:** Fixed-difficulty training may converge to local optima that curriculum training avoids. The reward dynamics in Figure 6 show curriculum models experience sharp drops after difficulty increases, suggesting they're being pushed out of comfort zones.
- **For Sample Efficiency:** Curriculum models achieve better final performance with the same number of training steps, implying more efficient learning.
- **For Research Design:** Future RLVR studies should treat curriculum design as a first-class experimental variable, not an implementation detail.

**Critical Caveat:** The paper explicitly notes curriculum learning's value "may be more limited in environments where progression paths are ambiguous or difficult to formalize." The 70% threshold over 20 steps is a heuristic — optimal thresholds likely vary by task. The paper does not ablate this hyperparameter, leaving uncertainty about whether better threshold selection could improve results further.

### 4.5 External Benchmark Transfer: Validating Real-World Applicability

**What We Learned:** RG-trained models improve on established held-out benchmarks. Table 3 shows the RG-Math model achieving **+9.7% on MATH**, **+7.66% on Big-Bench Hard**, and **+0.5% on GSM8K**. Table 4 shows consistent gains across MMLU-Pro categories, with **+6.02% on Psychology** and **+5.70% on Physics**.

**Why This Is Surprising:** RG contains *no* MATH, GSM8K, or MMLU-Pro data during training. The improvement demonstrates that procedurally generated reasoning tasks develop capabilities that transfer to human-curated benchmarks — validating RG as a training infrastructure, not just an evaluation tool.

**What Makes This Different from Prior Work:**

| Prior RLVR Training | REASONING GYM Approach |
|--------------------|----------------------|
| Train on benchmark data directly | Train on procedural data, test on benchmarks |
| Risk of benchmark overfitting | Clean separation between training and evaluation |
| Improvements may reflect memorization | Improvements must reflect genuine capability gains |

**Significance:** This closes the loop on RG's value proposition:

- **For Practitioners:** RG provides a viable alternative to expensive human-curated training data. The ~1500 A6000 hours invested (Section 4) produced models competitive with those trained on much costlier datasets.
- **For Research Integrity:** Clean separation between training and evaluation eliminates concerns about benchmark contamination — a growing problem in LLM evaluation.
- **For Scaling:** Since RG data is unlimited, future work can investigate whether continued RG training produces continued benchmark improvements, or whether diminishing returns emerge.

**Critical Caveat:** GSM8K shows only marginal improvement (+0.5%), while MATH shows substantial gains (+9.7%). The paper does not explain this discrepancy. One hypothesis: GSM8K may be closer to saturation for 3B models, leaving less room for improvement. Alternatively, GSM8K's arithmetic focus may not benefit as much from RG's diverse reasoning training. This warrants further investigation.

---

### Synthesis: Incremental vs. Fundamental Contributions

Distinguishing between incremental improvements and fundamental innovations clarifies RG's place in the research landscape:

**Incremental Contributions:**
- The specific task generators (individual puzzles, math problems) build on existing procedural generation work.
- The GRPO training setup follows established RLVR protocols from DeepSeek-R1 and Tülu 3.
- The curriculum learning threshold (70% over 20 steps) is a reasonable heuristic but not theoretically derived.

**Fundamental Innovations:**
- **Comprehensive procedural library at scale:** No prior work combines 100+ reasoning domains with verifiable rewards and parametric difficulty control.
- **The difficulty cliff discovery:** This empirical finding changes how we should evaluate reasoning models — single-difficulty benchmarks are now demonstrably insufficient.
- **Cross-domain transfer evidence:** The demonstration that algorithmic training improves geometry and algebra performance challenges assumptions about reasoning modularity.
- **Training/evaluation separation:** RG enables clean separation between procedural training data and fixed evaluation benchmarks, addressing contamination concerns that plague current LLM research.

The paper's lasting contribution may be less about any single task generator and more about establishing **procedural generation as viable infrastructure for reasoning research** — shifting the field from data curation toward data generation, from fixed evaluation toward difficulty-progressive assessment, and from domain-specific training toward cross-domain transfer investigation.

## 5. Experimental Analysis

This section examines how the authors validated REASONING GYM's claims through systematic experimentation. We will walk through each experimental setup, unpack the quantitative results, and critically assess whether the evidence supports the paper's conclusions about procedural generation, skill transfer, and curriculum learning.

### 5.1 Evaluation Methodology Overview

The paper employs a **multi-tiered evaluation strategy** designed to test RG at three levels:

| Evaluation Level | What It Tests | How It's Measured |
|-----------------|---------------|-------------------|
| **Zero-Shot Capability** | Whether frontier models can solve RG tasks without training | Accuracy on easy vs. hard configurations across 100+ tasks |
| **Transfer Learning** | Whether RLVR training develops generalizable reasoning skills | Intra-domain, cross-domain, and external benchmark performance |
| **Curriculum Effectiveness** | Whether adaptive difficulty progression improves learning | Curriculum vs. fixed-difficulty training comparison |

**Base Model:** All RLVR experiments use `Qwen2.5-3B-Instruct` [84], a 3 billion parameter instruction-tuned language model. This choice is deliberate — a smaller model makes transfer effects more visible than a saturated frontier model.

**Training Algorithm:** Group Relative Policy Optimization (GRPO) [63] with the following key hyperparameters (Appendix A.6):
- Learning rate: $1 \times 10^{-6}$
- Batch size: 32 (train), 64 (validation)
- Samples per prompt: $n=8$ (required for GRPO advantage estimation)
- KL coefficient: $0.001$
- Maximum prompt length: 4096 tokens
- Maximum response length: 2048 tokens
- Training steps: 500 (transfer experiments), 800 (RG-Math external evaluation)

**Compute Infrastructure:** All experiments ran on 4×A6000 GPU nodes rented from Runpod, totaling approximately **1500 A6000 hours** across all experiments (Section 4). This is a non-trivial investment but modest compared to frontier model training.

**Evaluation Metric:** The primary metric is `Acc@3` — accuracy averaged across three independent runs on identical evaluation sets of 50 problems per task. This provides robust estimates with reported standard errors. Training rewards include both accuracy ($R_{\text{acc}} \in \{0, 1\}$) and formatting components ($R_{\text{format}} = 0.2$), but evaluation tables report only accuracy rescaled to 0–100%.

**Critical Design Choice:** The evaluation separates training and test domains completely. For cross-domain transfer, models trained on one category (e.g., algorithmic tasks) are evaluated on held-out tasks from *different* domains (e.g., algebra, geometry). This isolates whether RLVR instills general reasoning capabilities versus domain-specific heuristics.

### 5.2 Zero-Shot Performance: Establishing the Baseline

Before testing RG's training capabilities, the authors first establish what frontier models can achieve without any RG-specific training. This serves two purposes: (1) validating that RG tasks are genuinely challenging, and (2) identifying which reasoning domains remain unsolved.

**Models Evaluated:** Eight frontier LLMs spanning both reasoning-optimized and general-purpose categories:
- Reasoning models: `o3-mini` (63.5%), `DeepSeek-R1` (59.5%), `Grok 3 Mini` (55.1%)
- General-purpose models: `Llama 4 Maverick` (41.5%), `Claude 3.5 Sonnet` (40.3%), `Gemma 3 27B` (20.3%)

**Key Finding 1: Reasoning Models Outperform by 22%**

Figure 3a shows the average score across hard configurations. The gap between the best reasoning model (o3-mini at 63.51%) and the best general-purpose model (Llama 4 Maverick at 41.50%) is **22.01 percentage points**. This is not marginal — it represents a step change in capabilities.

> "This 22% gap between the best reasoning and non-reasoning models represents more than a marginal improvement; suggesting that RLVR unlocks a step change in capabilities." (Section 3.1)

**Why This Matters:** The consistency of this advantage across RG's diverse task categories indicates that reasoning-specific training develops *broadly applicable skills* rather than narrow-domain expertise. If the gap were only present in math tasks, we might attribute it to mathematical pretraining. But the gap persists across logic, games, and algorithmic tasks — suggesting RLVR develops transferable reasoning primitives.

**Key Finding 2: The Difficulty Cliff Phenomenon**

Figure 3b quantifies performance degradation when transitioning from easy to hard task configurations. The numbers are striking:

| Model | Code | Graphs | Geometry | Algorithms |
|-------|------|--------|----------|------------|
| o3-mini | −71.9% | −33.8% | −33.1% | −25.6% |
| DeepSeek-R1 | −61.8% | −29.6% | −11.8% | −27.9% |

**What This Reveals:** Current models have **shallow competencies**. The dramatic performance drops suggest models are "learning to recognize and apply solution templates rather than developing robust reasoning strategies" (Section 3.2). A model scoring 60% on easy configurations may drop to 10% on hard variants of the *same task type*.

**Domain Variation:** The cliff is not uniform. Arithmetic shows minimal degradation (some models maintain near-ceiling performance), while code and graphs show catastrophic drops. This suggests certain reasoning capabilities are more fragile than others — an important insight for prioritizing future training efforts.

**Visual-Spatial Reasoning Remains Hard:** Tasks requiring visual-spatial reasoning represented in text format (cognition, games) prove particularly challenging, with even the strongest models achieving less than 50% accuracy (Section 3.1). This aligns with prior work [8, 9, 58] showing text-based spatial reasoning is a persistent challenge.

**Appendix Validation:** Figures 7 and 8 (Appendix A.4, A.5) provide per-task breakdowns for all 100+ tasks under easy and hard settings respectively. These granular results confirm the aggregate patterns — top reasoning models achieve consistently high accuracy on easy tasks but falter on long-horizon puzzles like `rush_hour`, `rubiks_cube`, and `rotten_oranges` under hard configurations.

### 5.3 Intra-Domain Transfer: Does Training Help Within Domains?

The first transfer experiment tests whether RLVR training on a subset of tasks within a reasoning domain improves performance on held-out tasks from the *same* domain. This establishes whether models develop domain-specific reasoning strategies that generalize beyond specific training tasks.

**Experimental Design:**
- Base model: `Qwen2.5-3B-Instruct`
- Training: GRPO on composite tasks from one category
- Evaluation: Held-out tasks from the same domain
- Runs: Three independent runs per experiment
- Evaluation set: 50 problems per task

**Results (Table 1):**

| Category | Baseline | RG-RLVR | Improvement |
|----------|----------|---------|-------------|
| Algebra | 5.0% | 16.7% | **+11.7%** |
| Algorithmic | 52.3% | 59.7% | **+7.4%** |
| Arithmetic | 89.7% | 96.0% | **+6.3%** |
| Cognition | 40.3% | 42.3% | **+2.0%** |
| Games | 0.0% | 3.3% | **+3.3%** |

**Critical Observations:**

1. **Consistent Improvement Across All Categories:** Every domain shows positive transfer, ranging from modest gains (cognition +2.0%) to substantial improvements (algebra +11.7%). This consistency suggests RLVR develops robust domain-specific strategies rather than task-specific solutions.

2. **Ceiling Effect in Arithmetic:** The base model already achieves 89.7% on arithmetic, likely due to extensive mathematical training during supervised fine-tuning. The +6.3% improvement is meaningful but limited by the ceiling — this provides a useful control showing training improvements reflect genuine learning rather than artifacts.

3. **Bootstrapping New Capabilities (Games):** The Games category is particularly striking — the base model achieves **zero accuracy** but develops measurable capability (3.3%) after RLVR training. This suggests domain-specific training can bootstrap *entirely new reasoning capabilities*, not merely refine existing ones.

**Training Dynamics (Figure 4):** Most categories exhibit rapid initial improvement, reflecting both format learning and genuine skill acquisition. The exception is arithmetic, where the base model's strong competency creates a ceiling effect. This pattern validates that RG's reward signals are informative enough to drive learning across diverse reasoning types.

**Assessment:** The intra-domain results convincingly demonstrate that RLVR training on RG tasks improves performance within domains. However, the modest absolute gains in some categories (cognition +2.0%, games +3.3%) suggest that 500 training steps may be insufficient for domains requiring more complex strategy learning.

### 5.4 Cross-Domain Transfer: The Surprising Finding

More significant than intra-domain transfer is the question of whether reasoning skills learned in one domain benefit performance in *entirely different* domains. This tests whether RLVR instills general reasoning capabilities that transcend specific problem types.

**Experimental Design:**
- Train separate Qwen2.5-3B-Instruct instances on individual RG categories
- Evaluate on held-out tasks from *different* domains
- Ensure models never see data from evaluation domains during training
- Three independent runs per evaluation for robust estimates

**Results (Table 2):**

| Test Dataset | Baseline | RG-Algebra | RG-Algorithmic | RG-Logic | RG-Games |
|--------------|----------|------------|----------------|----------|----------|
| Algebra | 23.83% | — | **52.89%** (+29.1) | — | **45.61%** (+21.8) |
| Algorithmic | 13.49% | **20.68%** (+7.2) | — | **16.43%** (+2.9) | — |
| Arithmetic | 29.56% | **46.14%** (+16.6) | **45.17%** (+15.6) | — | — |
| Cognition | 11.62% | — | — | **24.94%** (+13.3) | **24.71%** (+13.1) |
| Geometry | 0.83% | — | **23.17%** (+22.3) | — | **6.83%** (+6.0) |
| Graphs | 19.81% | — | — | **28.86%** (+9.1) | **22.49%** (+2.7) |

**Key Transfer Patterns:**

1. **Algorithmic Training Transfers Broadly:** Models trained on algorithmic tasks show substantial improvements in algebra (+29.1%) and geometry (+22.3%). This suggests procedural reasoning skills generalize across mathematical domains — a finding that challenges assumptions about reasoning modularity.

2. **Logic Training Enhances Pattern Recognition:** Training on logic tasks improves performance in cognition (+13.3%) and graph reasoning (+9.1%), indicating shared underlying reasoning mechanisms between formal logic and pattern-based tasks.

3. **Games Training Shows Selective Transfer:** Despite poor in-domain performance (3.3% from Table 1), games-trained models improve on algebra (+21.8%) and cognition (+13.1%). This suggests constraint-satisfaction skills transfer to other domains even when in-domain learning is limited.

**Negative Transfer Cases:** Not all transfer is positive:
- ARC tasks show **−2.3%** transfer from algorithmic training
- Games training slightly harms Logic performance (**−0.8%**)

**Why This Matters:** These negative transfers are as informative as positive ones. They suggest some task combinations may *interfere* rather than complement — possibly because different reasoning strategies conflict. Understanding when transfer helps versus hurts remains an open research question that RG enables investigators to explore systematically.

**Training Dynamics (Figure 5):** The Games category plateaus early, suggesting fundamental challenges in learning visual-spatial reasoning from text representations. However, despite this plateau, Games training still produces cross-domain benefits — indicating that even partial learning in one domain can transfer to others.

**Assessment:** The cross-domain results provide strong evidence that RLVR training develops transferable reasoning capabilities extending far beyond specific training domains. The magnitude of transfer (+29.1% algebra from algorithmic training) exceeds typical transfer learning expectations and validates procedural generation as a viable training infrastructure.

### 5.5 External Benchmark Transfer: Real-World Validation

The ultimate test of RG's utility is whether skills developed through procedural training transfer to established reasoning benchmarks. This addresses the critical question: does RG training produce models that perform better on *real* evaluation tasks, or does it only improve performance on RG's own tasks?

**Experimental Protocol:**
- **RG-Math Model:** Qwen2.5-3B-Instruct trained for 800 GRPO steps on algebra, arithmetic, and geometry composites
- **RG-Algorithmic Model:** Same checkpoint from Section 4.2 (cross-domain transfer)
- **Evaluation:** Standardized protocols via Language Model Evaluation Harness [15]

**Results on Mathematical Benchmarks (Table 3):**

| Model | GSM8K (8-shot, CoT) | MATH (0-shot, CoT) | Big-Bench Hard (3-shot, CoT) |
|-------|---------------------|--------------------|------------------------------|
| Baseline | 76.2% (±1.17) | 48.5% (±0.68) | 8.68% (±0.30) |
| RG-Math | **76.7%** (+0.5, ±1.16) | **58.2%** (+9.7, ±0.66) | **16.34%** (+7.66, ±0.40) |

**Results on MMLU-Pro (Table 4):**

| Task | Baseline | RG-Algorithmic | RG-Math |
|------|----------|----------------|---------|
| Math | 54.63% | 53.89% (−0.74) | **60.25%** (+5.62) |
| Computer Science | 37.80% | **40.73%** (+2.93) | **42.20%** (+4.40) |
| Physics | 38.49% | **39.26%** (+0.77) | **44.19%** (+5.70) |
| Psychology | 50.75% | **55.01%** (+4.26) | **56.77%** (+6.02) |
| Biology | 56.90% | **59.00%** (+2.10) | **61.09%** (+4.19) |

**Critical Analysis:**

1. **MATH Shows Substantial Gains (+9.7%):** This is the most significant improvement, suggesting RG's mathematical training develops capabilities that transfer to challenging competition-level problems. The 0-shot CoT protocol makes this especially meaningful — the model isn't benefiting from example prompts.

2. **Big-Bench Hard Shows Large Relative Improvement (+7.66%):** While the absolute score (16.34%) remains low, this represents an **88% relative improvement** over baseline. Big-Bench Hard contains diverse reasoning tasks, so this transfer validates RG's multi-domain training approach.

3. **GSM8K Shows Marginal Gains (+0.5%):** This discrepancy warrants attention. Possible explanations:
   - GSM8K may be closer to saturation for 3B models, leaving less room for improvement
   - GSM8K's arithmetic focus may not benefit as much from RG's diverse reasoning training
   - The 8-shot CoT protocol may already maximize baseline performance

4. **MMLU-Pro Shows Consistent Gains:** Both RG-Math and RG-Algorithmic models outperform baseline across most categories. Notably, RG-Algorithmic improves Psychology (+4.26%) and Biology (+2.10%) — domains seemingly unrelated to algorithmic reasoning. This suggests RLVR may develop general cognitive capabilities beyond narrow reasoning skills.

**Assessment:** The external benchmark results convincingly demonstrate that RG training produces meaningful improvements on established benchmarks. The cross-benchmark consistency confirms RG develops transferable reasoning skills, not task-specific pattern matching. The clean separation between RG training data and external benchmarks eliminates contamination concerns that plague current LLM evaluation.

### 5.6 Curriculum Learning: Does Difficulty Progression Help?

The final experimental series tests whether adaptive difficulty progression during RLVR training improves final model performance compared to fixed-difficulty training.

**Experimental Setup:**
- **Curriculum Condition:** Start with easiest level, increase difficulty when performance exceeds 70% over 20 training steps
- **Fixed-Difficulty Condition:** Sample uniformly from all difficulty levels throughout training
- **Training Duration:** Single epoch for both conditions
- **Evaluation:** 50 holdout examples from each difficulty level

**Training Dynamics (Figure 6):**

The reward trajectories reveal important patterns:
- Difficulty increases are often preceded by **sharp drops in reward**, reflecting the model encountering harder examples
- The curriculum model's lower terminal reward in some environments reflects exclusive exposure to maximum difficulty examples by training end
- For `Count Primes`, the curriculum model **never advanced past the initial difficulty level** (no vertical bars in Figure 6), yet still outperformed the non-curriculum model

**Results (Table 5):**

| Environment | Level | Baseline | Non-Curriculum | Curriculum | Improvement |
|-------------|-------|----------|----------------|------------|-------------|
| Spell Backwards | 4 (word length) | 12.00% | 30.00% | **70.67%** | **+40.67%** |
| Spell Backwards | 6 | 3.33% | 10.67% | **30.00%** | **+19.33%** |
| Spell Backwards | 8 | 0.00% | 1.13% | **3.37%** | **+2.24%** |
| Mini Sudoku | 4-6 (empty cells) | 1.13% | 54.00% | **56.00%** | **+2.00%** |
| Mini Sudoku | 8-10 | 0.00% | 6.67% | **20.00%** | **+13.33%** |
| Mini Sudoku | 10-12 | 0.00% | 1.13% | **5.33%** | **+4.20%** |
| Count Primes | 100-500 (range) | 12.00% | 4.00% | **30.67%** | **+26.67%** |
| Count Primes | 100-1000 | 3.33% | 5.03% | **12.67%** | **+7.64%** |
| Count Primes | 100-5000 | 1.33% | 0.00% | **3.03%** | **+3.03%** |

**Key Findings:**

1. **Curriculum Consistently Outperforms:** The curriculum-trained models achieve superior performance in all environments and difficulty levels. The margins are substantial at easier levels (+40.67% for Spell Backwards level 4) and remain meaningful at harder levels.

2. **Counterintuitive Count Primes Result:** Despite never advancing past the initial difficulty level, the curriculum model decisively outperforms the non-curriculum approach (+26.67% at level 100-500). This suggests that **attempting curriculum progression provides benefits even when progression fails** — possibly because training dynamics differ when the model knows harder examples await.

3. **Diminishing Returns at Higher Difficulty:** The improvement margins decrease at harder levels (Spell Backwards: +40.67% at level 4 → +2.24% at level 8). This suggests curriculum learning helps most when the model is building foundational capabilities, with less impact on already-challenging configurations.

**Assessment:** The curriculum results validate adaptive difficulty progression as more than a training optimization — it appears to fundamentally change *what* the model learns. However, the paper explicitly notes curriculum learning's value "may be more limited in environments where progression paths are ambiguous or difficult to formalize." The 70% threshold over 20 steps is a heuristic without ablation — optimal thresholds likely vary by task.

### 5.7 Critical Assessment of Experimental Design

**Strengths:**

1. **Rigorous Transfer Isolation:** The cross-domain experiments ensure models never see evaluation domain data during training. This cleanly isolates whether RLVR develops general reasoning capabilities versus domain-specific heuristics.

2. **Multiple Independent Runs:** All transfer experiments use three independent runs with reported standard errors (Tables 3, 4), providing robust estimates rather than single-run cherry-picking.

3. **External Benchmark Validation:** Testing on GSM8K, MATH, Big-Bench Hard, and MMLU-Pro validates that RG improvements transfer beyond RG's own tasks — addressing the "does this actually matter?" question.

4. **Granular Difficulty Analysis:** The easy vs. hard configuration comparison (Figure 3b, Appendix A.4-A.5) reveals the difficulty cliff phenomenon that aggregate scores would mask.

**Limitations:**

1. **Single Base Model:** All experiments use Qwen2.5-3B-Instruct. While this controls for confounding variables, it limits generalizability — we don't know if results hold for larger models or different architectures.

2. **Limited Training Steps:** Transfer experiments run for 500 steps, RG-Math for 800 steps. It's unclear whether longer training would produce continued improvements or diminishing returns.

3. **No Curriculum Hyperparameter Ablation:** The 70% threshold over 20 steps is not ablated. Different thresholds might produce better results, but this remains unexplored.

4. **Negative Transfer Unexplained:** The paper reports negative transfer cases (ARC −2.3%, Games→Logic −0.8%) but does not investigate why these occur. Understanding interference patterns would be valuable for training design.

5. **Compute Transparency:** While the paper reports ~1500 A6000 hours, it doesn't break down compute per experiment. This makes it difficult to assess the cost-benefit tradeoff of different training approaches.

**Overall Assessment:** The experiments convincingly support the paper's core claims:
- ✅ RG tasks are genuinely challenging (zero-shot results)
- ✅ RLVR training improves within-domain performance (intra-domain transfer)
- ✅ Skills transfer across domains (cross-domain transfer)
- ✅ Improvements extend to external benchmarks (external validation)
- ✅ Curriculum learning outperforms fixed-difficulty training

The main gap is mechanistic understanding — *why* does algorithmic training improve algebra performance by 29.1%? The paper demonstrates the effect but doesn't fully explain the underlying reasoning primitives being developed. This is a natural direction for future work enabled by RG's exposed data-generating process.

## 6. Limitations and Trade-offs

No research contribution is without constraints. This section examines the assumptions underlying REASONING GYM, the scenarios it does not address, the computational and scalability trade-offs involved, and the open questions that remain. Understanding these limitations is essential for researchers considering whether RG fits their use case and for identifying productive directions for future work.

### 6.1 Core Assumptions of the Approach

REASONING GYM rests on several foundational assumptions that shape what it can and cannot accomplish:

**Assumption 1: Verifiable Rewards Are Sufficient for Reasoning Development**

The entire RLVR framework assumes that binary accuracy signals ($R_{\text{acc}} \in \{0, 1\}$) plus a formatting reward ($R_{\text{format}} = 0.2$) provide adequate learning signals for developing reasoning capabilities. Section 7 acknowledges this limitation directly:

> "Verification functions, while comprehensive, may not capture all aspects of solution quality that humans consider important."

This assumption works well for tasks with unambiguous ground truth (mathematics, logic puzzles, algorithmic problems) but breaks down for reasoning tasks where:
- Multiple valid solution paths exist with different quality levels
- Partial credit should be awarded for correct intermediate steps
- Solution elegance or efficiency matters beyond correctness

**Assumption 2: Procedural Generation Captures Reasoning Diversity**

The paper assumes that parameterized procedural generators can adequately represent the space of reasoning challenges within each domain. Table 6 shows 100+ generators across 10 categories, but Section 7 notes:

> "Some reasoning domains, particularly those requiring extensive domain knowledge or creativity, are difficult to capture with procedural generators."

This assumption holds for structured domains (algebra, logic, games) but may fail for:
- Tasks requiring cultural or commonsense knowledge
- Open-ended creative reasoning (e.g., designing experiments, constructing arguments)
- Domains where the problem space cannot be easily parameterized

**Assumption 3: Cross-Domain Transfer Implies General Reasoning**

The interpretation of cross-domain transfer results (Table 2) assumes that performance improvements reflect development of general reasoning primitives rather than indirect effects. However, the paper does not fully rule out alternative explanations:
- Training may improve output formatting consistency, benefiting all tasks
- Models may learn better search strategies applicable across domains
- Improvements could reflect reduced hallucination rather than enhanced reasoning

The negative transfer cases (ARC −2.3% from algorithmic training, Games→Logic −0.8%) suggest transfer is not universal, but the paper does not investigate why interference occurs in some domain pairs.

**Assumption 4: Single-Turn Text Reasoning Generalizes to Multi-Turn Settings**

Section 7 explicitly states:

> "The current RG implementation focuses on single-turn, text-based reasoning and does not yet include multi-turn or multimodal reasoning tasks."

This assumption limits RG's applicability to:
- Agentic systems requiring multi-step interaction with tools or environments
- Vision-language models needing visual-spatial reasoning beyond text descriptions
- Dialogue-based reasoning where context accumulates across turns

The assumption may be reasonable for foundational reasoning research but restricts RG's utility for emerging agentic and multimodal applications.

---

### 6.2 Unaddressed Scenarios and Edge Cases

Several important reasoning scenarios fall outside RG's current scope:

**Domain Knowledge-Intensive Tasks**

Procedural generators struggle with tasks requiring extensive external knowledge. For example:
- Scientific reasoning requiring physics or chemistry principles
- Historical or legal reasoning requiring factual knowledge
- Medical diagnosis requiring domain-specific expertise

Section 7 notes that "procedural generators may struggle in domains where answers are unstructured, leaving room for alternative RL data." This creates a gap between RG's structured reasoning tasks and real-world reasoning scenarios that blend logical inference with domain knowledge.

**Open-Ended Creative Reasoning**

RG excels at tasks with verifiable answers but cannot handle:
- Generating novel proofs or mathematical conjectures
- Designing experiments or research proposals
- Creative problem-solving where multiple valid approaches exist

The algorithmic verifiability principle (P1) inherently excludes tasks where correctness cannot be automatically determined. This is a fundamental trade-off: verifiability enables scalable RLVR training but restricts the reasoning types RG can address.

**Visual-Spatial Reasoning in Text Format**

Section 3.2 identifies visual-spatial reasoning as a persistent challenge:

> "Tasks requiring visual-spatial reasoning represented in text format (cognition, games) prove particularly challenging, with even the strongest models achieving less than 50% accuracy."

Figure 5 shows the Games category plateauing early during training, suggesting fundamental difficulties in learning spatial reasoning from text representations. While RG includes tasks like `rubiks_cube`, `rush_hour`, and `arc_agi`, the text-based encoding may limit what models can learn compared to true multimodal inputs.

**Continual Learning and Catastrophic Forgetting**

Section 7 identifies this as future work:

> "Future work should examine continual learning settings where data arrives in non-stationary streams and investigate how regularization, model merging, and replay buffers affect the model performance under catastrophic forgetting."

The current experiments assume i.i.d. data sampling, which does not reflect real-world deployment scenarios where:
- Task distributions shift over time
- Models must retain old skills while learning new ones
- Training data arrives incrementally rather than all at once

This limitation is significant for practical applications but does not undermine RG's value for controlled reasoning research.

**Negative Transfer Mechanisms**

Table 2 documents negative transfer cases but does not explain them:
- Algorithmic training → ARC: −2.3%
- Games training → Logic: −0.8%

Understanding when and why transfer harms performance is as important as understanding positive transfer. The paper does not investigate whether:
- Conflicting reasoning strategies cause interference
- Some domains compete for model capacity
- Training order affects transfer outcomes

This remains an open question that RG's infrastructure could enable future work to address.

---

### 6.3 Computational and Scalability Constraints

**Training Compute Requirements**

Section 4 reports approximately **1500 A6000 hours** across all experiments. While modest compared to frontier model training, this represents a non-trivial investment:

| Experiment Type | Approximate Compute |
|-----------------|---------------------|
| Intra-domain transfer (5 categories) | ~375 hours |
| Cross-domain transfer (4 training domains × 10 test domains) | ~600 hours |
| External benchmark evaluation | ~300 hours |
| Curriculum learning experiments | ~225 hours |

For researchers without access to cloud GPU rentals, this compute requirement may be prohibitive. The paper does not report whether smaller-scale experiments (fewer training steps, smaller models) produce similar qualitative results, leaving uncertainty about minimum viable compute budgets.

**Generator Maintenance Overhead**

Maintaining 100+ procedural generators represents significant engineering effort:
- Each generator requires implementation, testing, and verification logic
- Difficulty parameters must be calibrated to produce meaningful gradients
- Edge cases in problem generation must be handled to avoid invalid instances

The open-source release mitigates this burden for users, but the maintenance cost limits how quickly the library can expand to new domains. Section 8 notes RG is released as "an open-source resource," but long-term maintenance commitments are not specified.

**Scalability to Larger Models**

All experiments use Qwen2.5-3B-Instruct, a 3 billion parameter model. The paper does not address whether RG's benefits scale to larger models:
- Do 7B, 13B, or 70B models show similar transfer patterns?
- Do larger models saturate faster on RG tasks, reducing training value?
- Does the difficulty cliff phenomenon persist at larger scales?

Section 3.1 shows frontier models (o3-mini, DeepSeek-R1) still struggle with hard RG configurations, suggesting room for improvement. However, without training experiments on larger models, it remains unclear whether RG provides diminishing returns as model capacity increases.

**Memory and Sequence Length Constraints**

The training configuration (Appendix A.6) specifies:
- Maximum prompt length: 4096 tokens
- Maximum response length: 2048 tokens

Some reasoning tasks may exceed these limits, particularly:
- Long-horizon planning problems requiring extended chain-of-thought
- Complex game states with large board representations
- Multi-step proofs with extensive intermediate derivations

The paper does not report how often tasks hit these limits or whether truncation affects learning quality.

---

### 6.4 Weaknesses in Experimental Design

**Single Base Model Limits Generalizability**

All RLVR experiments use Qwen2.5-3B-Instruct. While this controls for confounding variables, it limits claims about RG's broader applicability:
- Results may not generalize to different model architectures (e.g., MoE, decoder-only variants)
- The 3B scale may be uniquely suited to showing transfer effects (larger models may saturate faster)
- Instruction-tuned base models may respond differently than base-only models

Future work should replicate key findings across multiple model families and scales to establish RG's robustness.

**Limited Training Duration**

Transfer experiments run for 500 GRPO steps; RG-Math runs for 800 steps. The paper does not investigate:
- Whether longer training produces continued improvements or diminishing returns
- Whether different domains require different training durations
- Whether early stopping criteria could reduce compute without sacrificing performance

Figure 4 shows most categories still improving at step 400, suggesting training may be prematurely terminated. The plateau in Games (Figure 5) could reflect insufficient training rather than fundamental learning limits.

**Curriculum Hyperparameters Not Ablated**

The curriculum learning threshold (70% accuracy over 20 steps) is presented as a fixed heuristic without ablation:
- Would 60% or 80% thresholds produce better results?
- Would evaluating over 10 or 50 steps change outcomes?
- Do optimal thresholds vary by task domain?

Table 5 shows curriculum learning consistently outperforms fixed-difficulty training, but without hyperparameter ablation, it remains unclear whether the reported margins represent optimal curriculum design or merely one viable configuration.

**GSM8K Discrepancy Unexplained**

Table 3 shows a striking discrepancy:
- MATH: +9.7% improvement
- GSM8K: +0.5% improvement
- Big-Bench Hard: +7.66% improvement

The paper does not explain why GSM8K shows marginal gains while MATH shows substantial improvements. Possible explanations (not tested):
- GSM8K may be closer to saturation for 3B models
- GSM8K's arithmetic focus may not benefit from RG's diverse reasoning training
- The 8-shot CoT protocol may already maximize baseline performance

This unexplained variance weakens claims about RG's consistent external benchmark transfer.

**Standard Error Reporting Inconsistency**

While Tables 3 and 4 report standard errors for external benchmarks, the intra-domain and cross-domain transfer tables (Tables 1 and 2) do not include error bars despite Section 4.1 noting "three independent runs on identical evaluation sets." This makes it difficult to assess whether smaller improvements (e.g., Cognition +2.0% in Table 1) are statistically significant or within noise.

---

### 6.5 Open Questions for Future Research

RG's release enables several important research directions that the paper does not address:

**Question 1: What Reasoning Primitives Does RLVR Develop?**

The cross-domain transfer results suggest RG training develops generalizable capabilities, but the paper does not identify what these capabilities are:
- Do models learn better search strategies (e.g., systematic exploration, backtracking)?
- Do models develop improved verification habits (checking intermediate steps)?
- Do models learn to decompose problems more effectively?

RG's exposed data-generating process enables mechanistic analysis of these questions, but the paper does not pursue this analysis.

**Question 2: How Does Transfer Vary with Training Data Composition?**

The experiments train on single domains (algorithmic, logic, games) or math composites. Open questions include:
- Does multi-domain training produce better transfer than single-domain training?
- Are there optimal domain combinations that maximize cross-domain benefits?
- Does training order affect final capabilities?

RG's infrastructure supports investigating these questions, but they remain unexplored.

**Question 3: What Is the Relationship Between Difficulty Progression and Sample Efficiency?**

Curriculum learning improves final performance (Table 5), but the paper does not analyze:
- Whether curriculum training achieves the same performance with fewer samples
- Whether difficulty progression affects which reasoning strategies models learn
- Whether curriculum benefits persist with longer training durations

**Question 4: Can RG Support Multi-Modal or Agentic Reasoning?**

Section 7 identifies this as future work, but key questions remain:
- Can procedural generators produce vision-language reasoning tasks?
- Can RG environments support tool use and multi-turn interaction?
- How would verifiable rewards work for open-ended agentic behaviors?

Addressing these questions would significantly expand RG's applicability but requires substantial infrastructure development.

**Question 5: How Do Negative Transfer Cases Arise?**

The documented negative transfers (ARC −2.3%, Games→Logic −0.8%) suggest interference between some domain pairs. Understanding this phenomenon could inform:
- Which domains should be trained together vs. separately
- Whether curriculum design should avoid certain domain sequences
- Whether model architecture modifications could reduce interference

RG provides the infrastructure to investigate this systematically, but the paper does not pursue this analysis.

### 6.6 Summary: Trade-offs at a Glance

| Trade-off | Benefit | Cost |
|-----------|---------|------|
| **Verifiable Rewards** | Enables scalable RLVR training without human labels | Excludes open-ended reasoning tasks without clear ground truth |
| **Procedural Generation** | Unlimited data, no memorization risk | Requires significant engineering; cannot capture knowledge-intensive domains |
| **Single-Turn Text Format** | Simplifies verification and training | Cannot address multi-turn or multimodal reasoning scenarios |
| **Fixed Curriculum Threshold** | Simple implementation, consistent evaluation | May not be optimal across all domains; not ablated |
| **3B Model Scale** | Makes transfer effects visible; affordable compute | Limits generalizability to larger models used in production |
| **Clean Train/Test Separation** | Eliminates benchmark contamination concerns | May underestimate performance if training and evaluation domains overlap in practice |

These trade-offs are not flaws but deliberate design choices that position RG as a research infrastructure rather than a production-ready solution. Understanding them helps researchers determine when RG is the right tool for their questions and when alternative approaches may be more appropriate.

## 7. Implications and Future Directions

REASONING GYM (RG) represents more than a new dataset; it establishes a **new infrastructure paradigm** for reasoning research. By shifting from static data curation to dynamic environment generation, RG fundamentally changes how we train, evaluate, and understand reasoning capabilities in language models. This section outlines how this work reshapes the field, the specific research avenues it unlocks, and practical guidance for integrating RG into existing workflows.

### 7.1 Reshaping Evaluation and Training Standards

The most immediate impact of RG is the obsolescence of single-difficulty static benchmarks for *training* purposes. The "Difficulty Cliff" phenomenon (Figure 3b) demonstrates that aggregate scores on fixed datasets mask critical fragility in model capabilities. A model scoring 60% on easy configurations may drop to 10% on hard variants of the same task, indicating template matching rather than robust reasoning.

**Implication for Evaluation:**
Researchers must adopt **difficulty-profiled evaluation** rather than single-score reporting. RG enables this by exposing parametric difficulty controls (Appendix A.3). Future benchmarking should report performance curves across difficulty gradients (e.g., accuracy vs. polynomial degree or graph node count) rather than aggregate averages. This prevents the "dataset ceiling" problem where models saturate fixed benchmarks without genuine capability gains.

**Implication for Training:**
The success of curriculum learning in RG (Table 5) suggests that **adaptive difficulty progression** should become standard practice in RLVR. The paper shows that curriculum-trained models outperform fixed-difficulty models by margins up to +40.67% on specific tasks (Spell Backwards, level 4). This implies that training pipelines should dynamically adjust task complexity based on real-time performance metrics (e.g., the 70% threshold over 20 steps used in Section 5), rather than sampling uniformly from static datasets.

### 7.2 Enabled Research Avenues

RG's open-source infrastructure (released at https://github.com/open-thought/reasoning-gym/) enables several high-value research directions that were previously infeasible due to data scarcity or lack of verification.

#### 7.2.1 Mechanistic Interpretability of Reasoning Primitives
The cross-domain transfer results (Table 2) show that algorithmic training improves algebra performance by +29.1%, but the *mechanism* remains unexplained. RG's exposed data-generating process allows researchers to correlate specific model behaviors with task parameters.
*   **Research Question:** Do models develop shared computational subroutines (e.g., systematic search, backtracking, constraint propagation) that activate across domains?
*   **Method:** Use RG's parametric control to isolate specific reasoning demands (e.g., varying only graph depth while holding node count constant) and analyze activation patterns via mechanistic interpretability tools.

#### 7.2.2 Continual Learning and Catastrophic Forgetting
Section 7 explicitly identifies continual learning as a key future direction. RG's infinite data generation supports non-stationary data streams, which are essential for studying forgetting.
*   **Research Question:** How do regularization techniques (e.g., EWC, replay buffers) affect reasoning retention when task distributions shift over time?
*   **Method:** Stream RG tasks sequentially (e.g., Algebra → Logic → Games) and measure performance degradation on earlier domains. RG's verifiable rewards enable automatic monitoring of forgetting without human evaluation.

#### 7.2.3 Multi-Modal and Agentic Reasoning
The current RG implementation focuses on single-turn text reasoning (Section 7). Extending RG to multi-modal domains addresses the limitation that visual-spatial reasoning in text format remains challenging (Section 3.2).
*   **Research Question:** Can procedural generators produce vision-language tasks with verifiable rewards (e.g., diagram interpretation, spatial planning)?
*   **Method:** Adapt RG generators (e.g., `rubiks_cube`, `maze`) to output image-based states instead of text descriptions, enabling RLVR training for vision-language models (VLMs) with automatic verification.

#### 7.2.4 Negative Transfer and Interference
The paper documents negative transfer cases (e.g., Algorithmic → ARC: −2.3% in Table 2) but does not explain them. RG provides the infrastructure to systematically investigate interference.
*   **Research Question:** Which domain combinations cause interference, and can training order mitigate this?
*   **Method:** Conduct ablation studies on training sequences (e.g., Logic→Games vs. Games→Logic) using RG's modular generators to map the "transfer landscape" between reasoning domains.

### 7.3 Practical Applications and Downstream Use Cases

Beyond academic research, RG offers practical value for developing reasoning-capable systems in resource-constrained environments.

**Cost-Effective RLVR Training:**
The paper reports approximately **1500 A6000 hours** for all experiments (Section 4). Compared to the cost of human-curated datasets (e.g., paying experts for math problems), RG provides a scalable alternative.
*   **Use Case:** Startups or research groups without access to massive human annotation budgets can use RG to bootstrap reasoning capabilities before fine-tuning on domain-specific data.
*   **Benefit:** The RG-Math model achieved +9.7% on MATH and +7.66% on Big-Bench Hard (Table 3) using only procedural data, validating RG as a pre-training or mid-training infrastructure.

**Automated Evaluation Harness:**
RG's verification modules can serve as a local evaluation harness for debugging model failures before submitting to public leaderboards.
*   **Use Case:** During model development, run daily evaluations on RG's hard configurations to detect regressions in reasoning capabilities that fixed benchmarks might miss due to memorization.
*   **Benefit:** Immediate feedback on whether architectural changes improve genuine reasoning (measured by hard task performance) versus superficial pattern matching.

**Custom Domain Adaptation:**
Practitioners can extend RG's generator framework to create custom reasoning environments for specific applications (e.g., code verification, legal logic).
*   **Use Case:** A company building a code reasoning agent can implement a procedural generator for their specific codebase constraints, using RG's verification infrastructure to train specialized models.
*   **Benefit:** Leverages RG's RLVR pipeline while customizing the data distribution to match deployment needs.

### 7.5 Conclusion on Impact

REASONING GYM shifts the bottleneck of reasoning research from **data acquisition** to **environment design**. By providing over 100 verifiable, procedurally generated tasks, RG enables the community to move beyond the limitations of fixed datasets and explore the true potential of Reinforcement Learning with Verifiable Rewards. The evidence of cross-domain transfer (Section 4.2) and curriculum benefits (Section 5) suggests that RG is not merely a training tool but a **discovery engine** for understanding how reasoning capabilities emerge and generalize in language models. As the field progresses toward more capable agentic and multi-modal systems, RG's infrastructure provides a foundational layer for scalable, verifiable, and systematic reasoning development.
